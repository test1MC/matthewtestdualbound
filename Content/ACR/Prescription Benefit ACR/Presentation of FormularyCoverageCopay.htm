Presentation of Formulary Alternatives Coverage and Copay

4.10 At a minimum, the application shall display, without user action, for on-formulary/non- preferred, non-formulary, or non-reimbursable drugs:

  1. All alternatives that have a higher formulary status in the Formulary Alternatives List i. For Non-Formulary or Non-Reimbursable drugs, show alternatives that are
    1. On-Formulary/Non-Preferred or On-Formulary/Preferred.
    2. For On-Formulary/Non-Preferred drugs, show On-Formulary/Preferred.
  2. The alternatives identified in a formulary alternatives list shall be ordered according to the stated preference level indicated in that list.
  3. All commercially available, generic and branded alternatives with a higher formulary status in the same therapeutic class as the target drug.
    1. For Non-Formulary or Non-Reimbursable drugs, show alternatives that are On-Formulary/Non-Preferred or On-Formulary/Preferred.
    2. For On-Formulary/Non-Preferred drugs, show On-Formulary/Preferred.
  4. The default sort order for therapeutic alternatives shall be to order by formulary status from highest to lowest with Unknown formulary status last.
  5. The application shall display formulary status and abbreviated retail copay data for all alternatives. Abbreviated retail copay data displayed shall be, at a minimum, the tier, flat amount or percent rate.

To view the complete list of alternatives, the application may allow the user to scroll.

4.13 Using the representative NDC provided by the Formulary Publisher, the application shall display all strengths and dosage forms of alternatives provided in the formulary alternatives list.

4.10 For active coverages, during the prescription writing process and prior to the summary screen, the application shall display, at a minimum, alternative medications when the target medication is an On-Formulary/Non-Preferred, Non-Formulary, or Non-Reimbursable drug.

Best Practice

  • The application may further filter alternatives by utilizing decision support logic (e.g. only showing alternatives applicable due to gender, age, diagnosis, etc.).
  • If the target medication is a brand drug and the prescriber indicates that generic substitution is permitted, the application does not need to show alternatives if the generic equivalent has a higher formulary status.

Payer-Provided Alternatives

  1. If the payer provides a Formulary Alternatives List, the application shall show all alternatives that have a higher formulary status.
    1. For Non-Formulary or Non-Reimbursable drugs, show alternatives that are On-Formulary/Non-Preferred or On-Formulary/Preferred.
    2. For On-Formulary/Non-Preferred drugs, show On-Formulary/Preferred.
    3. Do not show alternatives for Unknown drugs.
  2. Show all strengths available that have a higher formulary status for the NDC provided in the Formulary Alternatives List.
  3. The alternatives provided in a Formulary Alternatives List shall be ordered according to the stated preference level in that list from high to low.
  4. Show all alternatives in the Formulary Alternatives List, even if the target medication is an OTC, multi-ingredient medication, or supply.
  5. The application shall not show drug database-provided alternatives if the PBM provides a Formulary Alternatives List.
  6. The application shall display formulary status and full or abbreviated retail copay data for all alternatives. Retail copay (defined as Pharmacy Type Retail, Any, or Null) data displayed shall be, at a minimum, the Copay Tier, Flat Copay Amount, or Percent Copay Rate.

If the application cannot display the complete list of alternatives on one page, the application may allow the user to take action to view additional alternatives.

Drug-Database Provided (Therapeutic) Alternatives

  1. If the PBM does not provide a Formulary Alternatives List, the application shall show all commercially available, generic, and branded alternatives with a higher formulary status in the same therapeutic class as the target drug as determined by an approved drug data source.

    1. For Non-Formulary or Non-Reimbursable drugs, show alternatives that are On-Formulary/Non-Preferred or On-Formulary/Preferred.

    2. For On-Formulary/Non-Preferred drugs, show On-Formulary/Preferred.
    3. Do not show alternatives for Unknown drugs.

     Best Practice

    • If the target medication is OTC, the application should not display alternatives.
    • If the target medication is a Supply, the application should not display alternatives.
    • If possible, the application should only show alternatives for the same route of administration as the target medication.

  1. If the target medication is single-ingredient, the application shall only display single-ingredient alternatives.
  2. If the target medication is multi-ingredient, the application shall only display multi-ingredient alternatives.
  3. The application shall display formulary status and full or abbreviated retail copay data for all alternatives. Retail copay (defined as Pharmacy Type Retail, Any, or Null) data displayed shall be, at a minimum, the Copay Tier, Flat Copay Amount, or Percent Copay Rate.

If the application cannot display the complete list of alternatives on one page, the application may allow the user to take action to view additional alternatives.
