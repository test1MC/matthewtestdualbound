Relates To Message ID

Message ID

M/C

Description/Comments

New Rx

C

Set by the prescriber if the New Rx is a follow-up message to a previously sent Denied New to Follow Refill Renewal Response. The value shall*106R-076 be set to the MessageID of that Refill Renewal Response. This will help the pharmacy close the business transaction started with a Refill Renewal Request.

Refill Renewal Request

C

If the Refill Renewal Request is being made for a prescription that was previously created from an electronic transaction (could have been a New Rx, Refill Renewal Response Approved or Approved with Changes or Rx Change Response Approved or Approved With Changes), the pharmacy shall*106R-077 set the value to the MessageID of that transaction. This will help the prescriber link the request to a previously approved transaction.

Refill Response

M

Shall*106R-078 always be set to the MessageID of the corresponding Refill Renewal Request being responded to.

Rx Change Request

C

If the Rx Change Request is being made for a prescription that was previously created from an electronic transaction (a New Rx, Refill Renewal Request Approved or Approved with Changes or Rx Change Response Approved or Approved With Changes), the pharmacy shall*106R-079 set the value to the MessageID of that transaction. This will help the prescriber link the request to a previously approved transaction.

Rx Change Response

M

Shall*106R-080 always be set to the MessageID of the corresponding Rx Change Request being responded to.

Rx Cancel Request

C

If the Rx Cancel Request is being made for a prescription that was previously created from an electronic transaction (a New Rx, Refill Renewal Request Approved or Approved With Changes or Rx Change Response Approved or Approved With Changes), the prescriber shall*106R-081 set the value to the MessageID of that transaction. This will help the pharmacy link the request to a previously approved transaction and cancel the correct prescription.

Rx Cancel Response

M

Shall*106R-082 always be set to the MessageID of the corresponding Rx Cancel Request being responded to.

RxFill

C

If the Rx Fill is being sent for a prescription that was previously created from an electronic transaction (a New Rx, Refill Renewal Request Approved or Approved With Changes or Rx Change Response Approved or Approved With Changes), the pharmacy shall*106R-083 set the value to the MessageID of that transaction. This will help the prescriber link the RxFill to a previously approved transaction. Surescripts does not support this message.

Resupply

C

If the Resupply is being made for a prescription that was previously created from an electronic transaction (a New Rx), the facility shall*106R-084 set the value to the MessageID of that transaction. This will help the pharmacy link the request to a previously approved transaction. Surescripts does not support this message.

Status

M

Shall*106R-085 always be set to the MessageID of the corresponding transaction being acknowledged.

Verify

M

Shall*106R-086 always be set to the MessageID of the corresponding transaction being acknowledged.

Error

M

Shall*106R-087 always be set to the MessageID of the corresponding transaction being errored.
