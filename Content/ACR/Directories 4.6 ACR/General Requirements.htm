General Requirements

Directories – General Application Certification Requirements

  1. Participant shall*46D-002 not reject incoming (e-prescribing or clinical) messages due to lack of appropriate entries within the participant’s local directories.
  2. In the event a participant has local prescriber/pharmacy directory information in addition to the Surescripts prescriber/pharmacy directory information, the participant shall*464D-005 utilize the Surescripts prescriber/pharmacy directory information for all prescriptions routed through the Surescripts network.
  3. Prescriber participants: While integrating data from the Pharmacy Organization directories, NCPDP ID shall*46D006 be used as a key element for pharmacy records. The following fields in the download shall*46D-007 remain non-editable.
  • NCPDP ID
  • Store Number
  • Store Name (A friendly store name for the pharmacy may be established but that shall*46D-008 not replace or eliminate the pharmacy name as supplied in Surescripts directory. Any updates received from Surescripts shall*46D-009 still be accepted at the NCPDP level irrespective of the ‘friendly store name’ applied).
  • Address
  • Fax
  • Service Level
  1. Pharmacy participants: While integrating prescriber directories, prescriber NPI or DEA shall*46D-010 be used as a key element for prescriber records. The following fields in the downloaded Surescripts directory shall*46D-011 remain non-editable.
  • SPI
  • Prescriber First Name
  • Prescriber Last Name
  • NPI and/or DEA
  1. Participants shall*46D-012 automate the process to download new and updated directory information within 24 hours. Participants shall*46D-013 have a process to ensure that all endpoints successfully receive the downloaded information within 24 hours. If a manual matching process is required, directory information shall*46D-014 be updated within 72 hours.
  2. Participant directory information shall*46D-015 be uploaded within 24 hours of any additions or changes to the participant (e.g. provider, organization) information utilizing the Surescripts Admin Console or Directory Messaging.
  3. Participants shall*46D-016 check the service level of the intended receiver before routing a message through Surescripts. The intended recipient shall*46D-017 have the service level that enables them to receive the given message type.
