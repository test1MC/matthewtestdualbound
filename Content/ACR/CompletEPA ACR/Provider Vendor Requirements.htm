Provider Vendor Requirements

For customers (participants) using CompletEPA® Accelerator, the items marked with * are certified by Surescripts on your behalf.

2.1 The application shall make available for display the <ResponseStatus> and each occurrence of a <PANote> from all response transactions (PAInitiationResponse, PAResponse, PAAppealResponse, and PACancelResponse). In cases where the PBM is not the PA processor or enabled for CompletEPA®, there may be multiple PAInitiationResponses for a single PAInitiationRequest and the application shall make available for display the <PANote> from all PAInitiationResponse(s).*

2.2 When the PBM or payer provides a question set in the PAInitiationResponse, the application shall display:

  1. <QuestionSetTitle>,
  2. <QuestionSetDescription>,
  3. <QuestionSetContactCommunicationNumber>, and
  4. <DeadlineForReply>.*

2.3 The application shall support the ePA question set conditional logic as outlined in the NCPDP ePA standard.*

2.4 The application shall require all specified questions to be answered prior to sending the PARequest.*

2.5 The application shall require users to start the ePA process from the beginning if they want to reinitiate an ePA that was cancelled by the providervendor or closed by the PBM/Payer.

2.6 The application shall not allow users/systems to delete an ePA task awaiting a response from the PBM/Payer, but shall send a PACancelRequest transaction.*

2.7 Provider vendors who are using Admin Console to identify PBMs/Payers with an ePA service level shall update their local PBM/Payer list on a weekly basis, at a minimum.

2.8 For all request message types, when someone other than the prescriber is the main contact and has a different <PrimaryTelephone> than the prescriber, the application shall populate the Provider composite with the <Name> and <PrimaryTelephone>.

2.9 Provider vendors shall not send a PAAppealRequest if the value of the <IsEAppealSupported> field is set to N on the PAResponse.*

2.10 Provider vendors shall populate the following fields in the PAInitiationRequest transaction when returned in the X12 Eligibility Response (271):

  • BIN
  • PCN
  • Plan ID
  • Group Number
  • PBM Member ID
  • PBM Participant ID

2.11 <Prescriber> <Address>: For ePA messages, this address shall match what is in the directory for this provider.

2.12 <DrugDescription> The item or drug description field shall include the full drug name, strength, and form (in that exact sequence).

2.13 <DrugCoded.ProductCode> When sending, the NDC must be an 11-digit numeric NDC in the 5-4-2 format. Dashes shall not be used and leading zeros shall NOT be suppressed.

2.14 If the <DrugCoded.Strength> composite is sent, the values shall be equivalent to the corresponding portions of the <DrugDescription> field (strength, unit of measure, and dose form).

2.15 <PotencyUnitCode> Quantity Unit of Measure Code Qualifiers shall be correctly associated with drug descriptions.

2.16 The drug name shall be an item that is currently available for dispensing in the marketplace. Participants are expected to work with their drug database vendor to satisfy this requirement.

2.17 <AddressLine1> PO Box information shall not be sent in this field.
