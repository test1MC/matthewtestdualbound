Response Delimited File

The response file format will be pipe-delimited (hex7c). Each field is delimited by the pipe character (|). If a pipe-character (hex7c) is used in field data, it will be escaped with \F\. For example, Main | Street will be listed as Main \F\ Street. Each line will be separated by a new line (Hex 0A) character.

W04

Change Record not found, record added.

This warning appears when a full file load was sent.

For example, if the Transmission Action is ‘F’ull, and one or more rows in the file had a Transaction Type value ‘001’, indicating a change to the record. The processor removes all previous records before loading the full file and thus it cannot find the original record that was supposed to be changed. The data from the row with the ‘001’ value is still loaded, so the net effect is the same as updating a record.

This warning is also triggered if the Transmission Action is ‘U’pdate, and the processor cannot find the original record that should be updated. The data from the row with the ‘001’ value is still loaded, but it does not replace any existing records.

W05

Record to add exists, record updated.

W08

Duplicate Patient ID(s) found in file. The file load process will load only one record with a given Patient ID and will ignore the remaining duplicate record(s).
