Adherence Profile Summary

The adherence-profiles/summary provides the member’s lowest PDC score and associated disease class. The patient’s unique PBM Id and payer Id obtained from an eligibility transaction are passed as parameters on this request.

Workflow Considerations

The adherence-profiles/summary API is lightweight and useful when only the lowest PDC score is displayed to the user such as on a dashboard.

The diagram below is an example of where the adherence-profiles/summary API could be used in a typical patient visit workflow.

 

The response to this API will return the patient’s lowest PDC value and disease class.

PDC values are categorized as:

  • PDC  value  <= 60%, low
  • PDC value  > 60% and < 80%, moderate
  • PDC value => 80%, healthy

[GET] {{host}}/mmpoc/api/v0/adherence-profiles/summary?memberId={Patient PBM Unique ID]&payerId={Payer Id}

Patient Worklist

Retrieving a worklist by patient retrieves a paged list of the active tasks for a given patient.  This API also returns a count of messages for the patient.

Workflow Considerations

The worklist API is a prerequisite to the CommunicationRequest API as this is where the task Id needed for the CommunicationRequest API is retrieved.

Communication Request

The communication request API retrieves a single message and the associated questionnaire.  The response includes the endpoint link to be used when posting the QuestionnaireResponse.  The QuestionnaireResponse endpoint link should be saved in your system and used as the endpoint when the end user submits a response.

Workflow Considerations

It is recommended to show the medication management message detail, questionnaire and the adherence profile in areas of the workflow where the provider will take action on non-adherence or gaps in care. 

  • The CommunicationRequest API response contains the medication management message detail. 
  • The Observation API response described in the next section of this document contains the member’s adherence profile (PDC scores).

The diagram below illustrates where the CommunicationRequest API should be used in a typical patient visit workflow.
