Worklist Summary

The worklist/summary provides summary information of task counts for a provider or a patient. The API takes an array of recipients or a patient and a products object, which can include all or just one product. If product is not sent, the customer will get back the task counts for all products implemented.

Workflow Considerations

The worklist summary API is lightweight and useful when only the count of messages is displayed to the user such as on a dashboard.

The diagram below is an example of where the worklist/summary API could be used in a typical patient visit workflow.
