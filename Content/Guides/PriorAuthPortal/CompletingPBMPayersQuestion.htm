Completing a PBM/Payer’s Question Set

The initial page for responding to a Question Set contains information that was submitted in the initial request as well as new information.

  1. Task/Patient Header – Displays information regarding the specific ePA task that is open. Includes patient name, medication requested and the PA Case ID (assigned by the PBM/payer).
  2. Question Set Information – Displays the title and subtitle of the question set.
  3. Information Area –Information specific to the ePA case is displayed within different boxes within the task screen. The details shown contain new information from the PBM/payer, potentially including payer contact information, notes and the deadline for reply. The Patient Information and Provider Information boxes contain data that was sent in the initial request.
  4. Start Button – By clicking the Start button, users are able to view and respond to the question set sent by the PBM/payer.

Users may need to scroll down on the dialog box in order to see all content and options.

  1. Options Drop-Down – From the Option’s drop-down menu, users are able to access task history for the current ePA task. Additional information regarding the current ePA including status, actions, previously answered questions, and required action can be found on the Task History screen.

Starting the Question Set/Responding to Questions

  1. Question Number – This represents the current question number being displayed.

Question set numbers may not be sequential and do not indicate progress towards completion.

  1. Progress Bar – Shows the user their progress through the current Question Set.
  2. Question – This shows the user the current question that the user is answering. Question types are based on the National Council for Prescription Drug Programs, (NCPDP) standard; see the Question Types section for more details on the different question types.
  3. Navigation Buttons – These allow the user to go back or forward one question.
  4. Options Drop-Down – The Options drop-down menu allows the users to navigate through the portal as needed.

After opening a task, the user can click the Start button to be presented with the PBM/payer’s first question. Question Sets are intended to have built-in logic that prevents users from being presented with irrelevant questions. Please note that content and complexity of the Question Set logic will vary from PBM/payer to PBM/payer. The progress bar indicates current progress towards completing the Question Set.

User sessions time out after a period of one hour, regardless of progress or action being taken within a task.

Navigating to the next question will automatically save progress on previously answered questions. Progress will not be automatically saved by clicking on Task History or Return to Worklist.

Different options are presented for answering a question based on the question type, see the Question Types section for more details on the different question types. The application performs data type validation on responses before allowing them to be saved or submitted to the PBM/payer. This means that a single select question will not allow users to submit multiple selections and a number type question will be limited to numerical values – the application does not check for validity of response, but merely follows the question logic provided by the PBM/payer within the Question Set.

Saving Progress

Question Set progress is saved based on the user's last answered question. This does not submit the Question Set to the PBM/payer but allows the user to return to finish at a later point or allows another user to finish responding.

Question Types

  • Select (Multi) – Indicates the answer to the question is to be selected from a set of choices. Check boxes are displayed to indicate that multiple responses may be selected.
  • Select (Single) – Indicates the answer to the question is to be selected from a set of choices. Radio buttons are displayed to indicate that a single response may be selected.
  • Date – Indicates the answer to the question is a date.
  • Numeric – Indicates the answer to the question is numeric. Conditional logic provided by the PBM/payer is used to determine the next question based on an exact value provided or upper/lower bound(s).
  • Free Text – Indicates the answer to the question is free text.
  • Additional Comments – PBM/payers can indicate if free text comments are allowed and/or required for specific responses to questions.

  • Attachments – PBM/payers can indicate additional documentation be submitted via an attachment (lab results, PACS imaging, chart notes, etc.) for a specific response to a question.

Reviewing & Submitting a Completed Question Set

  1. Question Set Questions – The questions and their number within the Question Set are provided here. Only questions that were displayed to the end user and answered will be displayed.
  2. Navigation/Submit Buttons – These buttons allow the user to navigate back to the most recently answered question or to submit all of their responses to the PBM/payer.
  3. Question Set Response – The user is able to see all of their responses to questions presented within the Question Set. To modify a previously provided response, the user is able to select the answer and will be brought to that specific question.

  4. Options Drop-Down – The Option’s drop-down menu allows the user to go to the Task History screen for the current ePA task.

  5. Attachments – Additional documentation can be added as an attachment with the completed question set.

  6. Comments – The user is able to enter optional comments for communication with the PBM/Payer.

Upon completion of the Question Set, a review/submission screen is presented to the user. Users must carefully review all responses to ensure accuracy and to prevent delay in PBM/payer processing. If a response needs to be changed, users can click the Back button at the bottom of the page to navigate to the most recent question, select the specific response for a particular question to go back and make changes or click the Start Over button to begin the Question Set from the beginning.

Due to branching Question Set logic, if a user changes a previously provided response, they may be required to answer additional questions.

Once responses have been verified, click the Submit button to send the completed Question Set to the PBM/payer.
