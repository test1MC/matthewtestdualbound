SPO Clinical Information Request/Response

The SPO Clinical Information Request/Response uses the clinically oriented standard FHIR® release 3 (STU) for structuring the message exchange.

HTTP Request Header

The FHIR® messages delivered to the Surescripts SPO Clinical Information FHIR® service must include the HTTP request headers listed below.

Content Types and Encodings

According to Section 2.21.0.6 of the FHIR® specification Content Types and encodings,

“The formal MIME-type for FHIR® resources is application/fhir+xml or application/fhir+json. The correct mime type SHALL be used by clients and servers.” Later in that same section, “FHIR® uses UTF-8 for all request and response bodies. Since the HTTP specification (section 3.7.1) defines a default character encoding of ISO-8859-1, requests and responses SHALL explicitly set the character encoding to UTF-8 using the charset parameter of the MIME-type in the Content-Type header. Requests MAY also specify this charset parameter in the Accept header and/or use the Accept-Charset header.”

Because of these requirements, requests/responses will need to include a Content-Type in the HTTP header. Below is an example:

  • Content-Type: application/fhir+json; charset=utf-8

Authorization

Surescripts will use basic authentication as the security scheme for these HTTP based transactions. Therefore, standard basic authentication credentials will need to be provided in the Authorization header.
