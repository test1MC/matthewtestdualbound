SPO Clinical Information Request Resource

Message Resources

This section provides guidelines for the data messaging interfaces between Surescripts and the provider vendor system.

For the complete set of requirements for each FHIR® resource see the FHIR® STU3 standards specifications here: https://www.hl7.org/fhir/resourcelist.html.

Resource Data Elements

This section contains message resource data elements used in the SPO Clinical Information Request.

  Observation

valueQuantity.value

1..1

Quantity should be used for single-valued numeric results such as height, weight, numeric lab results, etc.

Quantity should be used for lab results when the result is specified (e.g., "5").

C

  Observation

valueQuantity.code

0..1

When using the Quantity type to specify the value, the unit property should be specified if the numeric value has units.

If the number requires units, the value has to be a UCUM code.

For information about units, see Appendix A: Units for Lab Tests on SPO Enrollment Forms.
