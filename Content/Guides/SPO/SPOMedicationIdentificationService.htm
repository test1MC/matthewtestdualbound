SPO Medication Identification Service Request/Response

The SPO Medication Identification Service is a request/response message that allows provider vendor systems to determine if the prescribed medication is supported by SPO.

Provider vendors should enable customers with the ability to manually initiate the SPO process for medications even when they are not identified as an SPO medication.

Authentication

For information on authentication, see the User Security section.

SPO Medication Identification Service Response Example

A successful response contains a status code of 200 with a body similar to the following.

Medications that are identified as an SPO Medication are marked with a value of "true" in the IsSpecialty element if it is an SPO medication, or "false" if it is not.

HTTP/1.1 200 OK

Cache-Control: no-cache

Pragma: no-cache

Content-Type: application/json; charset=utf-8

Expires: -1

Server: Microsoft-IIS/10.0

X-AspNet-Version: 4.0.30319

X-Powered-By: ASP.NET

Date: Fri, 08 Dec 2017 18:55:46 GMT

Content-Length: 273

    

[

  {

    "NDC":"65757040303","RXCUI":"2315616-","IsSpecialty":true

  }

]
