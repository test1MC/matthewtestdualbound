List Processes

The following endpoints are available to retrieve a list of processes based on parameters and search criteria. Each item in the list is a summary of a process. The responses are the same for all List Processes endpoints.

List Process for Patient ID

List Processes for patientId retrieves a paged list of process summaries for a given patient ID. If the optional parameter of Facility is not provided in the request, then only the processes that are not affiliated with a facility will be included in the results.

Request Template

[GET] {{host}}/api/v0/processes?patientId={patientId}&facility={facility} &fromDate=

{fromDate}&createdThruDate={createdThruDate}&pageIndex={pageIndex}&pageSize=

{pageSize}

Request Parameters

M/C – Mandatory or Conditional

Parameter

Data Type

M/C

Description

patientId

an..35

M

Matches the MedicalRecordIdentificationNumberEHR field (…Patient/Identification/MedicalRecordIdentificationNumberEHR) specified in the SPOInitiationRequest.

This field is optional in the schema, but has to be included in the original SPOInitiationRequest in order for the resulting process to be visible when launching the Accelerator for a specific patient.

facility

an..35

C

The value sent in the MutuallyDefined field (…Patient/Identification/MutuallyDefined) of the SPOInitiationRequest. Used to identify a patient with the same ID across different locations/facilities.

Note: If this optional field is not included in the SPOInitiationRequest, it should not be included in Auth Token Request for the specified patient. The worklist items will show when patient id equals MedicalRecordIdentificationNumberEHR and facility equals MutuallyDefined. For example, if facility is "null" and MutuallyDefined is "test clinic", the task for that SPO Initiation Request will not be displayed.

fromDate

DateTime

C

Processes created on or after this date will be returned.

Note: DateTime format is CCYY-MM-DDTHH:MM:SSZ

createdThruDate

DateTime

C

Processes created on or prior to this date will be returned.

Note: DateTime format is CCYY-MM-DDTHH:MM:SSZ

pageSize

Positive Int

C

Integer representing the number of records returned in each request. If a pageSize is not specified, the default value of 25 is applied. This value is truncated to 100 if larger.

pageIndex

Positive Int

C

Integer representing the page index of data to be returned. If a pageIndex is not specified, the default value of 0 is applied.

Request Example

[GET] {{host}/uiservice/api/v0/processes?patientId=george123?facility=oakclinic77

List Process for Initiator(s)

List Processes for Initiator(s) retrieves a paged list of process summaries for the specified initiator(s). An initiator is an individual or physician that requested the SPO enrollment.

Request Template

[POST] {{host}}/processes

Request Parameters

M/C – Mandatory or Conditional

Parameter

Data Type

M/C

Description

initiators

Complex Type

M

A collection of initiators.

An initiator is a dataset containing the initiators identifying data.

  id

an13

M

SPI of the initiator.

  qualifier

an1

M

SPI's corresponding qualifier.

Value = D

createdFromDate

DateTime

C

Processes created on or after this date will be returned.

Note: DateTime format is CCYY-MM-DDTHH:MM:SSZ

createdThruDate

DateTime

C

Processes created on or prior to this date will be returned.

Note: DateTime format is CCYY-MM-DDTHH:MM:SSZ

pageSize

Positive Int

C

Integer representing the number of records returned in each request. If a pageSize is not specified, the default value of 25 is applied. This value is truncated to 100 if larger.

pageIndex

Positive Int

C

Integer representing the page index of data to be returned. If a pageIndex is not specified, the default value of 0 is applied.

Request Example

[POST] http://[host]/uiservice/api/v0/processes

  "pageSize": 3

}

List Processes for InitiatorsProcessId(s)

List Processes for initiatorsProcessId(s) retrieves a paged list of processes for the specified initiatorsProcessId(s). InitiatorsProcessId is the SPOReferenceID as defined by the provider vendor system.

Request Template

[POST] {{host}}/processes

Request Parameters

M/C – Mandatory or Conditional

Parameter

Data Type

M/C

Description

initiatorsProcessId

Complex Type

M

A collection of initiators process IDs.

fromDate

DateTime

C

Processes created on or after this date will be returned.

Note: DateTime format is CCYY-MM-DDTHH:MM:SSZ

createdThruDate

DateTime

C

Processes created on or thru this date will be returned.

Note: DateTime format is CCYY-MM-DDTHH:MM:SSZ

pageSize

Positive Int

C

Integer representing the number of records returned in each request. If a pageSize is not specified, the default value of 25 is applied. This value is truncated to 100 if larger.

pageIndex

Positive Int

C

Integer representing the page index of data to be returned. If a pageIndex is not specified, the default value of 0 is applied.

Request Example

[POST] http://[host]/uiservice/api/v0/intiators/processes

{

  "initiatorProcessIds": [

    "f050fc8601444af1be3c5ffeb4c1d703",

    "3fb345ccf3814adc835f756e752c851a"

}

Collection of process summary objects. See Response Field Description – Process Summary.

Response Field Description – Process Summary

Field

Description

processId

Unique Surescripts ID which represents the process instance.

processName

Name of the process.

initiatorsProcessId

The initiatorsProcessId is the SPOReferenceID as defined by the provider vendor system.

initiator

Dataset containing the recipient's identifying data.

Field(s) include:

id

qualifier

name

Note: Supported qualifier is D (indicating the ID is an SPI).

patientId

Matches the MedicalRecordIdentificationNumberEHR field (…Patient/Identification/MedicalRecordIdentificationNumberEHR) specified in the SPOInitiationRequest.

facility

The value sent in the MutuallyDefined field (…Patient/Identification/MutuallyDefined) of the SPOInitiationRequest. Used to identify a patient with the same ID across different locations/facilities.

patientFirstName

Patient’s first name.

patientLastName

Patient’s last name.

patientDob

Patient’s date of birth.

createdDate

Date/time process was created.

completedDate

Date/time process was completed.

status

Integer representing the status of the process, possible values are:

1: Draft

2: In Progress

3: Complete

4: Cancelled

5: Errored

statusName

Textual name representing the status of the process.

description

Description of the process instance.
