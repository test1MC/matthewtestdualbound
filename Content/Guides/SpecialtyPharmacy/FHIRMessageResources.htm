FHIR® Message Resources

The Specialty Patient Enrollment product uses the clinically-oriented standard FHIR® release 3 (STU) for structuring the message exchange.

HTTP Request Header

These FHIR® messages must include the HTTP request headers listed below.

Content Types and Encodings

Requests/responses need to include a Content-Type in the HTTP header. Below is an example:

  • Content-Type: application/fhir+json; charset=utf-8

Authorization

Surescripts will use Mutual Transport Layer Security (mTLS) as the security scheme for these HTTP based transactions.

Post Body Information

Messages will contain a number of FHIR® resources. Listed below are links to the FHIR® documentation for the resources.

If additional elements are requested by the Specialty Patient Enrollment messages that should not be a breaking change.
