Messages Overview

Specialty Patient Enrollment supports the ability to communicate with pharmacies using structured FHIR® messages containing the information captured as part of the Specialty Patient Enrollment process. The content of this message is a FHIR® bundle object, where the Bundle is "message". The Bundle contains a collection of entries – a MessageHeader followed by other FHIR® objects that hold the information captured and/or sent by the provider vendor to Specialty Patient Enrollment.

For more information on these FHIR® structured messages, see the FHIR® Message Resources section.

Message Descriptions

NewRx – New Prescription Request

The NewRx message for Specialty Patient Enrollment will follow the NCPDP standard prescription routing requirements with the addition of requiring the <TertiaryIdentifier> (UIB-030-03) in the UIB – Interactive Interchange Control Header to be populated with a value of "SPO". This identifier will be used as an indicator to the pharmacy to expect an SPOEnrollmentRequest message. The <PrescriberOrderNumber> (UIH-030-01) in the NewRx message will be used to tie the NewRx to the SPOEnrollmentRequest.

The NewRx message flow is transmitted prior to the Specialty Patient Enrollment for Pharmacies workflow and is not included in the message flow examples below.

SPOEnrollmentRequest

The SPOEnrollmentRequest is sent from Surescripts to the specialty pharmacy for the enrollment of the patient that was prescribed a specialty medication. The SPOEnrollmentRequest provides the ability to communicate with pharmacies with a structured FHIR® message containing the information captured as part of the Specialty Patient Enrollment process.

SpecialtyEnrollmentNote

The SpecialtyEnrollmentNote is a structured FHIR® message that is sent from the specialty pharmacy to the provider to request additional enrollment information for the patient.

SpecialtyEnrollmentNoteResponse

The SpecialtyEnrollmentNoteResponse is a structured FHIR® message that is sent from the provider vendor to the specialty pharmacy and contains the additional information requested from the specialty pharmacy to complete the enrollment of the patient.

SPOEnrollmentRequest Message Flow

1) Surescripts sends the SPOEnrollmentRequest to the specialty pharmacy.

2) The specialty pharmacy responds with acknowledgement message or error message.

SpecialtyEnrollmentNote Message Flow

This message flow only occurs when the specialty pharmacy requires additional information from the provider to dispense the medication. This process could be repeated as often as necessary to dispense the medication.

1a) The specialty pharmacy system sends the SpecialtyEnrollmentNote to Surescripts to request additional information from the provider to dispense the medication.

1b) Surescripts forwards the SpecialtyEnrollmentNote to the provider.

2a) The provider reviews the request and enters a response. The response is sent to the specialty pharmacy using the SpecialtyEnrollmentNoteResponse.

2b) Surescripts forwards the SpecialtyEnrollmentNoteResponse to the specialty pharmacy system.

Message Validation

Surescripts will ensure that customers are in compliance with the message specifications outlined in this Guide during Integration Testing and will continue to enforce once in production.

At a minimum, Surescripts validations include:

  • JSON schema validation

  • The sender identification and authentication

  • The recipient identification

  • Syntax of the message, including field lengths, data types, and code values

  • Surescripts business rules
