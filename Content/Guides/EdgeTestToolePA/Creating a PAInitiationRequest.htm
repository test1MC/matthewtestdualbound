Creating a PAInitiationRequest

Accessing the ePA Message Composer


The PAInitiationRequest message is the first message type in the back and forth communication that happens between prescribers and PBM/payers. For the purposes of testing question sets and schema validation, PBM/payers using the Edge Test Tool must start the process by sending a PAInitiationRequest to their payer ID.

Allowed recipients are defined by administrators and must be set up in advance of testing. Users are only be able to send and receive messages from allowed recipients, typically denoted by a PayerId.

To generate a PAInitiationRequest, use the Compose menu drop-down to select Compose ePA Message.

Composing a PAInitiationRequest Message

The Edge Test Tool relies solely on XML for the purposes of ePA messaging. Users are free to compose their own XML PAInitiationRequest message, utilizing the NCPDP SCRIPT standard v2013101, or may use a PAInitiationRequest template provided by Surescripts. All edits must be made either in the template prior to loading or directly in the XML message text box. If testing in the PBM/payer system requires specific patients, medications or pharmacy information, users should ensure that all of the appropriate fields are correct in the XML before submitting to the PBM/payer system.

Loading a PAInitiationRequest Template

To load an XML PAInitiationRequest template, users must have the template stored on their local machine or a network location that is accessible to the user.

  1. From the ePA Message Composer screen, click the Choose File button located at the bottom of the page.
  2. Select the appropriate template and click the Load File button.

    • The XML message content is displayed in the XML message text box in the center of the page. Users can modify the contents of the message from the text box directly as needed.

    • By default, the Generate Id option is enabled. This is to ensure a unique MessageID and PAReferenceID as the message goes across the Surescripts network. This option should remain enabled unless the user is attempting to test the impact of different message ids as they enter or leave the PBM/payer’s system.

  3. Once the message is verified for accuracy, click the Post To Mailbox button to send the PAInitiationRequest to the PBM/payer system.

  4. Once successfully sent, users can see response content on the lower portion of the screen. For help with specific errors, please contact Surescripts Customer Support.
