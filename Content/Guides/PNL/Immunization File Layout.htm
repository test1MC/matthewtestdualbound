Detail Info

Validation

All detail records are validated according to the following rules. Records that do not meet the base validations are not processed further, and are returned in the Immunization Delivery Results File with an applicable error code.

Initially record type and record number are validated. See Reject Code Summary on page 1 for list of all codes.

  • Elements noted as M (Mandatory) in the Validation column must be present
  • Elements noted as C (Conditional) must meet any conditionality rules stated in the Comments column. If comments column is blank it is optional.
  • Elements must conform to the element length indicated in the Data Type column.

Validation does not test data types, valid value rules or formats. All elements are treated as plain text during validation.

Elements containing an “M” or “C” value in the “M/C/NU” column are available to be included in notifications sent to primary care providers. These elements must:

  • Conform to the optionality indicated in the M/C/NU column (provider notifications will not be created for records missing elements noted as “M” in the PNL column)
  • Contain valid dates where applicable, in the format identified in the Data Type column

Records that do not meet these validations will not result in creation of physician notifications, and are returned in the Immunization Delivery Results File with an applicable error code.
