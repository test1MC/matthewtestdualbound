Secure FTP

Surescripts has implemented a Secure FTP server that supports the following file transfer protocols: FTP with SSL; FTP over SSH; and HTTPS. All of these protocols use an X.509 digital certificate to facilitate data encryption during the file transfer process. Clear-text file transfer is not allowed to or from Surescripts.

File Transfer

Files will be exchanged via SFTP protocol using Public Key Authentication. Each data source will be given a unique directory on Surescripts SFTP server

Surescripts SFTP Server Information:

  • IP/FQDN: 69.25.46.18 / sftp.surescripts.net
  • Protocol: SFTP
  • Port: 22
  • Authentication: RSA/DSA 1024 bit SSH-2 Public Key Authentication
  • Encryption: AES 128  or 3DES
  • MAC Algorithm: SHA-1

Surescripts will need the following information to complete setup on our SFTP server.

  • Static Public Source IP Address. Surescripts’ SFTP is Firewall IP restricted.
    • The IP provided cannot be private (i.e. RFC 1918)
    • The IP provided should to be registered with American Registry for Internet Numbers (ARIN) using the participant’s name and contact information (i.e. no Comcast or no Verizon).
    • If the provided IP is not registered to the participant, Surescripts requires that an email is sent to Surescripts affirming that the IP provided is operated by the participant. The email must come from the participant domain (i.e. no @yahoo.com or @gmail.com)
  • RSA/DSA 1024 bit Public key in SSH-2 format for Server Authentication

The participant will use their Surescripts-assigned Participant ID and Participant Password to log into the Secure FTP server.
