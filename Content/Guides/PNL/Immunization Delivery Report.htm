Immunization Delivery Report

An Immunization Delivery Report will be provided to the participant after the immunization file has been processed and the physician notification letters are sent. This file contains the records from the immunization file, with additional columns indicating:

  • the delivery method initiated for each record
  • the error code if an issue was encountered when processing a record.

The Immunization Delivery Report will be written to the participant’s FTP account in a folder named “from_surescripts”.

The Immunization Delivery Report file name will be based on the original immunization file name with a “.” followed by a unique processing id, and then a “rsp”. This single file will then be added to a compressed archive file. The name of the archive will match the response file name with “.gz” added to the end.

Example:

Original Immunization File Name: IMMAcmePharmacy2012112001.txt

Response File Name: IMMAcmePharmacy2012112001.1235413425555.rsp

The Compressed Archive File will contain the response file and be named: IMMAcmePharmacy2012112001.1235413425555.rsp.gz

For more detailed information on the error codes sent within the Response File, refer to Reject Code Summary on page 1.

Delimiters      

Each field is delimited by the pipe character (|).Each line is separated by a LF character.
