Configuration Options

For the initial integration process, the integration manager will set up a portal to conform to your interface requirements. If your XDR interface is updated after the initial integration process, contact your integration manager to determine if your portal will also require updates.

Surescripts recommends all XDR customers conform to the following typical XDR configuration:

  • Accept all document attachments
  • Support Delivery Notification Tracking
  • Manage messages addressed to multiple recipients

For non-typical XDR implementations, Surescripts provides the following configuration options:

  • Define attachments accepted (e.g., XML, PDF, txt, xsl, etc.)

When attachments are omitted, Surescripts creates and sends an Auto-Reply message to the original message sender identifying the message as successful and listing each omitted attachment. The receiving customer may opt out of Surescripts' generated Auto-Reply messages.

  • Omit "Healthcare Communication" text attachments
  • Require at least one XML attachment
  • Single Message/Recipient vs. Multiple Recipient Message
  • Opt-out of XDR Delivery Notification Tracking
