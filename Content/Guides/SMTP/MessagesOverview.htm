Messages Overview

Clinical Direct Message SMTP

An SMTPXDR message (clinical document interchange) is used to transfer patient-related clinical documents between health care providers.

The maximum allowable size for an SMTPXDR message, including attachments, is restricted to 20 MB.

Delivery Status Notification (DSN)

DSN: Success – indicates a Clinical Direct Message was delivered electronically to the recipient and is generated by Surescripts when the receiving system returns either a:

  • Success Response – generated when the receiving system receives the XDR message (will be used as the validation of delivery if the receiving system is not configured for Delivery Notification Tracking), or a
  • DSN Success – generated when the receiving system is configured to send Delivery Notification Tracking and is able to successfully deliver the message to the intended recipient.

DSN: Failure – indicates that an error has occurred, the message has not been delivered to its intended recipient, and is generated by Surescripts when the receiving system returns either a:

  • Failure Response – generated when there is a communication problem delivering the XDR message to the receiving system, or a
  • DSN: Failure – generated when the receiving system is unable to deliver the Clinical Direct Message to the intended recipient.

Provide and Register Document Set-b Response (Response)

A response generated between the XDR customer’s system and the Surescripts system. The system receiving the XDR message or DSN must generate the response to the system sending the message or DSN.

Success Response – indicates the message is routable and addressed to a valid Direct Domain.

Failure Response – indicates either an invalid Direct Domain or an invalid message format.

Responses only indicate that the message has made it to the next stage in the message processing and does not indicate the XDR message has been delivered to the receiving system.
