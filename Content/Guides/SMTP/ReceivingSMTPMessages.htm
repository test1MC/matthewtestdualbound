Receiving SMTPXDR Messages

Receiving an XDR message is when the customer XDR interface receives a message from Surescripts. This is when the Surescripts network has a message addressed to a domain that is owned by the XDR interface.

In terms of [ITI TF-1] section 15.1, Surescripts is the Document Source and the customer XDR interface is the Document Recipient.

Surescripts will conform to the Minimal Metadata specification, as defined by [XDR for Direct] section 6.1 of the reference document XDR and XDM for Direct Messaging Specification, Version 1, 9 March 2011.

The incoming message will contain a direct:addressBlock SOAP header with one direct:from element and one or more direct:to elements. The XDS metadata element authorPerson will contain the sender information and one or more intendedRecipient elements will contain the recipient address(es). If a message has multiple recipients, one or more of which are at the customer system and one or more of which are at a different customer, all addresses will be present on the message to each customer to ensure continuity of the care team.
