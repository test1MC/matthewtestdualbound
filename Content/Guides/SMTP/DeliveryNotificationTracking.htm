Delivery Notification Tracking

Surescripts recommends XDR customers support Delivery Notification Tracking. The format of the delivery status notification will conform to section – “1.5.2.1.1 Delivery Status Notifications Using XDR” within the “Implementation Guide for Direct Edge Protocols, Version 1.1, 25 June 2014”.

Send/Receive Delivery Notifications

Send DSN – Receiving System

When configured to send delivery notifications, the receiving system must send a follow-up DSN, per recipient, indicating the successful or failed delivery of the message to the intended recipient. If the receiving system does not send a follow-up DSN, the message remains “In Process.”

A DSN must be returned for each recipient managed by the receiving system.

Receive DSN – Sending System

When configured to receive delivery notifications, the sending system should always expect to receive a follow-up DSN, per recipient, indicating success or failure of the message regardless of whether the header includes the request for the follow-up message.

A DSN will be returned for each recipient on the XDR message.

No Delivery Notification Tracking

For sending systems that are not configured to send delivery notifications, the final status of the message can be obtained either through viewing the message in Surescripts Admin Console (if additional notifications are provided) or by contacting the recipient directly.

Send – Receiving System

When the receiving system is not configured to send delivery notifications, the

  • Success response will be translated into a DSN: Success
  • Failure response will be translated into a DSN: Failure

The XDR message will be identified with its final status and no further responses will be expected.

Receive – Sending System

When the sending system is not configured to receive delivery notifications, Surescripts verifies if the direct address can be routed and responds with a success if the message is routable. The success response does not indicate if the message has been successfully delivered to the receiving system. The sending system will receive no further messages confirming delivery of the message.
