Changes from SCRIPT 8.1

Refer to for the NCPDP’s SCRIPT Standard Format Implementation Guide (Version 10, Release 6). Changes are found in Appendix B. The following list of changes includes all changes from NCPDP SCRIPT 8.1 to 10.6. General Changes

  1. The values of “D”, “W”, “M”, and “Y” and the comments pertaining to the time period have been removed from DRU-060-01 Refills Quantity Qualifier. Added new guidance on how to use qualifiers based on message type.
  2. Date/Time Period Qualifier – DRU-040-01 additional values added.
  3. Date/Time – DRU-040 increased repetitions from 5 to 9.
  4. Added Date/Time qualifier for more specific date down to hours and minutes. All dates with qualifiers – Drug Dates, Medication History Start & End Dates (Not Patient DOB).
  5. Added prior auth status to Drug Segment – DRU-120.
  6. PVD loops were increased from 3 to 4.
  7. Added a qualifier for remaining refills on a Medication History Response – DRU-060-01.
  8. Added Source Segment to the Medication History Response.
  9. Added qualifier for processor control number and increased loops on Payer ID from 1 to greater than 1 – COO-010.
  10. Added Structured SIG Segment.
  11. New field and code list for Provider Specialty Code – PVD-040-03.
  12. For communication numbers at least one occurrence must be “TE” for telephone number.
  13. FMT Code list added as new field for:
    1. Drug Form – DRU-010-14
    2. Drug Strength – DRU-010-16
    3. Unit of Measure / Potency Unit Code – DRU-020-05 and OBS-030-08
    4. DEA Schedule DRU-010-17
  14. DEA Schedule added to the Drug Segment DRU-010-17. Use this field instead of the Controlled Substance flag in DRU-110 <DrugCoverageStatusCode>.
  15. Drug Use Evaluation – DRU-100 added fields 06 - Clinical Significance Code and 07 – Acknowledgment Reason.
  16. Added provider specialty for pharmacy segment.
