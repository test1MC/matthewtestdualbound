M

UIB-030-01

Transaction Control Reference

<MessageID>

an..35

Each message sent from a participant shall*106M-009 have a unique MessageID. MessageIDs should not be reused for 18 months.

A minimum set of standards/ algorithms should be used when generating MessageIDs to ensure uniqueness. If possible, participants should utilize Global Unique Identifiers (GUIDs).

D

UIB-030-02

Initiator Reference Identifier

<RelatesToMessageID>

an..35

Requests for Claim History: Mandatory - The claim medication history request for a patient shall*106M-013 be tied to the most recent eligibility response for that patient. This is accomplished by using the ISA control number (ISA-13) from the 271 Eligibility response in the UIB-030-02 <RelatesToMessageID> of the RXHREQ Medication History Request.

Responses to Fill and Claim Requests History: Mandatory - This field is mandatory and is used to link the original RXHREQ Medication History Request message (value in UIB-030-01) to the RXHRES Medication History response. The MessageID shall*106M-010 be echoed back in the response to the message that was received.

Requests for Fill History: Not used

UIB-060-01

Sender ID – Level One

<From>

an..35

This field contains the identification number of the sender. This field is used in conjunction with UIB-060-02 Level One ID Code Qualifier, which qualifies which ID was used.

Will contain the Participant ID assigned by Surescripts.

For request transactions, this will contain prescriber vendor ID assigned by Surescripts.

For response transactions, this will contain the PBM/Payer ID assigned by Surescripts.

For Medication History Panel Management (Surescripts to the PBM), this field will contain the Surescripts ID “S00000000000002” or “S00000000000003”.

Request transactions from:

Ambulatory prescriber vendors will contain either a Payer/PBM ID when requesting claims data or the literal value of “S00000000000001” when requesting pharmacy fill data.

Response transactions from PBMs will contain the Prescriber Vendor ID assigned by Surescripts.
