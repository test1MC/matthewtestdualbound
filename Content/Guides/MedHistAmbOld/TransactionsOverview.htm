Messages Transactions Overview

Message Descriptions

RxHistoryRequest

The prescriber sends a Medication History request for a patient to Surescripts. The request is routed to the appropriate PBM/payer for processing. For fill data (which is optional), the request is handled by Surescripts internally using the pharmacy database.

Customers need to send two messages to get both fill and claim based medication history. One indicates it is for fill data from the pharmacy and the other contains the PBM/payer identifier indicating it is requesting history from the PBM/payer. In addition, if the patient has multiple coverages each coverage type would need a separate RXHREQ to the PBM/payer(s).

Error

This NCPDP SCRIPT message transmits that an error has occurred, indicating the request was terminated. An error can be generated when there is a communication problem or when the message actually had an error (e.g. a formatting problem).

Status

This NCPDP SCRIPT message is used to communicate an acceptance message. Status messages will be sent to indicate receipt of a valid message.

Format Options

Two formats are supported for Medication History: EDIFACT and XML. For XML documentation, refer to the schema, SS_SCRIPT_XML_10_6MU.xsd, which will be distributed upon request. Where appropriate, XML tags are listed in the Element Name column of the EDIFACT Segment Tables. Refer to XML on page 1 for additional documentation. It is a decision of the network participant whether their application will use the XML or EDIFACT constructs of NCPDP SCRIPT.

EDIFACT

Also known as UN/EDIFACT, EDIFACT is an acronym for “EDI for Administration, Commerce, and Transport”. EDIFACT is the international message standard for the exchange of electronic data, developed and managed through the cooperation of the United Nations and the Economic Commission for Europe (UN/ECE). The NCPDP SCRIPT standard was originally based on the EDIFACT standard.

XML

Extensible Markup Language (XML) is a simple, very flexible text format derived from SGML (ISO 8879). Originally designed to meet the challenges of large-scale electronic publishing, XML now plays an increasingly important role in the exchange of a wide variety of data on the Web and elsewhere. NCPDP has approved the XML representation of the SCRIPT standard and this guide includes both EDIFACT and the related XML tags. More information on XML can be obtained at: http://www.w3.org/XML/.

The Surescripts XML schema is available to all network participants and can be obtained as a supplemental file to this document by sending a request to your Implementation Project Manager or your Account Manager/Executive.
