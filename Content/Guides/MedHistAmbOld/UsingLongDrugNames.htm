Using Long Drug Names

DRU 010-02 Item Description is mandatory and must be populated with the drug name. Commonly, this field is populated with the drug name from the prescriber / pharmacy system files, which contains the drug name, strength, and dosage form. In cases where the 35 characters are not sufficient to properly display the full drug name, strength, and dosage form, three Item Description fields may be used – DRU -010-10 Item Description, -11- Item Description, and -12- Item Description.

DRU-010-02 Item Description must always be populated. If the 35 characters are sufficient for the drug name, strength, and form, elements 10-12 are not to be used.

If the 35 characters are not sufficient for the drug name, strength, and dosage form, elements 10-12 are to be used sequentially. The three fields should result in the concatenation of the full drug name, strength, and dosage form, using as many of the bytes necessary of the 35 times 3 fields. The information should begin in elements 010-10 Item Description and continue to 010-11 Item Description for as many bytes are needed and if necessary continue to 010-12 Item Description for as many bytes are needed up to the 35 length.

The following examples show the use of the 02, 10, 11, and 12 Item Description fields.

Example 1:  CALAN SR 240mg

010-02

CALAN SR 240mg

010 -10

Not transmitted

010 -11

Not transmitted

010 -12

Not transmitted

Example 2:  Hydrocholorothizazide; lisinopril 12.5 mg;10 mg tablets

010 -02

Hydrocholorothizazide; lisinopril

010 -10

Hydrocholorothizazide; lisinopril

010 -11

12.5 mg;10 mg tablets

010 -12

Not transmitted

The space before “12.5mg” is part of the text and occupies a byte.
