XML

Refer to SS_SCRIPT_XML_10_6MU.html or SS_SCRIPT_XML_10_6MU.xsd for complete documentation of the XML version of SCRIPT. The HTML version of the documentation provides an indexed version where you can drill down into the detail of each message and each type.

XML Nuances

  1. Long XML Dates are in the format CCYY-MM-DDTHH:MM:SS.F – example: 2006-12-07T12:20:46Z (UIB-080) where F represents the conditional fractional sections. F could be one or more digits.
  2. Short XML Dates are in the format CCYY-MM-DD – example: 2006-12-07 (PTT-020).
  3. When mapping to XML, if the drug name is longer than 35 characters then DRU-010-10-12 are used for the description instead of DRU-010-2.
  4. Zip codes can only contain numbers – no dashes and must be either 5 or 9 digits long. When mapping to XML the non-numeric portion will not be mapped. (PVD-080-04, PTT-060-04).
  5. The Status message supports codes of 000,001,002,003,005,010 in the STS-010; however, since this implementation does not support mailboxing or password change; only code 010 should be used.
  6. Do not use field definitions such as lengths and data types from this guide; instead, refer to the schema.
  7. Fields that are concatenated into one field when mapping from EDIFACT to XML:
    1. UIB-080-01 & UIB-080-02  <SentTime>
    2. DRU-010-02, DRU-010-10, DUR-010-11 and DRU-010-12 <DrugDescription>
    3. DRU-030-02 and 030-03 <Directions>
    4. DRU-090 Repeats 3 times <Note>
    5. OBS-020 Repeats 2 times <ObservationNotes>
  8. Most symbols should be ‘safe’ within most alphanumeric fields; however, XML validation has issue with the following characters and they shall*106M-035 be escaped. The most common escape encodings are:

    9. ASCII Characters

    XML Escape

    quote (")

    &quot;

    apostrophe (')

    &apos;

    ampersand (&)

    &amp;

    less than (<)

    &lt;

    greater than (>)

    &gt;

    String lengths are calculated based on the un-escaped value. For example, “>98 degrees” should be encoded as “&gt;98 degrees” but would only be counted as a string length of 11, not 14.

  9. Fields that have a different implied length in XML. These fields have a specific size defined in EDIFACT but in XML have a list of values with an implied length.
    1. PTT-040 Patient Gender
    2. DRU-010-03 Product Code Qualifier
    3. DRU-010-08 DB Drug Qualifier
    4. DRU-050 Substitution
    5. DRU-070-01 Diagnosis / Clinical Information Qualifier
    6. DRU-070-03 & 05 Diagnosis Qualifier
  10. Diagnosis Qualifier DRU-070-03 & 05– EDIFACT supports values of E, F, M, ABF, and DX. XML also supports ICD9 which is equivalent to DX and ICD10 which is equivalent to ABF.
