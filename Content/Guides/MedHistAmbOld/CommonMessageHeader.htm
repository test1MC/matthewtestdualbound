Common Message Header

The message header follows the NCPDP SCRIPT standard version 10.6MU. This section only applies to customers using XML format. The common message header is not used for customers using the EDIFACT format. Surescripts uses a common message header for all messages.

For this implementation the digital signature is not used.

Message Header XML Diagram

In the following diagram, blocks that are solid are mandatory, whereas blocks that have dotted lines are conditional.

For this implementation the digital signature is not used.

<MessageID>

an..35M

 

 Each message sent from a participant shall*106M-009 have a unique Message ID. Message IDs should not be reused for 18 months. A minimum set of standards/algorithms should be used when generating message IDs to ensure uniqueness. If possible, customers should utilize Global Unique Identifiers (GUIDs).

D

<RelatesToMessageID>

an..35

 

Requests for Claim History: Mandatory - The claim medication history request for a patient shall*106M-013 be tied to the most recent eligibility response for that patient. This is accomplished by using the ISA control number (ISA-13) from the 271 Eligibility response populated in the UIB- 030-02 <RelatesToMessageID> of the RXHREQ Medication History Request.

Responses to Fill and Claim Requests History: Mandatory - This field is mandatory and is used to link the original RXHREQ Medication History Request message (value in UIB-030-01) to the RXHRES Medication History response. The MessageID shall*106M-010 be echoed back in the response to the message that was received.

Requests for Fill History: Not used
