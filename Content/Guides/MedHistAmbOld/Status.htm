Status

The Status message is an NCPDP transaction that indicates the acceptance of the request. In the Medication History flow a Status message is used in the case of an error in the response from the PBM/payer. Because the PBM/payer is responding with an invalid message a new Error message is initiated to the PBM/payer. The PBM/payer will then need to respond to the Error message with a Status message.

This message is used to communicate the data content status of a transaction. Segments used for Status include:
