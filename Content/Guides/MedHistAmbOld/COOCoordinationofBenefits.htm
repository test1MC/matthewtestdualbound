COO-090

DATE

<EffectiveDate>

<ExpirationDate>

 

Repeats twice.

<EffectiveDate> is the beginning date for the desired history (e.g., 1 year ago).

<ExpirationDate> is the end date for the desired history. This should be set to the date the RXHREQ was initiated for the initial request, if populated. This would be set to the last drug's <LastFillDate> in the previous RXHRES if the "AQ" indicator was in the <ApprovalReasonCode>.

Up to one year of medication history fill data can be requested.

It is suggested that no more than one year of medication history claim data be requested.

If the Date composite (COO-090) is used, both the effective date and expiration date are required. See XML on page 1.
