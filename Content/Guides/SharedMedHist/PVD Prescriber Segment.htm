PVD – Prescriber Segment

For the drug listed above, this is the prescribing physician. This is used in two places: for the requesting physician and in the drug loop it is used for the prescribing physician.

PVD-020

REFERENCE NUMBER

Repeats up to 3 times

M = Rx History Request

C = Rx History Response

For the Rx History Request at least one occurrence must contain the individual (not organizational) NPI of the prescriber.

Receivers of the 10.6 messages cannot expect the NPI to always be present due to translation from lower SCRIPT versions on the network.

When present, the NPI will be validated against the check digit routine for the requesting physician. For specific information see http://www.cms.gov/NationalProvIdentStand/Downloads/NPIcheckdigit.pdf

M

PVD-090-01

Communication Number

<Number>

an..80

Phone Number shall*106M-025 be in the format 5552223333X4444 - where 555 is the area code, 2223333 is the main number and X4444 is the extension. Phone number shall*106M-028 not be populated with all zeros or other default values. This field is 80 characters long to handle email addresses. (Please note the extension X4444 is just an example. Less/more than four digits are allowed).

M

PVD-090-02

Code List Qualifier

<Qualifier>

an..3

If a communication number is sent, one occurrence must contain “TE” for telephone.

For values, see the NCPDP External Code List (X12 DE 365).

Receivers of 10.6 messages cannot expect this information to always be present due to translation from lower SCRIPT versions on the network.
