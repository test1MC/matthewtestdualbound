RES-010

RESPONSE TYPE

<Approved> <Denied>

an..3

A “Denied” (D) response must be accompanied by Free Text (RES-040) since there is not a valid code list qualifier.

Values:

A = Approved

D = Denied. Use the denied code for a scenario where data cannot be released for a patient due to contractual or business constraints. This code should not be used to represent processing or syntax errors, instead an Error message should be sent.

C

RES-020

Code List Qualifier

<ApprovalReasonCode>

an..3

AQ = More History Available. There may be less than 50 drugs in this response due to payer processing (e.g. drug filtering which is used to remove sensitive drugs.).

Only the value of AQ is supported.

RES-040

Free Text<DenialReason> for Denied and <Note> for approved

an..70

XML tag <Note> is used for Approved responses and XML tag <DenialReason> is used for Denied responses.

A “Denied” (D) response must be accompanied by Free Text (RES-040) since there is not a valid code list qualifier.

Receivers of 10.6 messages cannot expect this information to always be present due to translation from lower SCRIPT versions.
