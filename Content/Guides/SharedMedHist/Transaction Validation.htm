Transaction Validation

Surescripts will certify that participants are in compliance with the transaction specifications outlined in this Guide during implementation and will continue to validate once in production.

To validate a transaction, Surescripts:

  • Validates the sender identification and password.
  • Validates the recipient identification.
  • Verifies that the sender and recipient are in agreement contractually to exchange information.
  • Validates the syntax of the transaction including field lengths, data types, number of repeats and code values.
