ORU Message Format

The ORU message is defined as an unsolicited observation message (Event R01). Surescripts utilizes it to return a Medication History – Acute report that is ready for viewing or printing. The data is contained in the OBX segments in a free text format.

If any errors occurred while processing the Medication History – Acute request, the ORU message is used to communicate the details of the errors that occurred.

Surescripts does not utilize every segment available within the ORU message. The table below explains the segments and elements used.

PID – Patient Identification Segment

Surescripts will return the PID Segment with the same values and in the same format as it was delivered on the ADT request. Surescripts will not modify or add values to this segment.

PV1 – Patient Visit Segment

Surescripts will return the PV1 Segment with the same values and in the same format as it was delivered on the ADT request. Surescripts will not modify or add values to this segment.

OBR – Observation Request Segment

Surescripts utilizes the OBR Segment to indicate the status of the Medication History – Acute request. The OBR Segment will indicate if the Medication History – Acute request produced history results or if an error occurred. The fields utilized by Surescripts are indicated in bold italics.
