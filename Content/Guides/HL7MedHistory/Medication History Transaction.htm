Medication History Transaction Flow

The following flow takes place between Surescripts and a source participant. It is triggered by the existence of the ADT event from the hospital.

Request/Response

  1. From the ADT message transaction flow, Surescripts has identified the patient and associated PBM.
  2. Surescripts sends a Medication History request for a patient, supplying the PBM Unique Member ID for the patient.
  3. Surescripts also checks its retail fill history for matching records.
  4. The PBM processes the request. If it fails, the PBM returns an Error message instead of a Medication History response. The processing steps are listed below:
    1. PBM validates the format of the request.
    2. PBM finds the patient via the PBM Unique member ID.
    3. PBM determines that the patient is still eligible (optional).
    4. PBM checks for the dates of the requested medication history. The PBM searches the claims repository for medications that have a fill date between the begin and end date provided.
    5. If more drugs are available within the date provided and the PBM is unable to transmit additional drugs, a ‘More Drugs Available’ response is returned in the RES segment. Surescripts will revise the date range and make another request.
  5. The PBM creates the Medication History response file and submits it back to Surescripts.
  6. Surescripts processes the result set and any matching fill records and formats it into the desired response. 
    Note:  This process is explained in the Medication History – Acute Response sections.
