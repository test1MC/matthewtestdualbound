Transactions Overview

Medication History – Acute Transaction Flow

The Surescripts Medication History – Acute service provides clinicians with convenient access to up-to-date medication history for patients in an inpatient setting. The hospital system provides basic patient demographic information, and the Surescripts Medication History – Acute service responds with information about prescriptions that the patient’s pharmacy benefit manager (PBM) has paid for over a selected period of time. It also can contain retail fill data from pharmacies connected to Surescripts for Medication History.

The specifications required for requesting and receiving a medication history through the Surescripts Medication History – Acute service were created with the following assumptions:

  • A Medication History request requires patient authorization in order to be retrieved. If Surescripts receives a transaction requesting a medication history, Surescripts will assume consent has been granted.

  • A requester can receive multiple response formats from one request. The responses returned are determined by what formats the participant is certified to receive.

The processing flow for the Medication History – Acute service is below:

  1. The Requester sends an ADT event to Surescripts when a Medication History – Acute response is desired. 

    The hospital system determines when it is appropriate for Surescripts to get a request. The ADT message that Surescripts will expect and process is any of the valid event codes in the HL7 version 2.3 standard, including the admit/visit notification (A01), A04 register a patient (Emergency), or A08 update patient information.

  2. Surescripts validates that the message is structurally sound, that the sender is authenticated and authorized to make a request, and that the request is sent to the proper environment. If the process succeeds, the sender will get an ACK with an AA (Application Accepted) acknowledgement code. If it fails, the sender will get an AE (Application Error) acknowledgement code or an AR (Application Reject) error, depending on the error. The ACK is a synchronous response to the ADT event received by Surescripts. Surescripts will audit and investigate any errors that are transmitted within the message.

  3. Surescripts utilizes the patient information sent within the PID (Patient Identification) segment to locate the desired patient coverage in the Surescripts MPI (Master Patient Index).

  4. Once the patient is identified and the coverages are determined, Surescripts queries each source for medication history utilizing the Medication History request and response transaction.

  5. Surescripts compiles the results and creates an HL7 ORU message and/or an HL7 RDS message for the original requester.

  6. Surescripts sends the appropriate messages to the original requester.
