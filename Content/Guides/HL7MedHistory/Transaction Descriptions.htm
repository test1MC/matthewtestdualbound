Transaction Descriptions

ID Load/Update (Flat File)/Response

This transaction is used to load a PBM’s patient directory into a directory at Surescripts. This directory is an index Surescripts uses when looking up a patient’s prescription benefit. The Patient Directory indicates which PBM(s) can provide current coverage information. The elements provided are limited to the demographic data needed for patient searches.

ADT Events

The ADT events are utilized within the hospital system as patient events.  A hospital system sends one of the events as a ‘trigger’ for the Medication History – Acute request to Surescripts. The ADT events are HL7 standard transactions.

Medication History Request/Response

The Medication History Request (RXHREQ) and Response (RXHRES) transactions are used to communicate between Surescripts and a source system to retrieve a listing of dispensed medications that were paid for by a patient’s PBM or health plan. t also can contain retail fill data from pharmacies connected to Surescripts for Medication History. The transaction format uses the NCPDP SCRIPT standard 8.1.

ORU R01 Event

The ORU R01 event is defined as an unsolicited observation message. Surescripts utilizes this message format to transmit a medication report in a printable or displayable form. The ORU R01 event is a HL7 standard transaction.

RDS O01 Event

The RDS O01 event is defined as a pharmacy treatment dispense message. Surescripts utilizes this message to transmit medication history information in discrete data elements allowing a receiving system to process the data within their system. The RDS O01 event is an HL7 standard transaction.

MSA ACK

An MSA ACK is a general acknowledgement sent in response to receiving a message. It indicates that the message has been received, its contents are understood, and it will be processed as needed. The ACK event is an HL7 standard transaction.
