Failure Mode/Response Approach

Surescripts’ error processing approaches are defined below. For more information on Medication History – Acute error processing, refer to Section 12.

Medication History Error Processing

Surescripts has identified different levels of failure for the Medication History request/response transaction. 

  • In instances where Surescripts or a participant receives a transaction that is unrecognizable, the recipient will send back an XML formatted NAK.

  • If an error occurs after the sender has been identified and validated, and the message has been determined to be an NCPDP message, an NCPDP ERROR message is returned.

  • If the recipient identifies an error, an ERROR message is created and Surescripts will pass the error text on to the original requester.

HL7 Error Processing

Surescripts has identified different levels of failure for HL7 transactions.

  • In instances where Surescripts or a participant receives a transaction that is unrecognizable, the recipient will send back an XML formatted NAK.

  • If an error has occurred after Surescripts has determined that this is an HL7 transaction, the recipient will respond with an HL7 ACK and the appropriate error.

  • If an error occurs after the message has been accepted, the responder will respond with an error message within the appropriate response format.

 

Flat File Processing

Some transactions are very large file loads that are defined in a fixed width flat file. Each row is separated by a new line (hex 0A) character. Surescripts will accept these transactions, validate the format, and process the file. Surescripts then produces a fixed width response file that details out any file errors, row errors, and processing acknowledgements.
