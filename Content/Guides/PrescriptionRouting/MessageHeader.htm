Message Header

Surescripts uses a message header for all messages.

Message Header XML Diagram

In XML diagrams, blocks that are solid are mandatory, whereas blocks that have dotted lines are conditional.



Message Header Elements

Requirement Designation

Element Attributes

Code

Description

M

Required/Mandatory - the element must be used.

C

Situational/Conditional - the element must be used if conditions are met. Some fields do not have specific conditions. Data should be sent if available.

D

Dependent - the element is required or conditional based on the message type.

M/C – Mandatory or Conditional

This Guide only includes data elements where Surescripts has specific requirements or further explains the field usage. Refer to the NCPDP SCRIPT Schema listed in Document References for a complete list of fields.

SecondaryIdentification

For Sender - Password. This field is used for the sender's password to the recipient's system. It is recommended that passwords be used for security checking and to validate identification. For receiver - may be used as a secondary identification of the recipient, or may be used to identify the switch.

 

TertiaryIdentification

 May be used as a tertiary identification of the recipient.
