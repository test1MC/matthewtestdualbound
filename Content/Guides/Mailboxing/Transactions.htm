Messaging Overview

This section lists the GetMessage and acknowledgement messages used in the Mailboxing service message flow.

Message Descriptions

This section describes the Mailboxing Service GetMessage and Acknowledgement messages.

Mailboxing Service GetMessage

The Mailboxing service uses the GetMessage message in the service message flow. The GetMessage is used to retrieve messages from the Mailboxing service. The response to each GetMessage request is a routing message that is retrieved from the mailbox, or a Status message, indicating that the mailbox is empty.

Refer to the Surescripts E-Prescription Companion Guide for additional prescription routing messages.

Acknowledgement Messages

The following acknowledgement messages are supported.

Verify

Error

This NCPDP SCRIPT message transmits that an error has occurred, indicating the request was terminated. An error can be generated when there is a communication problem or when the message actually had an error (e.g. a formatting problem).

Status

This NCPDP SCRIPT message is used to communicate an acceptance message. Status messages will be sent to indicate receipt of a valid message.

Push Message Flow Without Mailboxing

The following diagram shows the push message flow without Mailboxing for a prescriber system sending a NewRx to Surescripts and Surescripts “pushing” the e-prescribing messages directly to the pharmacy recipient over a persistent Internet connection. The Mailboxing service is not required because the persistent Internet connection enables a continuous message flow from the prescriber system to Surescripts and on to the pharmacy. The push message flow is further described in the Surescripts E-Prescribing Companion Guide.

 

 

Push Message Flow without Mailboxing Steps

  1. The prescriber system sends an HTTPS Request: NewRx to Surescripts.
  2. Surescripts sends the HTTPS Request: NewRx to the pharmacy.
  3. The pharmacy responds to Surescripts with an HTTPS Request: Status or Error message.
  4. Surescripts forwards the HTTPS Request: Status or Error message to the prescriber system.
