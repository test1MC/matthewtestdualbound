Mailboxing Message Flow

Prescribers and pharmacies can use the Mailboxing service. Prescribers send messages through the Surescripts network to a pharmacy mailbox for future retrieval. Pharmacies retrieve stored messages immediately or up to 24 hours later from their Surescripts pharmacy mailbox. Prescribers and pharmacies have their own message flow.

The prescriber system sends a NewRx message to Surescripts and the message is saved in the pharmacy mailbox. A response is sent back to the prescriber from the Surescripts pharmacy mailbox, indicating that the message was successfully saved in the pharmacy mailbox for pharmacy future retrieval.

The pharmacy, immediately, or up to 24 hours later, sends a GetMessage message, along with a NewRx request, to their Surescripts mailbox. The NewRx saved messages sent from the prescriber are retrieved from the Surescripts pharmacy mailbox and returned to the pharmacy for filling. Retrieved mailbox messages continue to be returned to the pharmacy until the mailbox is empty.

Mailboxing Message Flow Diagram

The Mailboxing message flow diagram shows the message communication between the prescriber system, pharmacy 1 mailbox, and pharmacy 1.

 

 

Prescriber System to Pharmacy 1 Mailbox Message Flow Steps

  1. The prescriber system sends an HTTPS request: NewRx message to Surescripts.
  2. The received NewRx message is saved in the pharmacy 1 mailbox.
  3. A response is sent from the pharmacy 1 mailbox indicating the NewRx is saved.
  4. An HTTPS response: Status 000 is sent from Surescripts to the prescriber system verifying the NewRX message.
  5. The pharmacy 1 sends an HTTPS request: GetMessage to Surescripts.
  6. The get NewRx message is passed to the pharmacy 1 mailbox.
  7. The pharmacy 1 mailbox responds with a send NewRx message.
  8. The HTTP response: send NewRx message is sent to pharmacy 1.

Prescriber System Sending NewRx Message to Pharmacy 1 Mailbox Flow

This section shows how the prescriber system sends a NewRx message to a pharmacy 1 mailbox with a Verify or Error response.

 

 

Prescriber System to Pharmacy 1 Mailbox Flow Steps

  1. A prescriber system sends an HTTPS request: NewRx to Surescripts.
  2. Surescripts saves the NewRx in the pharmacy 1 mailbox.
  3. The pharmacy 1 mailbox responds to the saved message.
  4. Surescripts returns an HTTPS response: Status 000 message to the prescriber system, verifying the NewRx message was successfully received by the pharmacy 1 mailbox. Other Status messages returned could indicate a pharmacy 1 mailbox error.

Pharmacy 1 Retrieving a NewRx Message from Pharmacy 1 Mailbox Flow

This section shows how pharmacy 1 retrieves a message from a pharmacy 1 mailbox.

 

 

Pharmacy 1 Retrieves Message from Pharmacy 1 Mailbox Flow Steps

  1. The pharmacy 1 sends an HTTPS request: GetMessage to the pharmacy 1 mailbox.
  2. The pharmacy 1 mailbox returns the HTTPS response: Send NewRx to pharmacy 1.
  3. The pharmacy 1 sends an HTTPS request: GetMessage with acknowledge to the pharmacy 1 mailbox that the message was successfully received.
  4. The pharmacy 1 mailbox sends an HTTPS response: Status 002 message to pharmacy 1, indicating the mailbox is empty. The pharmacy 1 "sleeps" and does not access the pharmacy 1 mailbox for a time. Thereafter, the pharmacy 1 polls the pharmacy 1 mailbox again for new messages and the process, indicated in Steps 1-4, repeats.

The message recipient must send additional HTTPS requests to acknowledge receipt of a message from the mailbox and notify the original sender when the message is processed. The process is described in Pharmacy 1 Sends Error or Verify Message to the Prescriber System Flow.

Each message retrieved from a customer mailbox contains a DeliveredID in the message header. A customer acknowledges receipt of a message by sending the DeliveredID from the message in the AcknowledgementID header field of the next GetMessage request. When Surescripts receives that AcknowledgementID, it marks the corresponding message as confirmed.

If the next GetMessage request from the customer does not acknowledge delivery of the previously retrieved message, Surescripts sends the previously delivered message again. If more than three GetMessage requests are sent without acknowledging the receipt of the previous message, that message is marked with an Error indicator and the next message from the mailbox is returned.

Processing the Message in the Recipient's System

The recipient’s software is responsible for processing any message returned by the GetMessage request. There are many ways the system could be designed to handle the received message. It might continuously loop through getting the next message from the mailbox and putting the message in an internal queue for later processing. It could also be designed to process each message as it is retrieved from the mailbox. The message processing specifics are up to the recipient’s software designer.

Error or Verify Messages

Most messages require that an Error or Verify message is sent back to the originator, once the message has been processed. This process is different for a customer using the Mailboxing service than a customer that is not using the Mailboxing service. The request and response cycle for the GetMessage request was closed by Surescripts returning the message. The only way to let Surescripts and the message originator know if the message was processed successfully, or if it contained errors, is to send a free standing Error or Verify message to Surescripts.

A mailbox recipient could receive an Error or Verify message in their mailbox. After retrieving an Error or Verify message from the mailbox, a free standing Verify or Error message should not be sent back to Surescripts. It is sufficient to have an acknowledged receipt of the Error or Verify message on the subsequent GetMessagerequest.

Pharmacy 1 Sends Error or Verify Message to the Prescriber System Flow

This section shows how pharmacy 1 sends an Error or Verify message to the prescriber system.

 

 

Pharmacy 1 Sends Error or Verify Message to Prescriber System Flow Steps

  1. The pharmacy 1 sends an HTTPS request: Error or Verify message to Surescripts.
  2. An HTTPS forward: Error or Verify message is sent from Surescripts to the prescriber system.
  3. The prescriber system sends an HTTPS response: Status 010 to Surescripts.
  4. Surescripts sends an HTTPS response: Status 010 to pharmacy 1.

Cleaning Up Mailbox Messages

Surescripts software periodically checks customer mailboxes for messages that have been marked with an Error either because they were sent three times without being acknowledged, or because they have been in the mailbox for more than 24 hours. When such messages are found, they are removed from the mailbox and an appropriate Error message is sent to the customer originating the message.
