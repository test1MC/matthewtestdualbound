Sec. 5.3 Error Message
For PBM/Payers, an asynchronous error message (also called Free Standing Error message) is sent in response to an invalid RXHRES message. The asynchronous error message is a separate message type and shall*106M-022 be responded to with a status of 010. An Error message shall*106M-023 never be responded to with an Error message; this could create an endless loop.

Sec. 6.10 DRU – Drug Segment
M DRU-010-02

Item Description

<DrugDescription>

 an..35

Drug Name.

The self-contained full drug name, strength, and form.

The item or drug description field shall*106M-031 include the full drug name, strength and form (in that exact sequence).

If the drug name is longer than 35 characters, use long drug name. Refer to Using Long Drug Names for instructions.

See XML for XML Nuances.

C DRU-010-03

Item Number

<DrugCoded.ProductCode>

 an..35

Drug number

If either DRU-010-03 or DRU-010-04 is sent, then both are required.

When sending, the NDC must be an 11 digit NDC in the 5-4-2 format. Dashes shall*106M-032 not be used and leading zeros shall*106M-033 not be suppressed. Note: This is for the NDC.

Example: 12345123412.

Note: When this is all 9s for the NDC and this is for fill data, this means that the prescription is a compound.
C DRU-010-16

Item Strength Code

<StrengthCode>

 an..15

Drug Strength – measurement value of drug strength information should be consistent across all fields in which it appears.

If either or both the Drug Coded Strength (DRU-010-06) and the Strength Code (DRU-010-16) are sent they shall* 106M-034 match the Drug Strength in the Drug Name Description field (DRU-010-02). Drug strength units should not be sent in free text Drug Strength field (DRU-010-06).

For a list of FMT codes, go to: http://evs.nci.nih.gov/ftp1/NCPDP/About.html. Use the NCPDP StrengthUnitOfMeasure Terminology rows from the downloaded data.

This list should be updated on a monthly basis.

If either DRU-010-15 or DRU-010-16 is sent, then both are required.
C DRU-010-17

DEA Schedule

<DEASchedule>

 an..15

If the medication is categorized as a controlled substance by the Drug Enforcement Agency (DEA), this field shall*106M-027 be sent.

The DEA Schedule would be populated by the system generating the message, and would utilize the Federal DEA Schedule classification code list, based on federal classification of the medication or the state reclassification of the medication.

For a complete list of values, see the NCPDP External Code List (7996).
