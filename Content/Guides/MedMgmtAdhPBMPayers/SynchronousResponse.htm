Synchronous Response

The following describes the synchronous responses provided by the Surescripts FHIR® message service.

The examples in this section do not cover all possible responses that could be sent.

Status Codes

HTTP Status Code

Reason

Response

200 OK

The request has been accepted for processing.

A basic response message.

400 Bad Request

Invalid credentials have been provided in the x-sender-uid custom header.

A response message indicating the reason message is being rejected.

401 Unauthorized

The x-sender-uid header element is missing or its value does not conform to the specification.

A response message specifying that the transaction is Unauthorized.

422 Unprocessable Entity

The message violated applicable FHIR® profiles or server business rules.

A fatal error response message.

500 InternalServer Error

The server encountered an unexpected condition which prevented it from fulfilling the request.

A response message stating that the sender should contact Surescripts support.

Example FHIR® Responses

XML Snippet – Response

The following is an example of a basic response that is returned with a 200 status.

<Bundle

    xmlns="http://hl7.org/fhir">

    <id value="9f1f0b32-e3ac-4ae5-aa39-3aee95ea384b" />

    <meta>

        <lastUpdated value="2015-09-18T01:24:47.063+00:00" />

    </meta>

    <type value="message" />

    <entry>

        <resource>

            <MessageHeader>

                <id value="124ab4a3-d41e-498b-9f0a-618b29522c6b" />

                <meta>

                    <lastUpdated value="2015-09-18T01:24:47.063+00:00" />

                </meta>

                <extension url="http://surescripts.net/fhir/StructureDefinition/message-types">

                    <valueCode value="BasicResponse" />

                </extension>

                <identifier value="335bf8d7-8c52-4635-905d-5295c983dda1" />

                <timestamp value="2015-09-18T01:24:47.063+00:00" />

                <event>

                    <system value="http://surescripts.net/fhir/mmpoc/message-events" />

                    <code value="message-processing-complete" />

                </event>

                <response>

                    <identifier value="pbm-sender-message-id" />

                    <code value="ok" />

                </response>

                <source>

                    <name value="Surescripts" />

                    <software value="Surescripts MMPOC" />

                    <version value="1.0.0-MMPOC-CI$" />

                    <endpoint value="http://surescripts.net" />

                </source>

                <destination>

                    <name value="PBM Name" />

                    <endpoint value="http://somepbm.com" />

                </destination>

            </MessageHeader>

        </resource>

    </entry>

</Bundle>
