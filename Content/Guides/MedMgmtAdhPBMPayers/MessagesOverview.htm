Messages Overview

Message Descriptions

Medication Management for Adherence uses a new clinically oriented standard FHIR® for structuring the message exchange. The PBM/payer will therefore use this standard when sending messages to, and receiving responses from, Surescripts.

Medication Adherence

An adherence message, as indicated by claim activity, occurs when a patient is not taking the medication as frequently as prescribed.

Example:

"ABC Health Plan records indicate this patient may have discontinued taking Lipitor as instructed. Please discuss adherence with your patient."

High-Risk Medication

A high-risk medication (HRM) message occurs when a patient has been prescribed a drug that is considered high-risk due to the potential for side-effects where alternative drug therapies exist.

Examples:

“ABC Health Plan records indicate this patient has been prescribed glyburide which is considered to be a high risk medication in the elderly causing prolonged hypoglycemia. Please evaluate the risk and benefit of continuing use.”

“ABC Health Plan records indicate this patient has been prescribed glyburide which is considered to be a high risk medication in the elderly causing prolonged hypoglycemia. Please consider prescribing a safer alternative to this medication, if appropriate. Alternatives: glipizide, glimepiride.”

Missing Medication

A missing medication is a medication that is expected based on a patient’s diagnosis codes or existence of another drug.

Example:

“ABC Health Plan records indicate this patient may benefit from a statin. Please evaluate the use of a statin for a diabetic patient.”

Proportion of Days Covered (PDC)

The PDC calculation is based on the fill dates and days supply for each fill of a prescription. PDC is defined as the total numbers of days with possession of medication in a period of time.

A PDC ratio will be provided by the PBM/payer on a monthly basis for cholesterol, diabetes and hypertension regardless if the PDC is below or above 80%.

PDC ratios are categorized as:

  • Ratio value <= 60%, color red
  • Ratio value > 60% and < 80% , color orange
  • Ratio value >= 80%, color green

Disposition Request

A Disposition Request message is sent by the PBM/payer to retrieve the disposition of messages sent to Surescripts.

Disposition Acknowledgment

A Disposition Acknowledge message is sent by the PBM/payer as the final step to close processing on a message.

Patient Identifier

The PBM Unique Member ID returned in the 271 eligibility response (Subscriber, NM109) is used as the patient identifier.

Message Priority

PBM/payers have the ability to designate the message priority.

Provider vendors choosing the Web Application API implementation option for worklist will see messages sent with severity=H (high) sorted to the top of the worklist with an alert icon indicator.

Provider vendors choosing the Web Service API implementation option for worklist will see the workflow task priority set to alert in the data brought back from the call to obtain worklist tasks.

Message priority is designated in the Contraindication.severity element. Supported severity values are found here:  http://www.hl7.org/FHIR/2015May/v3/vs/SeverityObservation/index.html

Expiration

PBM/payers may send an optional expiration date in the messages to Surescripts that reflects the cadence of analytics and expected message refresh.

Additionally, Surescripts will enforce an upper limit expiration date for messages that do not contain an expiration date, and to ensure medication repository is consistently refreshed. This limit will be 30 days from the time it is received by Surescripts. Expired messages are retained by Surescripts for purposes of reporting, analytics, longitudinal products, etc.

PBM/payers designate an expiration date within the CommunicationRequest.extension.valuePeriod element.

Message types that support expiration are:

  • High Risk Medication
  • Medication Adherence
  • Missing Medication

<CommunicationRequest>

PDC Refresh Cadence

The PBM/payer should send an updated PDC ratio for each patient in the target population at least once every 30 days to assure the information presented to the provider is current. The Medication Management for Adherence will display the most current PDC for the patient that is less than 30 days old.

The PBM/payer may designate the last updated date in the FHIR® message Observation.meta.lastUpdated@value.

The PDC will display if the last updated date is < 30 days old.

Provider Opt-Out - Inbox

All providers will be opt-in for all message types by default. Providers may opt-out of receiving messages into their inbox by message type through the Surescripts Directory by disabling the Medication Management for Adherence message service levels. Opt-out is supported for the following message types:

  • High Risk Medication
  • Medication Adherence
  • Missing Medication

Providers that opt-out of receiving messages in their inbox will continue to have access to messages through the patient worklist.

Response from the Provider Vendor

Responding to a medication adherence message is strongly encouraged, but is optional.

The practitioner will be given the ability to select a response from a predefined drop down list and optionally free form text.

This feedback will be stored in a Surescripts repository and can be queried for by PBM/payers at any given time.
