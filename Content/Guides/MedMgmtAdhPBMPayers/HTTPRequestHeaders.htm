HTTP Request Headers

The FHIR® messages delivered to the Surescripts Medication Management for Adherence FHIR® service must include the HTTP request headers listed below.

Content Types and Encodings

According to Section 2.1.0.6 of the FHIR® specification Content Types and encodings,

“The formal MIME-type for FHIR® resources is application/xml+fhir or application/json+fhir. The correct mime type SHALL be used by clients and servers.” Later in that same section, “FHIR® uses UTF-8 for all request and response bodies. Since the HTTP specification (section 3.7.1) defines a default character encoding of ISO-8859-1, requests and responses SHALL explicitly set the character encoding to UTF-8 using the charset parameter of the MIME-type in the Content-Type header. Requests MAY also specify this charset parameter in the Accept header and/or use the Accept-Charset header.”

Because of these requirements, requests will need to include a Content-Type in the HTTP request header. Below are two examples:

  • Content-Type: application/xml+fhir; charset=utf-8
  • Content-Type: application/json+fhir; charset=utf-8

There is a bit of ambiguity in the HTTP 1.1 specification regarding whether a charset parameter included in the Accept header takes precedence over the Accept-Charset header or vice-versa. Surescripts currently will not look at the Accept or Accept-Charset header when creating the response. Surescripts will reply to the message using the MIME-type and charset that was specified in the Content-Type request header.

Authorization

Surescripts will use basic authentication as the security scheme for these HTTP based transactions. Therefore, standard basic authentication credentials will need to be provided in the Authorization header. The username and password will be provided by a member from the Surescripts Implementation team during the beta period.

Custom HTTP Request Header

In order for Surescripts to identify the sender of the transaction, we require senders to use a custom HTTP request header named x-sender-uid. The format of the value of this header is UidQualifierType-UidValue. The following is an example where the sender is a PBM/payer who is providing their payer identifier in the x-sender-uid header with T00000000012345 being the payer identifier.

x-sender-uid: PY-T00000000012345
