FHIR® Message Resources

Message Resources

This section provides guidelines for the data messaging interfaces between the PBM/payers and Surescripts.

For the complete set of requirements for each FHIR® resource see the FHIR® DSTU2 standards specifications here: http://www.hl7.org/FHIR/2015May/index.html

The tables below include only the data elements where Surescripts has specific requirements.

Message Data Element Requirements

This section contains message data element requirements for High Risk Medication, Medication Adherence and Missing Medication messages.

Disposition Request

This will return the disposition of PBM/payer messages. In order to keep the load on clients, servers and the network minimized, the server will return the results in a series of pages. The search result set contains the URLs that the client uses to request additional pages from the search set. For a simple RESTful search, the page links are contained in the returned bundle as links.

Disposition Request Example

[GET]{{host}}/fhir/api/v0/$get-disposition?pagesize={page size}&pageindex={page index}&startDate={start date YYYY-MM-DD}&endDate={end date YYYYMMDD}

Request Parameters

M/C - Mandatory or Conditional
