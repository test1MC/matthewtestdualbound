By default, the User ID is usually the first initial of the first name and the entire last name. In cases of duplicate names, a unique user ID will be created for the individual.

User Permission Levels

User permission levels are assigned to User IDs. The vendor participant is responsible for determining the permission levels for their users. Not all permission levels may be available to everyone.

To assign the permission level with visibility to patient information, which is protected health information (PHI), an administrator or manager of the vendor participant must log a case in the Self-Service Portal. See Support Case Management (Self-Service Portal) on page 1 for more information on the self-service portal.

Additional user permissions (e.g. CSPMedHistoryAccess, CSPMailboxAccess) are available and associated with User IDs based on certified service levels and access needs. These additional permissions are mentioned in the appropriate sections of this document. A request to Surescripts Customer Care must be made to assign additional user permissions.

A User Management Access permission level is available to a single admin or management team member of each vendor participant account. This permission level will allow the user to:

  • Unlock User IDs
  • Reset passwords
  • Create individual User IDs with the Basic permission level (no PHI access)

General Support and Directory User IDs

These are established when a new account is added to production. Both initial User IDs are created for new accounts to allow for initial interaction with the tool and both allow concurrent access.

General Support User IDs

General Support User IDs are assigned an Admin User permission level. The General Support User ID should be available to any vendor participant team member who requires the ability to review transactions on the network and view/update account records.

Directory User ID

By default the user ID is assigned an email from the accounts’ Implementation team. The Directory User ID is designed for behind-the-scenes directory administrative tasks. The Directory User ID is assigned the CSPDirectoryUpdateBasic User permission level.

General support and Directory User IDs do not receive PHI visibility access.

Individual User IDs

Individual User IDs are created upon request of the vendor participant. These Individual User IDs are often set up to provide specific access, including visibility to PHI. It is up to the vendor participant’s account administrator to determine who should have PHI access; the vendor participant is responsible for any internal background investigations or protocols to determine PHI access eligibility.

To request an Individual User ID, an administrator or management team member of the vendor participant account must log a case with Surescripts Customer Care, using the Self-Service Portal, and provide the necessary profile information to include:

  • Name
  • Email address
  • Direct telephone number
  • User Permissions
  • PHI Access (yes or no)

Password Reset or Unlock User ID

Password Reset

To reset your password, enter your User ID, click Forgot your password, and the email address associated with the User ID will receive an email with a new password.

Resetting a password will not unlock an account.

Unlock User ID

Security protocols dictate that after three failed attempts, the User ID is locked. The user may contact Surescripts Customer Care to unlock their ID before they are able to proceed, or the individual with the User Management Access permission level for their account may also unlock the User ID.

The following four methods may be used to have your User ID unlocked with Surescripts Customer Care:

  • Email
  • Live Chat
  • Phone
  • Self-service portal (SSP)

All passwords must be changed every 90 days, except Directory User IDs, which have passwords that do not expire due to their functionality.

Main Screen View

Once you’ve logged in successfully, the Notices screen opens. The menu bar on the left-hand side controls the functionality in the active window on the right-hand side of the screen.

Notices are used by Surescripts to communicate information to our participants on the network. Users will see the notices upon login and can check for any updated notifications by selecting the Notices link in the top right-hand corner.

If no notices are available, the Welcome message will display.

Change Password

To change your password, click the Change Password button, enter the old password and enter the new Password twice.

Fields with a red asterisk (*) are required.

User Profile (Time Stamp)

UTC (Coordinated Universal Time). A time standard based on International Atomic Time with leap seconds added at irregular intervals to compensate for the Earth’s slowing rotation. This is the baseline time stamp for all Surescripts transactions. By default, a user’s profile is set to UTC.

The user may prefer to set the Admin Console Message Search interface to their local time zone. In order to set your local time zone, click the User Profile button and select the desired time zone from the drop-down list. Check the Observe Daylight Savings Time check box if applicable; this feature will automatically engage. This time zone will be used for all search functionality.
