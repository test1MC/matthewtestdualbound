Directory Information Elements

DirectoryInformation is common to all Add and Update messages and their corresponding responses.

M

Element

Data Type

Comment

M

AccountID

an..35

ID of the account associated with the record, provided by Surescripts.

M

ActiveStartTime

Date Time

Date when the record will be enabled (in conjunction with a Service Level) and published to the Surescripts directory. Can set the ActiveStartTime to a future date to enable the record on the Surescripts network.

M

ActiveEndTime

Date Time

Date when the record is deactivated and will no longer be active in Surescripts directory. Will only display when within 7 days of end date; otherwise, expect this field to be blank. Note: To deactivate a record, ActiveEndTime must be set to the current date and ServiceLevel must not be present in the message. the record is removed from the directory download files 7 days after the ActiveEndTime.

Note: The DeactivateProviderLocation andDeactivateOrganization messages remove all service levels.

C

DirectAddressDefaultLocation

Boolean

The same Direct Address may be assigned to multiple directory records. This field allows customers to specify a default location for Clinical Direct messaging. When a provider or healthcare organization has multiple Direct Addresses, this field will be used by Surescripts to apply routing controls and configuration when delivery messages addressed to a Direct Address. If no record has the DirectAddressDefaultlocation set, Surescripts may be unable to deliver the message.

Used in conjunction with SPI records that include Direct Addresses. This field allows customers to specify a single default location when the same Direct Address is used across multiple SPIs for the same Provider. I no one single SPI record (with the same direct address present) has the DirectAddressDefaultLocation set, Surescripts may be unable to deliver certain clinical direct messages. The field is not required when a single SPI with a specific direct address is in place.

Required if DirectorySpecialties is present. Can repeat up to 50 times.

M

DirectorySpecialtyName

an..35

Required if Specialty is present. Surescripts directory identifier that can be used to designate additional information and characteristics that apply to the Pharmacy Organization record.

This field is required for Pharmacy Organization type and is not used for General or Payer Organization types. Values are case sensitive.

Values:

Types: Each pharmacy must be designated with one of the following Types:

  • MailOrder
  • Retail
  • InHouseDispensing

The following directory specialties may also be included (as applicable):

  • Compounding
  • DME
  • FaxPharmacyExternal (may only be set by Surescripts)
  • FaxPharmacySurescripts (may only be set by Surescripts)
  • Kiosk
  • LongTermCare
  • Specialty
  • SupportsDigitalSignature
  • TwentyFourHourStore

Note: In Admin Console, this field is called "Directory Specialties"

Note: Additional values may be added. Systems should not fail if an unexpected value is received.

C

FaxBackup

n..1

Boolean. Indicated that messages addressed to this directory entry allow Surescripts to attempt Fax delivery for failed messages if all attempts for electronic delivery have failed.

Values

-1 = Turned Off

1 = Turned On

Note: Pharmacies in certain states, e.g. NY, may only use the value "-1" BackupPortalID. These states do not permit use of fax backup for pharmacy.

M

PortalID

an..35

ID of the portal associated with the organization provided by Surescripts. If an account needs to support more than one application version, Surescripts can set up multiple Portal IDs.

Note: Enforced by business rule, not schema.

C

ServiceLevels

Complex Type

ServiceLevels is a collection of ServiceLevel elements in the schema. The schema allows up to 50 ServiceLevel elements. One ServiceLevelName is allowed for each ServiceLevel element. Do not send if there are no services.

M

ServiceLevel

Complex Type

Required if ServiceLevels element is present. Currently a maximum of 50 service levels are allowed.

M

ServiceLevelName

an..50

Required if ServiceLevel is present.

Service Levels(s) associated with the account record. Designated by the account record capabilities. Values are case sensitive. For more detail, please refer to ServiceLevelTypes.

C

Test

 

For Surescripts' internal use only.

C

UseCases

Complex Type

Used to further define a message. A use case has a defined set of expectations for message content for those participating.

M

UseCase

Complex Type

Required if UseCases is used. Use cases are used to identify customers whose applications are capable of sending and receiving specifically defined content.

Up to 10 occurrences of UseCase are permitted. More could be added in the future. Refer to schema for more details.

Valid values will be distributed in documentation specific to each use case.
