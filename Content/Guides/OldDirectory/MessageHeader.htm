Message Header

Message Header Elements - Descriptions

All directory functions will use the following message header.

Requirements Designation

Each segment and data element has a requirement designation. If a data element is listed as Mandatory, it must be present in the transmission. If a Conditional complex type data element is used; all mandatory data elements contained within it are required.

Surescripts uses a message header for all messages.

Code

Description

M

Mandatory — the element must be used.

C

Conditional — the element must be used if conditions are met. Some fields do not have specific conditions. Data should be sent if available.

This guide only includes data elements where Surescripts has specific requirements or further explains the field usage. Refer to the Directory Schema listed in Document References for a complete list of fields.

M MessageID an..35M

The message sender must ensure the combination of the SenderID (value of the FROM element) and the MessageID cannot be reused for at least 18 months. Surescripts recommends unformatted Globally Unique Identifiers (GUID) for generating message IDs.

M SentTime Date time

Date and time of initiation. Coordinated Universal Time (UTC) should be used.

YYYY-MM-DDTHH:MM:SS.FZ

Example: 16:09:04.5Z would be 16 hours, 09 minutes, 04 seconds, 5 fractional seconds, and Z to denote UTC time.
