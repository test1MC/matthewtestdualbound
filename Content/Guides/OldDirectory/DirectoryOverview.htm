Directory Overview

About Directory

Surescripts maintains organization and provider directories for message routing. Proper management and maintenance of records by customers is critical to successful message exchange. Customers rely on the information maintained by other customers to find the appropriate communication paths. The directory is an enabling capability used in conjunction with the customer's implemented product(s).

The Surescripts Directory Messaging Interface is the primary method for customers to perform the task of directory administration. Directory Messaging is a set of API calls that can perform many of the same tasks as the Surescripts Admin Console. Admin Console will provide the latest capabilities and validations available for Surescripts directory information.

The Directories messaging allows for single-source transaction flow and automation of directory maintenance. Applications using Surescripts Directory Messaging can be implemented on any platform in any programming language, as long as the application can create and consume messages defined for the service interface. For additional information, see Examples.

About This Guide

This guide illustrates the use of Surescripts Directory messaging, which allows for updates to the directory information, as well as the ability to download aggregate copies of the data. Updates may consist of adding new customers, making changes - such as location - and deactivating the directory entry of customers that leave the network.

The terms "message" and transaction" are used interchangeably throughout this guide. "Transaction" is used when referencing language pulled directly from a standards organization document which uses the term "transaction".
