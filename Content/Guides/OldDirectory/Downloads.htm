Downloads

Network customers can download flat file lists of organizations and providers registered within the network. The lists are filtered based on the access of the network customer performing the download. Customer may choose to do a full nightly download or nightly delta downloads and complete a full update at least once a week. The same rules apply to downloads via the Admin Console or Directory Download Request messages.

Surescripts generates Full download files every night, which contain every organization/provider available to the network customer performing the download. Nightly download files are a delta file of the records that changed during the course of that specific day.

The Directory Download Response will either be an Error message (when there are no new updates or an error is encountered) or a DirectoryDownloadResponse message. There are two ways to get the directory downloads:

  1. Submit a directory download request message via directory messaging process. Surescripts processes the request and responds with either an Error Message or a DirectoryDownloadResponse, which includes the file name.
  2. Admin Console: “Directory Downloads” web page (refer to the Admin Console guide for directory downloads.)

Provider Flat File Overview

Provider demographic and routing information are valuable assets to our provider network customers. A provider/prescriber vendor has access to demographic data for providers they maintain. An organization/pharmacy vendor has access to demographic data for all e-prescribing providers. In addition, all vendors who support clinical messaging have access to demographic data for providers registered with a clinical messaging service level, such as CIMessage or CIEvent, regardless of which vendor registered the provider.

Newly registered e-prescribing providers will be active in the directory immediately. However, pharmacy customers will see the Refill service level in the download only after five successful NewRx messages have been sent by the provider.

Directory download files include all records to which the requester has access.

Full directory download files:

  • Records with ActiveEndTime more than 30 days in the past will not be included in directory files.
  • The record detail will remain in the Full download file for 30 days from the ActiveEndTime for a deactivated provider.

Nightly/Delta directory download files:

  • All records with ActiveStartTime within the last 24 hours will be included with RecordChange field value of "N" (for New).
  • All records with ActiveEndTime within the last 24 hours will be included with the RecordChange field value of "D" (for Deactivated).

A deactivated record contains no service levels and an ActiveEndTime in the past.

  • If none of the above applies and LastModifiedTime is within the last 24 hours, the record will be included with a RecordChange field value of "U" (for Updated).

Flat File Format

Each field is delimited by the pipe character (|). Each line is separated by a new line (hex 0A) character. The tilde character (~) is used as a repetition character. If a pipe character (hex 7C) is used in field data, it will be escaped when returned in the directory download file with \F\. For example, Clinic | Name would be returned in the directory download as Clinic \F\ Name.

Organization Flat File Overview

An organization is an entity that can send and receive messages using the proper credentials. An organization can be a Pharmacy, Payer or General Organization, such as a clinic, private practice, or hospital department. Organization demographic and routing information is a valuable asset to our organization customers.

An organization/pharmacy vendor has access to demographic data for organizations they maintain. In addition, organization/pharmacy vendors who support clinical messaging have access to demographic data for organizations registered with a clinical messaging service level, such as CIMessage or CIEvent regardless of which vendor registered the organization. A provider/prescriber vendor has access to demographic data for all pharmacy, payer, and general organizations with matching service levels. In other words, provider/prescriber vendors with a clinical messaging service level will have access to organizations with a clinical messaging service level and e-prescribing providers will have access to e‑prescribing organization/pharmacies, etc.

The Flat File Name and location is contained in the Directory Download Response.

Directory download files include all records to which the requester has access.

Full directory download files:

  • Records with ActiveEndTime more than 30 days in the past will not be included in directory files.
  • The record detail will remain in the Full download file for 30 days from the ActiveEndTime for a deactivated provider.

Nightly/Delta directory download files:

  • All records with ActiveStartTime within the last 24 hours will be included with RecordChange field value of "N" (for New).
  • All records with ActiveEndTime within the last 24 hours will be included with the RecordChange field value of "D" (for Deactivated).

A deactivated record contains no service levels and an ActiveEndTime in the past.

  • All records with ActiveEndTime in the past -- but not with ActiveEndTime within the last 24 hours will not be included.
  • If none of the above applies and LastModifiedTime is within the last 24 hours, the record will be included with RecordChange field value of "U" (for Updated).
