Organization Messages

AddOrganization

A customer can use the AddOrganization message to add a new organization to the directory if the organization is managed by the customer account (Account ID). The organization type must be specified in the message as either: General, Pharmacy, or Payer. All organizations added to the directory by a customer will be permanently associated with that customer. When adding a Pharmacy organization type, the NCPDPID specified in the AddOrganization message must not already exist in the Surescripts pharmacy directory or an error message will be returned.

<PhoneNumbers>

ComplexType

Required unless Organization has clinical messaging service levels ONLY.

Phone number should be where organization would like to be contacted.

  • The first Phone number (TE) is required for pharmacy organization and always the primary phone number in 5553334466x1234 format, where x1234 is extension and optional. Please note this extension is just an example. Less/more than four digits is allowed.
  • The second Phone number (FX) is required for pharmacy organization and is always the primary fax number in format 5553334466. The fax number will be used to communicate patient confidential information and therefore must be secure from the public.

  • For general organization types, a phone (communication) number of any type is required.

Note: An additional PhoneAlt6 and PhoneAlt6Qualifier can be added via the Admin Console and the Update and Add messages; however, this phone number/qualifier will not be populated in the organization download.

AddOrganizationResponse

When an organization is successfully added, the AddOrganizationResponse message is returned to the customer noting that the organization was added successfully. The message contains a new OrganizationID for the organization which may be up to 10 digits, and echoes back the information that was in the AddOrganization message.

UpdateOrganization

A customer can update directory information for any organization record that is managed by the customer account (Account ID). Maintenance may include: updates of demographic information, activation or deactivation of organizations and maintenance of service levels.

UpdateOrganization will override (set to null or default) any optional element that is not included in the update message. To avoid unintentional deletion of data, it is recommended to first utilize a GetOrganization message to retrieve current information prior to making the desired updates; then submit the UpdateOrganization message.

GetOrganization

The GetOrganization function is used to retrieve organization information based on an existing organization’s OrganizationID for any organization type or the pharmacy NCPDPID for a Pharmacy Organization type. A recommended use for this message is to obtain real-time information prior to updating an organization. The organization information will be returned only if the account (software vendor) owns the organization record.

Error Scenarios:

  • If the data supplied or the structure of message is invalid, an error message will be returned.
  • If there are no organization records found that match the given Organization or NCPDP ID, or the record is not managed by this account, an error message indicating that the organization is not found will be returned.

The following options are available to obtain the latest information on the organization you wish to update.

Using Directory Messaging, send a GetOrganization message. Receive a response of GetOrganizationResponse or an Error Message.

GetOrganization Workflow

  1. Retrieve the existing organization information using GetOrganization message.
  2. Change <GetOrganizationResponse> to <UpdateOrganization> to generate the base UpdateOrganization message.
  3. Update the desired values in the base UpdateOrganization message.
  4. Submit the UpdateOrganization message for processing.
  5. Receive either a response of Status or Error.

To ensure that an organization is completely disabled, the organization should set the ActiveEndTime to a date in the past and service level should not be present.
