Troubleshooting/FAQ

Edge Test Tool Launch Errors: If users cannot see the Edge Test Tool login screen, they must ensure that they are launching from a white-listed IP. Contact Surescripts Customer Support to identify which IPs have been white-listed for Staging environment or to add additional IPs.

400 BadRequest: See the response raw body in the Edge Test Tool for the cause of the error message. The message content is malformed or not valid according to FHIR®.

401 Unauthorized: Ensure that the Sender UID, Username and Password entered into the Edge Test Tool are the values provided to you by the integration project manager.

422 Message Fails Validation: See the response raw body in the Edge Test Tool. The specific error message is found in the entry.resource.OperationOutcome data element.
