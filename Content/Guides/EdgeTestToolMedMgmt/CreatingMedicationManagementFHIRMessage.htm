Creating a Medication Management FHIR® Message

Accessing the FHIR® Message Composer

To generate a Medication Management FHIR® message, use the Compose menu drop-down to select Compose FHIR Message.

Composing a FHIR® Message

The Edge Test Tool will accept a FHIR® message in XML or JSON format for the purposes of Medication Management for Adherence messaging. Users are free to compose their own message, utilizing the HL7® FHIR® standard v0.50, or may use a templates provided by Surescripts. All edits must be made either in the template prior to loading or directly in the message text box. If testing in the EHR system requires specific patients, medications or providers users should ensure that all of the appropriate fields are correct in the XML or JSON before submitting to the system.

Loading a FHIR® Message

To load an XML template, users must have the template stored on their local machine or a network location that is accessible to the user.

  1. From the FHIR® Message Composer screen, click the Choose File button located at the bottom of the page.
  2. Select the appropriate template and click the Load File button.

    3. The XML message content is displayed in the XML message text box in the center of the page. Users can modify the contents of the message from the text box directly as needed.

  3. Refer to your STAGE test plan for the values to be used for the Sender UID, Username and Password. These values are used for authentication when the FHIR® message is Posted to the Mailbox.

    4.  

    Input Name

    Notes

    Sender UID

    Payer Id prefaced by PY-

    UserName

    Example: user1

    Password

    Example: password1

    NPI

    The value for the NPI is your provider’s NPI. The NPI is used to target a message to the provider’s inbox.

    Patient Member Id

    Patient Member Id will override the Patient.Identifier.value@value included in the FHIR® message loaded. Leave this text box blank, for STAGE testing we will not override the Patient Member Id.

    Example: 1215926753

    Content Type

    Content Type allows sending XML or JSON formatted FHIR® transactions.

    Defaults to application/xml

    Accept Type

    Content Type allows sending XML or JSON formatted FHIR® transactions.

    Defaults to application/xml

    Generate Id

    Enable the Generate Id option. This is to ensure a unique MessageID as the message goes across the Surescripts network. This option should remain enabled unless the user is attempting to test the impact of different message ids.

  4. Once the message is verified for accuracy, click the Post To Mailbox button to send the FHIR® message to the Surescripts system.

  5. Once successfully sent, users can see response content on the lower portion of the screen. For help with specific errors, please contact Surescripts Customer Support.
    • A response status = 200 OK is success. If an error occurs an error response code and error message will indicate the root cause.
