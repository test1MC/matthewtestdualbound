Customizing FHIR® Messages

The default set of FHIR® messages you receive for testing may be customized to your test plan. This section assumes you are familiar with the FHIR® standard. This Guide utilizes the HL7® FHIR® standard messaging standard DSTU 2 version 0.5 as a baseline. Documentation can be obtained through HEALTH LEVEL SEVEN®.

The HL7® FHIR® standard is copyrighted and may be obtained from: http://hl7.org/fhir/2015May/index.html

Targeting Message to a Provider

To send a High Risk Medication, Medication Adherence or Missing Medication message to a provider worklist enter the provider’s NPI into the Edge Test Tool NPI text box.

If the Edge Test Tool NPI text box is blank, the message will NOT appear on the provider worklist.

The provider must have the High Risk Medication, Medication Adherence and Missing Medication service levels enabled for a message to be targeted to the provider inbox.

PBM/payers have the ability to designate the intended target of the communication/message through the CommunicationRequest.recipient element. Inclusion of this element will send the message to a practitioner inbox.

High Risk Medication, Medication Adherence and Missing Medication messages you received include the recipient data element.

Changing Priority of a Message

In the Surescripts accelerator UI urgent worklist messages have a priority of alert and are sorted to the top of the worklist with a red flag alert icon indicator.

Message priority is designated in the Contraindication.severity element. To designate a High Risk Medication, Medication Adherence or Missing Medication Message with an alert priority, add the severity data element with a value of “high”.

Changing PBM/Payder Message Text

The message seen on the task detail UI is included in the Contraindication.text.div value.

To customize the text message change the text included in the Contraindication.text.div value.

Customizing Proportion of Days Covered (PDC) Score

The PDC score is contained in the PDC FHIR® message Observation resource within the contained.MedicationDispense.dosageInstruction.scheduleTiming.code.coding.value.

PDC Value Interpretation based on the PDC score by the adherence profile summary API.
