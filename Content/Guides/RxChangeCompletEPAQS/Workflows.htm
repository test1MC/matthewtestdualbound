Workflows

The following section outlines the steps a pharmacy should take depending on whether the physician is enabled for RxChange and if the pharmacy or Surescripts does the faxing. Surescripts supports two workflows depending on whether the pharmacy is checking for the provider’s ability to accept RxChange or allowing Surescripts to perform that check on the pharmacy’s behalf.

Surescripts Determines Route – RxChange Request to Generic SPI

In this workflow the pharmacy sends the RxChangeRequest to a generic SPI and Surescripts determines RxChange or fax route.

The following steps depict the RxChange message flow when Surescripts determines whether the provider has an RxChange service level and faxes the prior authorization notification on behalf of the pharmacy.

Surescripts uses a generic faxing template at this time and there is no pharmacy-specific branding. The fax template provides instructions for using CompletEPA® QuickStart™ and it also outlines the other options available to providers for responding to the PA notifications.

  1. Pharmacy processes the prescription and identifies the need for prior authorization.
  2. Pharmacy sends Surescripts an RxChangeRequest to SPI “0000000000000” (a generic SPI).

    3. Surescripts requires the pharmacy to send the provider’s fax number. Pharmacies should review fax number accuracy on a regular basis and have a secondary process in place if the fax process fails.

  3. If the RxChangeRequest is related to a NewRx, include the BenefitsCoordination from the NewRx.
    • Pharmacy must always send the BIN.
    • The pharmacy must always send the PCN, if known.
  4. Surescripts attempts to match the provider demographics in the RxChangeRequest to a provider with an RxChange service level in the Surescripts Directory.
    1. If Surescripts finds a match:
      1. Surescripts sends the RxChangeRequest to the provider’s RxChange-enabled SPI.
      2. Provider vendor receives the RxChangeRequest and initiates the prior authorization with the PBM (pharmacy benefit manager)/payer identified in the BenefitsCoordination composite.
      3. Provider vendor completes the ePA process with the PBM/payer. 
      4.  Provider vendor sends an RxChangeResponse to the pharmacy via Surescripts and notes whether the prior authorization was approved or denied.
    2. If Surescripts does not find a match:
      1. Surescripts faxes a prior authorization notification, including a Surescripts-generated unique transaction (TRX) code, to the prescriber using the pharmacy-provided fax number. The fax includes directions to the provider to complete the PA.

        2. Once Surescripts generates the unique transaction code, the provider has seven days to sign into CompletEPA® QuickStart™ to complete the ePA. The transaction code expires after that time.

      2. If the provider chooses CompletEPA® QuickStart™, the provider uses a unique transaction code that is found on the fax to initiate the PA..

        3. If the PBM/payer is not enabled for CompletEPA®, Surescripts allows the provider to complete a fillable PDF form which is faxed to the PBM/payer. Please see the section on ConnectiveRx Functionality for more details.

      3. Once the ePA process is complete, Surescripts sends an RxChangeResponse to the pharmacy with an approval/denial response and the authorization number, if applicable.

      4. The RxChangeResponse is addressed from 8398058138001, not the generic SPI the pharmacy initially addressed the RxChangeRequest to.

        5. In cases where the prescriber does not complete the PA using CompletEPA® or CompletEPA® QuickStart™, Surescripts cannot send an RxChangeResponse as the NCPDP standard does not have an appropriate reason when the outcome of the PA is unknown. Pharmacies should utilize a secondary process for notifying the patient if they do not receive an RxChangeResponse after 72 hours.

Pharmacy Checks Directory to Determine Route – Surescripts Faxes

In this workflow the pharmacy checks the Surescripts directory to determine the provider’s ability to accept the RxChange.

The following steps depict the RxChange message flow when the pharmacy determines whether the provider has an RxChange service level and Surescripts faxes the prior authorization notification on behalf of the pharmacy.

Surescripts uses a generic faxing template at this time and there is no pharmacy-specific branding. The fax template provides instructions for using CompletEPA® QuickStart™, and it also outlines the other options available to providers for responding to the PA notifications.

  1. Pharmacy processes the prescription and identifies the need for prior authorization.
  2. Pharmacy attempts to match the provider demographics to a provider with an RxChange service level in the Surescripts Directory.
    1. If the pharmacy finds a match:
      1. The pharmacy sends the RxChangeRequest to the provider’s RxChange-enabled SPI.
      2. Provider vendor receives the RxChangeRequest and initiates the prior authorization process with the PBM/payer identified in the BenefitsCoordination composite.
    2. If the pharmacy does not find a match or if the provider is not in the Surescripts Directory:
      1. The pharmacy sends Surescripts an RxChangeRequest to SPI “0000000000000” (a generic SPI).

        2. Surescripts requires the pharmacy to send the provider’s fax number in the RxChangeRequest.

      2. If the RxChangeRequest is related to a NewRx, include the BenefitsCoordination from the NewRx.
        • Pharmacy must always send the BIN.
        • The pharmacy must always send the PCN, if known.
      3. Surescripts attempts to match the provider demographics to a provider with an RxChange service level in the Surescripts Directory.
    3. If Surescripts finds a match:
      1. Surescripts sends the RxChangeRequest to the provider’s RxChange-enabled SPI.
      2. Provider vendor receives the RxChangeRequest and initiates the prior authorization with the PBM/payer identified in the BenefitsCoordination composite.
      3. Provider vendor completes the ePA process with the PBM/payer.
      4. Provider vendor sends an RxChangeResponse to the pharmacy via Surescripts and notes whether the prior authorization was approved or denied.
    4. If Surescripts does not find a match:
      1. Surescripts faxes a prior authorization notification, including a Surescripts-generated unique transaction (TRX) code, to the prescriber using the prescriber’s fax number from the RxChangeRequest. The fax template provides instructions for using CompletEPA® QuickStart™ and it also outlines the other options available to providers for responding to the PA notifications.

        2. Once Surescripts generates the unique transaction code, the provider has seven days to sign into the CompletEPA® QuickStart™ to complete the ePA. The transaction code expires after that time.

      2. If the provider chooses CompletEPA® QuickStart™, a unique transaction code that is found on the fax will be used to initiate the PA.

        3. If the PBM/payer is not enabled for CompletEPA®, Surescripts allows the provider to complete a fillable PDF form which is faxed to the PBM/payer. Please see the section on ConnectiveRx Functionality for more details.

      3. Once the ePA process is complete, Surescripts sends an RxChangeResponse to the pharmacy with an approval/denial response and the authorization number, if applicable.
      4. Surescripts addresses the RxChangeResponse From 8398058138001, not the generic SPI.

        5. In cases where the prescriber does not complete the PA using CompletEPA® or CompletEPA® QuickStart™, Surescripts cannot send an RxChangeResponse as the NCPDP standard does not have an appropriate reason when the outcome of the PA is unknown. Pharmacies should utilize a secondary process for notifying the patient if they do not receive an RxChangeResponse after 72 hours.
