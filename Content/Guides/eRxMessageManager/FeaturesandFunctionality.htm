Features and Functionality

The eRx Message Manager includes features that allow a user to create and customize messages using templates, to impersonate a message receiver, to generate response messages, and to send messages through the Surescripts Staging switch.

The application reads from the Surescripts staging directory to supply the user with test endpoints (e.g., providers or pharmacies).

  1. On the Sign In page, enter your Login Id and Password (these are same as your admin console credentials).

  2. Click eRx Message Manager.

  3. The home page for eRx Message Manager appears. From here, you can Compose a new message within this tool, Search for a transmitted message, or create/edit a Template to build a message in Compose.

Compose

Compose is used to create messages to send to the application for processing.

  1. Select the endpoints - who the message is From and To.
    1. From (default is a drop-down list of Surescripts prescriber or pharmacy test endpoints); and
    2. To (default is a type ahead search list of the customer’s Surescripts staging directory data).
  2. Select the Version.
    1. Select either 10.6MU or 6.0 that will apply to the message.
  3. Select a message Type to send.
    1. Graphical user interface (GUI) options will pop up after message Type has been selected.
  4. Choose a message Template.
    1. The Compose Form (GUI fields) and Edit XML function will automatically be populated when Template is selected, or
    2. Enter the data into the Edit XML function.
  5. Users can view, add or change data in two ways:
    1. In the Compose Form, or
    2. In the Edit XML function.

      3. Any change to a Compose Form field, will also change the XML, and a change in the Edit XML will change the corresponding field in the Compose Form. More fields can be added into the XML, but those additional fields will not show in the Compose Form.

    3. The Edit XML small view enables the user to select a section to move around the message content in a quicker manner than scrolling. The selected section will be gray, and the message content window will jump to the selection that is highlighted.
  6. Click Send Message.

Always remove PHI before sending the message.

Search

Searching will result in a display of all messages sent from a customer’s account to the eRx Message Manager.  The most recent messages will appear at the top of the list.

  1. Use the Search feature to find a message that was sent to a Surescripts endpoint. Search parameters allow searching to be based on the:
    1. Sender (From field).
    2. Message Type (organized by version then type).
    3. Combination of both (1a and 1b).
  2. The search results displays the most recent messages for the account associated with the user who is logged in. Each page of results will show up to 25 records (click on the next page to see more).
  3. Click the Sender Message ID to see the XML details.
  4. Click Respond to begin constructing a response message based on the initial message. Only applicable response message types applicable to the original message will be in the list.
  5. The Compose page will be displayed showing the endpoints reversed and a choice of appropriate response type messages.

    1. Select the Response Type, and a new response message will be built based on the original message content (the same drug, patient, provider and pharmacy).
    2. The response message will include the Relates To Message Id (the ID from the initial message).

Templates

  1. Templates are created by the user and saved to an account, or the user can choose to use sample Surescripts templates. Templates are helpful for test cases that are frequently tested or for edge cases that will need to be tested after a vendor’s software changes.
  2. All users will be able to view, copy, edit or delete templates after saving them. Only user created templates can be edited or deleted. Surescripts templates are read-only, but can be copied and edited to make them account specific. All users that belong to a particular account have access to those templates. Key fields in the template are automatically updated when it is selected (e.g., the WrittenDate is set to today's date). Provider and pharmacy information is also updated and is pulled from the directory when the endpoints are selected.

For EPCS, the element that indicates that a message has been digitally signed can be populated, but the actual digital signature is not included.
