Processes

Test Message Compliance

There are two levels of compliance:

  • Validation of XML.
  • Compliance with business rules. Business Rules are data validations applied by Surescripts beyond the schema requirements that will cause a message to be successful or rejected.

XML Validation

  1. Create a new message from the Compose page or create a response message.
  2. Select the message, and click Validate XML to validate the message against the syntax rules of the XML schema.

  3. Validation errors are displayed at the bottom of the screen.

Translation and Business Rules

When sending test messages through the eRx Message Manager, Surescripts will perform any necessary data translation and business rule verification.

  1. If any business rules are violated, an error will display on the screen.

Example Test Scenarios

Users will be able to send messages from a Surescripts test prescriber or pharmacy sender to their application to test functionality.

These process flows will apply to any of the response messages a user may want to test.

Provider Vendor or IDN Scenario Test Receipt of RxRenewalRequest

  1. Send the NewRx from your application to a Surescripts Test Pharmacy endpoint.
  2. Search for the NewRx in eRx Message Manager using the Search function.
  3. Click Respond to build a response message.
  4. Select RxRenewalRequest type message from the drop-down.
    • Edit fields, or add to the response message if desired.
  5. Validate XML.
  6. Send the RxRenewalRequest to your application endpoint.

Pharmacy Scenario Test Receipt of NewRx, RxRenewalResponse, and Creation of RxRenewalRequest

  1. Send the NewRx from the eRx Message Manager to your Pharmacy.
  2. Select Compose.
  3. Select a Surescripts test provider from the From drop-down list.
  4. Select your target test pharmacy from the To type-ahead search.
  5. Select Version.
  6. Select NewRx in the message Type drop-down.
  7. Select the desired message from the Template drop-down and edit if needed.
  8. Select Validate XML if the message was modified.
  9. Click Send.
  10. Send the RxRenewalRequest from your application based on the NewRx that was received, back to the Surescripts test endpoint.
  11. Select Search in eRx Message Manager.
  12. Find the RxRenewalRequest message sent from your application.
  13. Click the Respond link to begin building a response message.
  14. On the new Compose page, select RxRenewalResponse in the message Type drop-down.
  15. Select a template from the Template drop-down.
  16. Edit the message, being sure to choose a RxRenewalResponse type (e.g. Approved, Denied).
  17. If the XML was edited, click Validate XML.
  18. Click Send.
