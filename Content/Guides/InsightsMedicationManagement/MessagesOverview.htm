Messages Overview

The APIs allow provider vendors to quickly offer customers core Insights for Medication Adherence features. By incorporating the data returned from the APIs into the provider vendor system, the provider vendor will manage the process of viewing and responding to medication management messages.

The provider vendor determines how best to integrate the medication management features into their application.

Medication management may be initiated at the time of a patient visit or when the provider views their inbox.

Insights for Medication Adherence Message Flow

Insights for Medication Adherence provides provider vendors with the electronic delivery of PBM/payer medication adherence messages and patient medication adherence profile. Through the provider vendor interface, the provider can view and respond to PBM/payer medication adherence messages and access the patient's adherence profile. The graphic below depicts the transaction flow between customers in this process.

  1. PBM/payer sends messages (High Risk Medication, Medication Adherence, and/or Missing Medication) and PDC ratios to Surescripts destined for a provider and a specific patient. Messages and PDCs are XML or JSON based, discrete and structured data elements.
  1. Surescripts processes messages and PDC ratios upon receiving, forwards urgent messages directly to the provider’s inbox.
  2. Non-urgent messages and PDC ratios are available during the patient visit.
  3. Surescripts closes out those messages which have been acted upon, ensuring provider’s inbox and patient’s messages list only display messages needing viewing and attention.
  4. There are two ways Surescripts delivers messages into a provider’s environment:
  5. When the PBM/payer designates a provider, messages will be delivered to the provider’s inbox.
  6. All messages and PDC ratios will be available for viewing within patient’s chart.

The PBM/payer is responsible for sending the NPI to Surescripts, which is then used to route the message to the appropriate provider SPI(s).

  1. Physicians can provide feedback for messages read and feedback is sent to Surescripts for processing.
  2. PBM/payer pulls feedback to complete the communication loop.

Provider Vendor Responsibilities

An integration with Insights for Medication Adherence requires implementing a few components in the provider vendor’s system.

User Interface

Customer will be supporting five user-interface components in their system.  Examples below are from the Surescripts Accelerator UI

  1. Viewing summary information for each patient in the patient visit workflow showing the patients adherence summary and the list of actionable messages along with the priority of the messages
  2. Viewing message details for High risk medication, medication adherence and missing medication messages
  3. Viewing the patient’s adherence profile of PDC ratios
  4. Responding to high risk medication, medication adherence and missing medication messages
  5. Viewing the patient or provider worklist of actionable messages

User Access

Provide role-based access controls to manage delegation of responsibilities among providers (prescribers and their staff).

Webserver

Run a webserver to support GET and POST operations to the Surescripts endpoints

Provider

Managing the provider service levels in the Surescripts Directory, which controls the messages that will appear on the provider worklist.

Provider Inbox Opt-Out

All providers will be opt-in to the provider inbox for all high-risk medications, medication adherence, and missing medication messages once they are set up in the Surescripts Directory. See the Surescripts Directory Implementation Guide for information on adding and removing service levels for providers. Providers may opt-out of receiving messages into their inbox by message type through the Surescripts Directory by disabling Insights for Medication Adherence message service levels. Opt-out is currently supported for the following message types:

  • High Risk Medication
  • Medication Adherence
  • Missing Medication

Providers that opt-out of receiving messages in their inbox will continue to have access to messages through the patient worklist.

Authorization and Authentication

The provider vendor is responsible for authenticating the user and determining what data and activities users will have access to.

The provider vendor:

  • Maintains its users, their roles, their relationships (e.g., delegation of authority from providers to staff), and their authentication credentials, etc.
  • Includes pertinent user information when initiating the CommunicationRequest and QuestionnareResponse:
  • the specific roles the user possesses
  • the provider SPI on whose behalf the user can act

The medication management system:

  • Logs the user’s role(s) assigned by the provider vendor

User Roles

Users are authenticated by the provider vendor. The provider vendor authorizes a user to perform a task by assigning them to a role. The types of roles are:

  • Viewer: The Viewer has read-only permissions.
  • Submitter: The Submitter has create, read, edit, and submit permissions.
  • Prescriber: This is an optional submitter role that may be used when the EHR vendor is able to discern if the end user is a prescriber.

The user may have different roles depending on their relationship to the prescriber (e.g., a user may have the Submitter role when working on behalf of Dr. Smith and the Viewer role when working on behalf of Dr. Jones).

The users and their roles are not stored by Surescripts. The provider vendor must securely communicate the roles to Surescripts.
