RLE 1.0 Messages Overview

Responsibilities of an IHE Enabled Vendor

As described above, the primary responsibility of a requesting location is to submit record requests to the Surescripts.

In addition, IHE enabled requesting locations are also asked to be IHE enabled, responding locations. This involves two responsibilities:

  1. Provide Surescripts with covered patients. This could be done through loading the Master Patient Index (MPI) or by responding to XCPD Requests from Surescripts.
  • Reference the Surescripts Master Patient Index (MPI) File Load Supplement Guide or the INSERT XREF TO ADT MESSAGES OVERVIEW SECTION Surescripts ADT Data Feed Implementation Guide for more information.
  1. Respond to XCA Query and Retrieve requests from Surescripts and other customers.

Message Descriptions

XCPD ITI-55 (Cross Community Patient Discovery)

The IHE XCPD Request/Response supports the means to locate communities that hold patient records and the translation of patient identifiers across communities holding the same patient's data. The request is sent from the requesting location to the Surescripts to discover if a patient matching the demographics sent from the requester is known. The response sent from Surescripts to the requesting location indicates if there was a patient match or no match.

PLQ ITI-56 (Patient Location Query)

The IHE PLQ Request/Response queries for a list of communities that may have records for a patient referenced by a patient identifier. The request message is sent from the requesting location after receipt of an XCPD Response from Surescripts indicating a patient match. The response sent from Surescripts to the requesting location contains the patient’s record locations.

XCA Query ITI-38 (Cross-Community Access)

The IHE XCA Query Request/Response supports the means to request retrieval of the list of available documents from responding locations.

XCA Retrieve ITI-39 (Cross-Community Retrieve)

The IHE XCA Retrieve Request/Response supports the means to directly obtain documents from IHE enabled responding locations.

Surescripts Location Summary CDA

Surescripts Location Summary CDA contains information on locations where a patient has been seen based on information gathered from the Surescripts network.

Per IHE standards, the Location Summary CDA is an on-demand document.  Metadata for the document will be returned in response to an XCA Query transaction. The actual CDA will be returned in response to an XCA Retrieve for the Location Summary CDA Unique Document ID. As an on-demand document, the content of the Location Summary CDA will reflect the most recent data available as of the date and time the XCA Retrieve transaction is processed.

Discover the Patient

1a) The requesting location sends an XCPD Message to query the Surescripts and discover if a patient matching the demographics sent is known.

1b) responds with a patient match or no match.

Discover Responding Record Locations (optional)

2a) The requesting location sends a PLQ Message to retrieve from the a list of locations that have records for the patient.

2b) The responds with record locations identified by home community and patient identification.

Query and Obtain Patient Records Message Flow when PLQ is used to Discover Responding Locations

does not maintain a Clinical Document Repository. Documents are retrieved from the source.

There are three possible paths for retrieving documents from the responding locations including the following:

  • Obtain patient records from IHE enabled vendor through Surescripts
  • Obtain Location Summary Document from Surescripts
  • Obtain patient records directly from IHE enabled vendor

Scenario 1: Obtain Patient Records from IHE Enabled Vendor through Surescripts

3a) An XCA Query Request is sent to Surescripts requesting patient records from the record location. The request is forwarded to the responding location.

3b) The responding location sends an XCA Query Response to Surescripts, which is forwarded to the requesting location.

4a) An XCA Retrieve Request is sent to Surescripts requesting retrieval of the patient records. The request is forwarded to the responding location.

4b) The responding location sends the patient records to Surescripts in an XCA Retrieve Response, which is forwarded to the requesting location.

Scenario 2: Obtain Location Summary Document from Surescripts

3a) An XCA Query Request is sent to Surescripts.

3b) Surescripts sends an XCA Query Response with the Document ID and metadata of the dynamically generated Location Summary Document.

4a) The requesting location sends an XCA Retrieve Request for the patient record locations.

4b) Surescripts sends the Location Summary Document in the XCA Retrieve Response.

Scenario 3: Obtain Patient Records Directly from IHE Enabled Vendor

3a) An XCA Query Request is sent directly to the IHE enabled responding location requesting patient records.

3b) The responding location sends an XCA Query Response to the requesting location.

4a) An XCA Retrieve Request is sent to obtain the records from the IHE responding location.

4b) The responding location sends the patient records in the XCA Retrieve Response.

Query and Obtain Patient Records Message Flow Without PLQ

does not maintain a Clinical Document Repository. Documents are retrieved from the source.

For systems that do not utilize PLQ, Surescripts is able to provide the detail of documents available from multiple sources using customized logic when processing an XCA Query request and response.  This customized version allows systems to submit an XCA Query to Surescripts using the Surescripts assigned patient identifier returned in the XCPD response and receive information on documents available from the following sources:

  • Patient records from IHE enabled vendors
  • Location Summary Document from Surescripts

Scenario: Obtain documents from any responder using a single XCA Query followed by XCA Retrieve(s)

Step 1 – XCA Query

1a) An XCA Query Request is sent to Surescripts.

1b) Surescripts requests metadata from all possible responders:

1b-1) An XCA Query is sent to available IHE responders.

1b-2) A search is issued for Location Summary data.

1c) Surescripts aggregates responses and builds an XCA Query Response containing the combined document IDs with metadata for all available documents.

1d) Surescripts sends the XCA Query Response to the requester.

Step 2 – XCA Retrieve

2a) The requesting location sends an XCA Retrieve Request for the desired documents.

It is recommended that each document be requested in a separate XCA Retrieve Request.  This avoids any issue with a slow responder delaying return of all requested documents.

2b) Surescripts requests documents from all requested responders:

2b-1) An XCA Retrieve is sent to IHE responders.

2b-2) A Location Summary is generated.

2c) Surescripts aggregates responses and builds an XCA Retrieve Response containing the requested documents.

2d) Surescripts sends the XCA Retrieve Response to the requester.

Documents from IHE Enabled Vendors may be retrieved directly using peer-to-peer connectivity, when available.
