Launching Accelerator with Web Application API

Accelerator Launch Process

Provider vendors systems launch the Accelerator using the following steps:

  1. Provider vendor system sends an Auth Token Request through a single vendor aggregation server. The request includes the System User ID, System User Qualifier, End User ID, and roles.
    • The user’s roles areis . See Roles for more information.
    • The user could be the provider or someone working on behalf of the provider. All users will be working on behalf of one or more providers.
    • The user could be the prescriber or someone working on behalf of the prescriber. All users will be working on behalf of one or more prescribers.
    • The user will be the Care Manager. There could be multiple Care Managers working off the same worklist.
  2. Surescripts returns a Token in the response. The response includes information on Token expiration.
  3. The provider vendor system launches the Accelerator using the Token. The launch includes styling parameters as well as launch options. See Accelerator Launch Options for more information on worklists and process lists.
  4. The provider vendor system closes the session.
