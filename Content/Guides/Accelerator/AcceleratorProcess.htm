Accelerator Integration Approach

Workflow Integration Approach

The Accelerator features support the ePA portions of provider vendor workflows. The provider vendor continues to be responsible for the remaining e-prescribing processes (e.g., creating a NewRx, communicating with the pharmacy, and managing users). The provider vendor determines how best to integrate the Accelerator features into their application.

Accelerator Process

The accelerator delivers a method for provider vendors to quickly offer customers core features. By embedding an iframe (or using a web browser control) in the provider vendor system, the accelerator will manage the process of completing an ePAof viewing and responding to medication management messages.

The accelerator is comprised of two APIs:

  • Web Application API - provides a customizable user interface
  • Web Service API - gives programmatic read-only access to the current state of workflow processes and tasks

The Web Application API provides a customizable interface that is displayed in an iframe (or a web browser control) in the provider vendor system. The interface interacts with the accelerator and its managed workflow. In order to use the Web Application API, the provider vendor system must first call the Web Service API to request and receive an authorization token.

Authorization tokens are issued for 60 minutes and contain the expiration date and time. A new authorization token is not required to relaunch the Accelerator if still active or before the expiration period has elapsed.

When an end user sends a PAInitiationRequest via the native CompletEPA® route, the accelerator workflow engine creates a workflow process. The workflow process is given an InitiatorsProcessID, which is the PAReferenceID from the PAInitiationRequest. The workflow engine tracks all subsequent messages related to the PAInitiationRequest and distributes workflow tasks related to the process.

When a PBM/payer sends Surescripts a medication management message, the accelerator workflow engine creates a workflow process. The workflow engine tracks all subsequent actions on the medication management message and distributes workflow tasks related to the process.

End users are able to interact with a worklist, which is the collection of active tasks that require action from the current user. The end user may complete the task via the user interface provided by the Web Application API. The Web Service API gives programmatic read-only access to the current state of workflow processes and tasks.

When an accelerator process reaches a point where the end user needs to take action on a task, e.g.; complete a question set, the end user can complete the task via the user interface provided by the Web Application API. Additionally, the Accelerator provides a historic view of in-process and completed ePA activities.
