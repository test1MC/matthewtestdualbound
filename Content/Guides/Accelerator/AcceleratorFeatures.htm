Accelerator Features

The Accelerator capabilities are highlighted below:

  • Enables users to view, answer, and submit the ePA Question Set — supporting all question logic.
  • Automatically populates of provider details, patient demographics, and medication information into PA messages.
  • Enables users to view process outcomes, including authorization details.
  • Provides worklist tracking to allow users to manage tasks requiring attention.
  • Enables users to review completed ePA cases.

    • Completed ePA cases are available indefinitely.

  • Provides automated appeal and cancel processes.
  • Provides role-based access controls to manage delegation of responsibilities among providers (prescribers and their staff).
  • Provides ability to view PDF attachments.
  • Provides ability to automatically search for an alternative PA processor, if the PBM/payer is not the processor.
  • Allows for PDR (Physician’s Desk Reference) integration with the ability to provide a PDR URL.
  • Ability to aggregate notes from multiple processors.
  • Allows users to view notifications on patients enrolled in the Patient Notifications program based on user-customized notification criteria.
  • Provides worklist tracking to allow users to manage notifications within a task requiring attention. For example, viewing the details of the medication record triggering the notification, removing the notification from the task list once worked, etc.
  • Allows user to view which patients need a Specialty Enrollment Form completed
  • Allows provider staff to select the appropriate Specialty Enrollment Form for each patient
  • Allows provider staff to complete the Specialty Enrollment Form they selected
  • Allows provider staff to submit the completed Specialty Enrollment Form to the specialty pharmacy

Prerequisites for Using the CompletEPA® Accelerator

Use of the Accelerator requires that the prescribing system also supports:

  • Eligibility request/response (X12 270 and 271 messages) for purposes of identifying PBM/Payer and member eligibility.
  • Formulary information in the e-prescribing process, including utilization of the PA coverage list (NCPDP Formulary & Benefits Standard) for purposes of prospectively identifying drugs that may require prior authorization.
  • Provider vendors utilizing the Directory in cases where the 271 does not indicate an active coverage, but the provider vendor still wants to initiate a prior authorization with a specific PBM/payer.
  • Beginning the ePA process by sending a PAInitiationRequest to Surescripts.
  • Ability to proxy HTTP requests for all Web Service API requests through a single aggregation server, including the auth token request. An auth token is required to use the Web Application API.
  • Provider vendor applications must handle download prompts for links to view PBM/payer response supported attachment types.

Prerequisites For Using The Specialty Patient Onboard Accelerator

Use of the Accelerator requires that the provider vendor system also supports:

  • Ability to send a NewRx electronically
  • Ability to proxy HTTP requests for all Web Service API requests through a single aggregation server, including the Auth Token Request. An auth token is required to use the Web Application API.

Prerequisites for Using the Accelerator

Use of the Accelerator requires that the provider vendor system does the following:

  • Initiates the Patient Notifications process by sending the Patient Enrollment File to Surescripts. This file must include patient demographics and the Organizational NPI, for more details see the Patient Enrollment File Load section.
  • Ability to proxy HTTP requests for all Web Service API requests through a single aggregation server, including the Auth Token Request. An auth token is required to use the Web Application API.

Browser Requirements

The Accelerator requires a web browser with JavaScript and session cookies enabled.

Surescripts expects, but does not guarantee, performance on supported and up-to-date web browser or client/server web browser control.

Please be aware that individual browser security settings may prevent the Accelerator from properly launching and should be evaluated based on each provider vendors’ implementation. The website surescripts.net may also need to be added as a “trusted site” for proper functioning.
