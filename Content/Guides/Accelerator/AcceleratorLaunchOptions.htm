Accelerator Launch Options

POST /uiservice/app/v0/launch/#/worklist

POST /uiservice/app/v0/launch/#/worklist

The worklist contains the user’s actionable tasks. The user may select a task and trigger the Accelerator to go to the appropriate view for that task. Tasks related to ePA are assigned to the recipient of the ePA Message, e.g. the SPI of the provider sent in the PAInitiationRequest. Tasks related to Medication Management for Adherence are optionally assigned to the recipient sent by the PBM/payer in the medication management message, e.g. the NPI of the provider sent in the PBM/payer medication management message. Tasks related to Patient Notifications are assigned to the recipient of the notification. Tasks related to SPO enrollment are assigned to the recipient of the SPO Message, e.g., the SPI of the provider sent in the SPOInitiationRequest.

The user viewing and working on a task does not have to be a providerprescriber, but the user must have the permissions for the task as described in Authentication and Authorization.

Launch by provider or by patient is controlled by the “provider” or “patient” section being included in the Auth Token Request. These roles are then encoded in the Auth Token and used when querying for tasks.

Product is a field in the auth token request as well. It is an array of objects that have an id and an array of processTypes. If nothing is sent for “products”, the user will see all products implemented for that launch context.

POST /uiservice/app/v0/launch/#/process

The process list shows the status of the SPO enrollment and its associated tasks. This endpoint launches a view showing a list of workflow processes matching the search filter criteria. The user may drill down into each process to see the state of all tasks associated with the process.

The process list shows the status of the ePA and its associated tasks. This endpoint launches a view showing a list of workflow processes matching the search filter criteria. The user may drill down into each process to see the state of all tasks associated with the process.

POST /uiservice/app/v0/launch/#/process/[InitiatorsProcessId]

For an SPO enrollment process, the InitiatorsProcessId is the SPOReferenceID assigned by the provider vendor system in the SPOInitiationRequest and is associated with all subsequent messages. This endpoint launches a view that shows the state of the workflow process including the status of the process and context specific details.

For an ePA process, the initiator_process_id is the PAReferenceID assigned by the provider vendor system in the PAInitiationRequest. This value is assigned by the provider vendor system and is associated with all subsequent messages. This endpoint launches a view that shows the state of the workflow process including the status of the process and context specific details.

POST /uiservice/app/v0/launch/#/worklist/[taskid]

For an SPO enrollment process, the taskId is a Surescripts defined value that is assigned to a specific step within an SPO enrollment process. A new taskId is generated when the status of a task is updated or a new task type (represented by taskName) is created within an SPO enrollment process. The endpoint will launch a user to the task-specific view within the process.

If a task’s most recent status is ‘Complete’ the user should not be allowed to launch to that specific taskId. Launching to a completed task will ultimately result in an error at the end.

To obtain taskID for purposes of launching Accelerator by taskID, use “Retrieve Process Detail” API endpoints. See Retrieve Process Detail for more details.

For an ePA process, the taskId is a Surescripts defined value that is assigned to a specific step within an ePA process. A new taskId is generated when the status of a task is updated or a new task type (represented by taskName) is created within an ePA process. This endpoint will launch a user to the task-specific view within the process.

If a task’s most recent status is ‘Complete’, the user should not be allowed to launch to that specific taskId. Launching to a completed task will ultimately result in an error at the end.

Some tasks may be associated with action that is required on the part of the PBM/payer and will not require interaction on the part of the user within the prescriber system. Launching by taskId should only be done when the status of a task is awaiting action from the user who initiated the PA (or their proxy). TaskNames that may require action on part of the prescriber (or their proxy) include: “Complete PA Form” and “Acknowledge” tasks (e.g. “Acknowledge Closed PA”).

To obtain taskId for purposes of launching Accelerator by taskId, use “Retrieve Process Detail” API endpoints. See Retrieve Process Detail for more details.
