Accelerator Integration

The Accelerator delivers a method for provider vendors to quickly offer customers core Specialty Patient Onboard Patient NotificationsMedication Management for Adherence ePA features. By embedding an iframe (or using a web browser control) in the provider vendor system, the Accelerator will manage the process of viewing and responding to Patient Notifications messagesmedication management messagesan EPASPO enrollment.

The Accelerator is comprised of two APIs:

  • Web Application API - provides a customizable user interface that is displayed in an iframe (or a web browser control) in the provider vendor system
  • Web Service API - gives programmatic read-only access to the current state of workflow processes and tasks

The Web Application API interacts with the Accelerator and its managed workflow. In order to use the Web Application API, the provider vendor system must first call the Web Service API to request and receive an authorization token.

Authorization tokens are issued for 60 minutes and contain the expiration date and time. A new authorization token is not required to relaunch the Accelerator if still active or before the expiration period has elapsed.

When an end user sends a SPOInitiationRequest the Accelerator workflow engine creates a workflow process. The workflow process is given an InitiatorsProcessID, which is the SPOReferenceID from the SPOInitiationRequest. The workflow engine tracks all subsequent messages related to the SPOInitiationRequest and distributes workflow tasks related to the process.

When an end user sends a PAInitiationRequest via the native CompletEPA® route, the Accelerator workflow engine creates a workflow process. The workflow process is given an InitiatorsProcessID, which is the PAReferenceID from the PAInitiationRequest. The workflow engine tracks all subsequent messages related to the PAInitiationRequest and distributes workflow tasks related to the process.

When a notification is generated, the Accelerator workflow engine creates a workflow process. The workflow engine tracks all subsequent actions on the notification and distributes workflow tasks related to the process.

When a PBM/payer sends Surescripts a medication management message, the accelerator workflow engine creates a workflow process. The workflow engine tracks all subsequent actions on the medication management message and distributes workflow tasks related to the process.

End users are able to interact with a worklist, which is the collection of active tasks that require action from the current user. They are also able to receive read-only information regarding the status of individual tasks or processes through the use of Web Service APIs.

When an Accelerator process reaches a point where the end user needs to take action on a task, e.g., complete a question set, the end user can complete the task via the user interface provided by the Web Application API. Additionally, the Accelerator provides a historic view of in-process and completed ePA activities.

When an Accelerator process reaches a point where the end user needs to take action on a task, (e.g., view a notification) the end user can complete the task via the user interface provided by the Web Application API.

When an Accelerator process reaches a point where the end user needs to take action on a task, e.g., Complete Specialty Enrollment, the end user can complete the task via the user interface provided by the Web Application API. Additionally, the Accelerator provides a historic view of in-process and completed SPO enrollment activities.
