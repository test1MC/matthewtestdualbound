Retrieve Worklist

Retrieve worklist returns the list of active tasks awaiting action based on the given filter criteria. The items returned by a worklist request either have the acceptedBy as the user in the worklist request or the acceptedBy is null.

Patient Worklist

Retrieve Worklist by Patient

Retrieving a worklist by patient retrieves a paged list of the active tasks for a given patient.

Request Template Example

[GET] {{host}}/worklist?alternateId={alternateId}&alternateOrganization={alternate
organization}&alternateQualifier={ alternatequalifier }&user={user}&pageIndex={pageIndex}&pageSize={pageSize}

[GET] {{host}}/worklist?patientId={patientId}&facility={facility}&user={user}&
pageIndex={pageIndex}&pageSize={pageSize}

Request Parameters

M/C – Mandatory or Conditional

Parameter

Data Type

M/C

Description

patientId

an..35

M

Matches the patientId specified in the PAInitiationRequest/Body/Patient/Identification/MedicalRecordIdentificationNumberEHR.

alternatetId

an..35

M

The patient PBM Unique Id returned from an eligibility 271.

alternateOrganization

an..35

M

The payer Id returned from an eligibility 271.

alternateQualifier

an..35

M

alternateQualifier: PBMUID

user

an..250

M

Unique provider vendor system user ID representing the user working the task list. This user is not required to be a clinician vendor who has a registered SPI.

facility

an..35

C

Optional qualifier the vendor may use to make their Patient ID unique across their installations. Facility is only used in conjunction with patientId.

pageSize

Positive Int

C

Integer representing the number of records returned in each request. If a pageSize is not specified, the default value of 25 is applied.

pageIndex

Positive Int

C

Integer representing the page index of data to be returned. If a pageIndex is not specified, the default value of 0 is applied.

Request Examples

[GET] {{host}}/uiservice/api/v0/worklist?alternateId=PBM123456&alternateOrganization=D44444444444444&alternateQualifier=PBMUID&user=jake

[GET] {{host}/uiservice/api/v0/worklist?patientId=george&facility=clinic1&user=jake

[GET] {{host}/uiservice/api/v0/worklist?patientId=george&user=jake

Get Worklist by Recipients

Retrieves a paged list of the active tasks for a given collection of recipients (providers).

Request Template Example

[POST] {{host}}/worklist

Request JSON Fields

M/C – Mandatory or Conditional

JSON Field

Data Type

M/C

Description

user

an..250

M

Unique provider vendor system user ID representing the user working the task list. This user is not required to be a clinician vendor who has a registered SPI.

recipients

List

M

Collection of recipients for which Accelerator based tasks will be retrieved.

id

an13

M

SPI of the initiator.

qualifier

an1

M

SPI's corresponding qualifier.

Qualifier = DZZ

pageSize

Positive Int

C

Integer representing the number of records returned in each request. If a pageSize is not specified, the default value of 25 is applied. This value is truncated to 100 if larger.

pageIndex

Positive Int

C

Integer representing the page index of data to be returned. If a pageIndex is not specified, the default value of 0 is applied.

Request Example

[POST] https://[host]/uiservice/api/v0/worklist

{

   "user": "jake",

   "recipients": [

       {

           "id": "6417473720001"

           "qualifier": "DZZ",

       }

   ]

}
