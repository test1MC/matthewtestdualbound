ADT Messages Overview

Overview

The healthcare facility (ADT system) sends Surescripts an ADT HL7 message when any of the following events occur at their facility:

  • ADT^A01 - Hospital Admission/Visit Notification
  • ADT^A03 - Patient Discharge
  • ADT^A04 - Register a Patient
  • ADT^A08 - Update Patient Information
  • ADT^A28 - Add Person Information
  • ADT^A31 - Update Person Information
  • ADT^A37 - Unlink
  • ADT^A40 - Merge

Surescripts will send an acknowledgment (ACK) or ACK with an error segment, if errors are present, to the customer notifying them that the message was received.

Minimum data points needed for the message to be actionable:

  • Event type
  • Home Community ID of the sending location
  • Assign Authority
  • Receiving Application
  • Patient ID
  • Patient First Name
  • Patient Last Name
  • Patient Zip Code
  • Patient Date of Birth

Completing By filling out as many fields as possible allowsit will help downstream users to perform the correct actions to improve the patient care plan.

ADT Message Data Flow

Figure #1 ADT Message Data Flow Diagram

The following steps depict the ADT message data flow:

  1. ADT HL7 message is initiated by an ADT System and received by Surescripts.
  2. Surescripts returns an ACK to the ADT system that sent the ADT message.

The ADT system receives the acknowledgment. If no acknowledgment is received, or a NAK is received, the ADT message was not accepted.

  1. Depending on the ADT message type sent, the following action will occur:
    • Add/Update Patient: ADT^A01, ADT^A04, ADT^A08, ADT^A28, and ADT^A31: If the patient sent in the message does not already exist in the MPI, Surescripts will update it with the new information.
    • Unlink Patient: ADT^A37: When Surescripts receives this message type and matches the patient information to the MPI, Surescripts will then expire the patient record so that the record is no longer returned in MPI searches.
    • Merging Patient IDs: ADT^A40: Surescripts will update the MPI with the patient information found in segment PID-3. The patient information found in the MRG segment will be expired.

ADT Message Error Processing

In instances where Surescripts or a customer receives a message that is unrecognizable, Surescripts will send back an XML formatted NAK.

If an error has occurred after Surescripts has determined that this is an HL7 message, the recipient will respond with an HL7 ACK that indicates an error.
