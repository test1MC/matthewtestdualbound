General Requirements

Message Format

General Message Requirements

E-Prescribing-only data sources are Surescripts approved sources for medication description information as defined in the item description data element of the NCPDP standard and for drug identifiers (e.g., NDCs, RxCUIs) for the purposes of Surescripts E-Prescribing. These data sources do not currently contain all of the required data attributes that are required for the Surescripts Eligibility and Formulary services.

For data sources not on the Surescripts approved list, there is a formal process in place to review and approve new data sources.

Prescribers and pharmacies must have the appropriate service levels enabled to send or receive messages. See the Surescripts Directories Implementation Guide for more information.

E-Prescribing Best Practices

The following general best practices apply to all E-Prescribing message types:

  • The application should make clear to the user which data elements will be sent to the receiver and which elements are for internal/local use.
  • Customers should default to electronically send messages within their application workflow when the receiver is electronically enabled.
  • The application should not duplicate messages; e.g., electronically delivered as well as faxed.
  • The application should be able to send all data elements/segments which can help the receiver better interpret the e-prescription message and make a more informed decision based on their scope of practice.
  • Notes and other free text fields should not contain information for which there is a designated field.
  • When using a codified field, ensure the code captures the intent and semantically matches the associated/related text field.

As a reminder, Surescripts conducts certification with prescriber and pharmacy systems to ensure the application adheres to network requirements. While we are able to enforce application adherence, we are unable to mandate specific behaviors or workflows be carried out by its users.

Message Validation

EPCS Message Validation Requirements

The EPCS flow for fillable messages follows the flow shown below with these additions:

  • The "ControlledSubstance" service level for both the prescriber and pharmacy must be enabled for controlled substances or the message will be rejected.

Pharmacies may send controlled substance requests without the "ControlledSubstance" service level enabled; however, to receive an affirmative response it must be enabled. Prescribers without a "ControlledSubstance" service level will be able to receive the request, but will only be able to send a Denied response.

  • The message must be digitally signed and contain either the signature or the indicator that it has been signed. The sender of the message is required to sign all fields in the digital signature. Surescripts will not validate the signature.
  • The message must contain the NDC of any controlled substance.
  • The DEASchedule of any controlled substance must be sent.
  • The Patient address must be sent.
  • The Prescriber address must be sent.

See Element Detail Tables and Element Usage for more details.

How Surescripts determines when to apply EPCS processing to a message

State Specific Validation for Controlled Substances

The reference of “state” throughout this section includes U.S. Territories and foreign U.S. military bases.

Surescripts will stop controlled substance messages from being sent or received that are prohibited by state law as outlined below. Prescriber systems and their users are still ultimately responsible for continued compliance with all applicable laws and requirements, including but not limited to, local and state laws and regulations in which the customer’s application is deployed and used.

Each message without a DEA Schedule will have the NDC checked to determine if the medication is a controlled substance at a federal or state level. Once a medication has been identified as a controlled substance, either federally or by the state it is written in or being sent to, the following validation checks occur:

  • Apply the most restrictive DEA Schedule, federal or state, for EPCS processing.
  • Validate that the state the prescription is written in/sent to allows e-Prescribing of controlled substances for the specific schedule of the medication in the message.
  • For mail order pharmacies, Surescripts will not check the NDC against the location of the mail order pharmacy; however, the prescriber state is still validated. Mail order pharmacies are still required to perform the appropriate state validations.

Display Requirements

Surescripts understands our customers are in the best position to make workflow design decisions and we are committed to encouraging innovative designs. Surescripts is also committed to ensuring the highest level of patient safety possible. If a workflow could potentially result in a patient safety issue, or if a patient safety issue is identified, Surescripts may require application code changes to ensure positive patient outcomes.

To allow for patient choice, the application shall allow a pharmacy search based on pharmacy type of Mail/Retail/InHouseDispensing at a minimum; other pharmacy search options may include specialty and long term care.

The application shall not filter out pharmacies of a specific type when a search by geography (city, state, and zip code) is performed, unless a specific pharmacy type is requested. The application shall display the following pharmacy information in the pharmacy search results: Pharmacy Name, Street Address, City, State, Zip Code, Phone Number, and Pharmacy Type.

Display Best Practices

Pharmacy system vendors should ensure that, when needed, Certificate to Prescribe (CTP) numbers always appear on primary electronic prescription screens for pharmacists to view during the dispensing process. The letters "CTP" should precede the number so pharmacists can easily identify it. Pharmacy system vendors should also ensure that CTP numbers appear on hard copies of electronic prescriptions when they are printed out.
