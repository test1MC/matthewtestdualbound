Message Linkage

Customers must follow all message linkage requirements and best practices as described in the table below. Four key fields are used to link messages together, as defined below. This linkage allows a software system to tie one message to another. For example, on an RxRenewalRequest the prescribing system needs to find the original NewRx to which the pharmacy is referring. This linkage should be done in an automated manner.

NCPDP refers to message linkage as trace number.

MessageID – The unique number assigned to each message. This ID initiates the electronic linking of subsequent message types.

RelatesToMessageID – The MessageID of the message being responded to.

RxReferenceNumber – The prescription number assigned by pharmacy system. This number must be populated on all pharmacy initiated messages.

PrescriberOrderNumber – The prescription number created by the prescribing system. PrescriberOrderNumber must be populated on all prescriber initiated messages.

Recommended Matching

The PrescriberOrderNumber or the RxReferenceNumber should be used as the primary matching criteria for matching messages. If it cannot be used, the following data in the message should be used:

  • RelatesToMessageID
  • Patient demographics
  • Medication information
  • Prescriber information
  • Pharmacy information

Identification

C = RxFill and Census

M = All other messages for prescriber

B

  DEANumber

For prescriber initiated messages for controlled substances at least one occurrence must contain the DEA number.

Each prescriber of controlled substances will need to have a unique DEA or Institutional DEA Number registered/assigned to them in the Surescripts Directory and the EPCS service level enabled.

The exception to this rule is a Public Health prescriber can use their Social Security Number in place of a DEA number. Military personnel are covered under Public Health prescribers. Military personnel are also required as part of the DEA audit to report their branch of service in the Note field.

Only one DEA number for the location where the prescriber is located should be sent.

The DEA number format in the Prescriber and Supervisor elements are validated. The first character must be alpha, and the second character can be alpha or 9 followed by 7 numbers. If there are more than 9 characters, the 10th must be a '-' (hyphen). The remaining characters are not validated.

C

  NPI

C = RxFill

M = All other messages for prescriber

M = All other messages for supervisor

For prescriber initiated messages (NewRx, RxRenewalResponse, RxChangeResponse, CancelRx, Resupply, and Census) at least one occurrence must contain the individual (not organizational) NPI of the prescriber and supervisor if the supervisor loop is sent.

The NPI check digit will be validated using the LUHN formula. (Applies to both Prescriber and Supervisor Loops.) For specific information see https://www.cms.gov/Regulations-and-Guidance/Administrative-Simplification/NationalProvIdentStand/Downloads/NPIcheckdigit.pdf.

C

CertificateToPrescribe

The CTP is a field that may be required for prescribing in certain states.

Due to translation requirements, when sending to a lower version, the CTP number should also be sent in the StateLicenseNumber field and should be preceded with CTP.

C

Data2000WaiverID

Prescriber NADEAN number should be sent in this field when required. The NADEAN number will be validated like other DEA numbers. Due to translation requirements for EPCS, when sending to lower version users, the NADEAN should also be sent in the Note field, “NADEAN:xxxxxxxxx” by the sending system, or it will not be transmitted to the pharmacy.

For Specialty of prescriber values, go to http://www.wpc-edi.com/reference/ and click on Health Care Provider Taxonomy Code.

For EPCS-fillable messages when sending the social security number for military personnel, this field must be populated with the MPH specialty type code of "2083P0901X", "171000000X", "1710I1002X", or "1710I1003X".

  ProductCode

Product Identifier.

The Representative NDC is required unless the CompoundInformation element is present or the message has a ProductCode Qualifier of 'UP' or 'DI'.

The NDC must be a valid 11 digit code.

For transmitting Supply items, the ProductCode Qualifier of "UP" should be used when the item does not have an NDC. If there is no UPC available, the text "Supply" should be placed in the ProductCode/Code field with the ProductCode Qualifier "UP". Customers are expected to work with the drug database vendor to obtain the necessary product identifiers.

In NCPDP SCRIPT, the NDC is not required.
