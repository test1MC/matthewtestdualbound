Detailed Error Message Scenarios

RTF/RTPB Error Message Scenarios

The following diagram depicts various scenarios where Error messages are sent in response to a Benefit Request that used the RTF or RTPB workflow.

Scenario 1: Surescripts Cannot Recognize the Message. Surescripts returns an error message.

1a) A provider vendor sends a Benefit Request to Surescripts.

1b) Surescripts cannot recognize the message (e.g. improperly formed XML) and sends an error back to the provider vendor.

Scenario 2: Surescripts Finds Message Errors in Benefit Request. Surescripts returns an ERROR message.

2a) A provider vendor sends a Benefit Request to Surescripts.

2b) Surescripts recognizes the format but finds errors (e.g. schema validation error) in the message. Surescripts returns an ERROR message.

RTF Error Message Scenario

The following diagram depicts a scenario where a Benefit Response message that includes business errors is sent in response to a Benefit Request message that used the RTF workflow.

Scenario 1: Surescripts Finds Business Errors in Benefit Request. Surescripts returns a Benefit Response message with Business Errors.

1a) A provider vendor sends a Benefit Request to Surescripts.

1b) During business processing, Surescripts finds business errors (e.g. Unknown Formulary ID, etc.) and returns a Benefit Response message to the provider vendor.

RTPB Error Message Scenarios

The following diagram depicts various scenarios where an ERROR message or Benefit Response message with business errors is sent in response to a Benefit Request message that used the RTPB workflow.

Scenario 1: PBM/payer finds Message Errors in Benefit Request. PBM/payer returns an ERROR message.

1a) A provider vendor sends a Benefit Request message to Surescripts.

1b) Surescripts forwards the Benefit Request to the PBM/payer.

1c) During validation, the PBM/payer finds message errors and sends an ERROR message to Surescripts.

1d) Surescripts forwards the ERROR message to the provider vendor.

Scenario 2: PBM/payer finds Business Errors in Benefit Request. PBM/payer returns a Benefit Response message with Business Errors.

2a) A provider vendor sends a Benefit Request to Surescripts.

2b) Surescripts forwards the Benefit Request to the PBM/payer.

2c) During business processing, the PBM/payer finds business errors (e.g. Patient Not Found, Drug Not Found, etc.) and returns a Benefit Response message to Surescripts.

2d) Surescripts forwards the Benefit Response to the provider vendor.
