  • This product previously used the acronyms RFS for Real-Time Formulary (now RTF) and PMBC for Real-Time Prescription Benefit (now RTPB). The old acronyms are still in place within the schema file name, schema, and XML message examples in this Guide and will not be updated to avoid versioning issues.

About Surescripts Eligibility and Formulary: Real-Time Formulary

Surescripts Eligibility and Formulary: Real-Time Formulary delivers plan level, formulary information to the prescriber at the point of care via a real-time Request and Response message. This service eliminates the need for the provider vendor to download large formulary files from Surescripts on a weekly basis. The data provided through RTF informs the prescriber of a medication's formulary status, payer-provided alternatives, coverage factors, and copay details. RTF data is intended to assist the prescriber in aligning their selected therapy with the patient’s benefit plan.

Notes:

  • RTF utilizes NCPDP Formulary and Benefit Standard 3.0. Provider vendors currently using Formulary 1.0 will need to make the required changes to handle Formulary 3.0-based RTF data.
  • PBM/payers are providing formulary files at the NDC-11 packaged medication level. All provider vendors prescribe at the dispensable medication level (medication name, form, and strength). To return formulary correctly on the Benefit Response, Surescripts rolls up and stores the existing formulary files at a dispensable medication level by selecting a representative NDC from an industry standard drug database.

About Surescripts Real-Time Prescription Benefit

Surescripts Real-Time Prescription Benefit is a real-time Request and Response message between the provider vendor and PBM. The RTPB message delivers patient-specific medication cost and benefit information to the point of care to help inform prescribers' prescription decisions. This includes point-in-time estimated cost, alternative medication options, pharmacy type options, and formulary information. RTPB data is referenced prior to the NewRx to inform the prescriber and the patient of the specific estimated out-of-pocket cost for the selected therapy. RTPB is provided in addition to, and not a replacement of, the group/plan level formulary information.

RTF and RTPB provide value by driving medication adherence, lowering health care costs, and improving health outcomes and patient safety. By providing accurate, up-to-date, patient-specific information and cost-effective medication options, prescribers and patients can make informed decisions that reduce payment obstacles and improve medication adherence.

Distinction Between RTF Data and RTPB

RTF and RTPB data both provide formulary, coverage, and estimated copay related information at two different levels. The table below outlines the differences:

RTF Data RTPB Data

Obtained through:

  • Sending a Benefit Request (RTF workflow) for a selected medication or group of medications.
  • Receiving a Benefit Response (RTF workflow) from Surescripts that includes representative formulary data.

Obtained through:

  • Sending a Benefit Request (RTPB workflow) for a selected medication via Surescripts to the patient's PBM using the patient's PBM Unique Member ID.
  • Receiving a patient specific Benefit Response (RTPB workflow) created by the PBM.

Plan/Group level

  • Plan level formulary data. Including: formulary status, payer provided alternatives, coverage factors, and copay data.

Patient level:

  • Patient level, point-in-time estimated patient out of pocket costs for a selected medication and alternatives.

Utilized to inform the initial medication selection, prior to the identification of the preferred pharmacy and Sig. Assists the prescriber in selecting the best option based upon plan/group-level formulary data. Commonly used to inform the following:

  • Formulary status on active medications
  • Formulary status on physician favorites
  • Formulary status on medications found in a search

Utilized after the Sig and preferred pharmacy have been populated to inform the prescriber of the estimated out of pocket costs and alternative medication options. Commonly used to inform the following:

  • Active medication list
  • Refill Request/Response
  • New prescription

RTF/RTPB Workflow

The Benefit Request and Benefit Response messages used by the RTF/RTPB product allow for separate workflow options, RTF or RTPB. A provider vendor can choose to implement one or both workflows. Each option is assigned a unique URL endpoint that designates which workflow is used.

About This Guide

The Surescripts Eligibility and Formulary: Real-Time Formulary/Real-Time Prescription Benefit Implementation Guide was created to assist provider vendors in implementing Benefit Request/Response messages that will use one or both of the RTF and RTPB workflow(s).

The audience includes any customer responsible for configuring or developing a system interface for the electronic messages.
