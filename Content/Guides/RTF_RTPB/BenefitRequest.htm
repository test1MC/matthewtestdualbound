Benefit Request Data Element Requirements

This Guide only includes data elements where Surescripts has specific requirements or further explains the field usage. Refer to the RFS PMBC XML Schema listed in Document References for a complete list of fields.

The Benefit Request for a patient will be tied to the Eligibility Response received within the last 72 hours for that patient. This is accomplished by using the Interchange Control Number ISA13 from the 271 Eligibility Response in the Relates To Message ID of the Benefit Request.

M

MedicationRequested
Information

Both

For the RTF workflow, the requester can either submit a single medication with multiple alternatives or multiple medications with no alternatives. Multiple medication requests must be limited to a maximum of 20 medications. Subsequent RTF requests could be submitted after the initial multiple medication request to bring back alternatives.

 

For the RTPB workflow, the requester may only submit a single medication.

C

AlternativeMedication
Information

RTF

Provider vendors can send the initial medication and any Alternatives that are appropriate for the requested therapy.

Alternative Medication NDCs will need to be supplied by the provider vendor. Only the first loop of the requested medication can contain alternatives.

A Benefit Request should be limited to a maximum of 500 therapeutic alternatives. If the number of therapeutic alternatives on a Benefit Request exceeds 500, then an additional Benefit Request with the remaining alternatives should be sent.
