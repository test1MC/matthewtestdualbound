Message Header

The message header follows the NCPDP SCRIPT standard version 2013101.

Message Header XML Diagram

In XML diagrams, blocks that are solid are mandatory, whereas blocks that have dotted lines are conditional.

Message Header Elements

Requirement Designation

Element Attributes:

Code

Description

M

Required/Mandatory - the element must be used per the specification (e.g., XML schema validation).

C

Situational/Conditional - the element must be used if conditions are met. Some fields do not have specific conditions. Data should be sent if available.

D

Dependent - the element is required or conditional based on the message type.

This Guide only includes data elements where Surescripts has specific requirements or further explains the field usage. Refer to the RFS PMBC PMBC XML Schema listed in Document References for a complete list of fields.

 

M/C

Element

Comment

M

To

The identification of the intended receiver of the message. This contains the Surescripts-assigned Participant ID from the Eligibility Response.

Contains the Surescripts ID.

Note: All response messages and free standing messages must be sent to the same sender that initiated the original message.

C To/@Qualifier

Value:

ZZZ = Mutually defined

M

From

The identification of the sender of the message. This is the Surescripts-assigned Participant ID.
C From/@Qualifier

Value:

ZZZ = Mutually defined

M

MessageID

A minimum set of standards/algorithms should be used when generating message IDs to ensure uniqueness. If possible, customers should utilize Global Unique Identifiers (GUIDs).

Note: The Message ID in the Benefit PMBC Response shall be populated in the Relates To Message ID field of the NewRx. The exception case is when a Denied New to Follow (DNTF) Refill Response was sent. If sent, populate the Relates To Message ID field with the Message ID of the DNTF Refill Response.

SeeMessage ID LinkageMessage Linkage for more details.

C

RelatesToMessageID

Relates to Message ID. This element is case sensitive.

PMBCRequest – Populate with value contained in the related Interchange Control Number ISA13 from the 271 Eligibility Response.

PMBCResponse – Populate with the Message ID from the PMBC Request.

Benefit Request (RTF & RTPB) – Populate with value contained in the related Interchange Control Number ISA13 field from the 271 Eligibility Response.

Benefit Response (RTF & RTPB) – Populate with the Message ID from the Benefit Request.

See Message ID LinkageMessage Linkage for more details.

M

SentTime

Date/time of initiation.

Refer to UTC Time Format UTC Time Format for information on how to properly format the date/time data.

C

Security/Sender/TertiaryIdentification

Surescripts populates the Account ID associated with the sender.

C

Security/Receiver/TertiaryIdentification

Surescripts populates the Account ID associated with the receiver.

M

SenderSoftware

Identifies the entity responsible that generated the message. Contains the software vendor name, product, and version / release.

M

SenderSoftwareDeveloper

Identifies the entity responsible for the software that generated the message. The developer may be a software vendor or, if the software was developed “in-house”, the developer is the entity actually sending the message (e.g., a chain). The value transmitted is determined by the Sender Software Developer.

M

SenderSoftwareProduct

The text field that identifies the software developer’s product, as determined by the software developer.

M

SenderSoftwareVersionRelease

The text field that identifies the version and release of the software product, as determined by the software developer.

C

TestMessage

When sent, allowable values must match platform that message is being sent to. If this field is not sent, then the default is production.

Values:

1 = Test

All other values = Live (Production)

 

PMBC Request Message Header Example

    <transport:To Qualifier="ZZZ">T00000000001001</transport:To>

    <transport:From Qualifier="ZZZ">T00000000001025</transport:From>

    <transport:MessageID>1213214</transport:MessageID>

    <transport:RelatesToMessageID>876543211</transport:RelatesToMessageID>

    <transport:SentTime>2016-10-25T19:00:33.2974466Z</transport:SentTime>

    <transport:Security>

      <transport:Sender>

      <transport:TertiaryIdentification>7025</transport:TertiaryIdentification>

      </transport:Sender>

      <transport:Receiver>

      <transport:TertiaryIdentification>21025</transport:TertiaryIdentification>

      </transport:Receiver>

    </transport:Security>

    <transport:SenderSoftware>

      <transport:SenderSoftwareDeveloper>EMRCOMPANY</transport:SenderSoftwareDeveloper>

      <transport:SenderSoftwareProduct>EMRSYSTEM</transport:SenderSoftwareProduct>

      <transport:SenderSoftwareVersionRelease>V1</transport:SenderSoftwareVersionRelease>

    </transport:SenderSoftware>

    <transport:TestMessage>0</transport:TestMessage>

  </pmbc:Header>

PMBC Response Message Header

  <pmbc:Header>

    <transport:To Qualifier="ZZZ">T00000000001025</transport:To>

    <transport:From Qualifier="ZZZ">T00000000001001</transport:From>

    <transport:MessageID>00000123</transport:MessageID>

    <transport:RelatesToMessageID>1231214</transport:RelatesToMessageID>

    <transport:SentTime>2016-12-01T12:43:39Z</transport:SentTime>

    <transport:Security>

      <transport:Sender>

        <transport:TertiaryIdentification>21025</transport:TertiaryIdentification>

      </transport:Sender>

      <transport:Receiver>

        <transport:TertiaryIdentification>7025</transport:TertiaryIdentification>

      </transport:Receiver>

    </transport:Security>

    <transport:SenderSoftware>

      <transport:SenderSoftwareDeveloper>PBMCOMPANY</transport:SenderSoftwareDeveloper>

      <transport:SenderSoftwareProduct>PBMSYSTEM</transport:SenderSoftwareProduct>

      <transport:SenderSoftwareVersionRelease>1.0</transport:SenderSoftwareVersionRelease>

    </transport:SenderSoftware>

    <transport:TestMessage>0</transport:TestMessage>

  </pmbc:Header>
