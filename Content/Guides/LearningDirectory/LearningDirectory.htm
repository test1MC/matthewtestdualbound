Learning Directory

The Learning Directory utilizes address demographic information content from within the NewRx message sent from the provider vendor. It appends previously unknown provider locations to the Surescripts directory and supplements the primary SPI registered (SPI-REG) in the directory.

Provider vendors currently managing their provider locations will continue to do so. Provider vendors choosing to use the Learning Directory must identify a primary SPI-REG location. Depending on the vendor registration model, there may be multiple "primary" SPI-REGs. For example, when a provider works at two separate institutions that both use the same software vendor, each institution would have a "primary" SPI-REG). Subsequent locations meeting the Learning Directory criteria will be learned and included in the Directory as Learned Locations (SPI-LL).

Pharmacies will be able to view SPI-LL records in the Admin Console and will see the SPI-LL appended to their provider full and nightly downloads. SPI-LL will use the same 13-digit SPI as the primary provider location registered (SPI-REG) by the provider vendor, but will have an alternate address. The SPI is no longer a unique provider identifier. As a result, pharmacies should be able to match the SPI to multiple records within their database. Development changes to accommodate the learned provider locations within their provider full and nightly export may be required.

Criteria for Learning New Provider Locations

  1. Learning Directory functionality must be enabled on the portal configuration for the provider vendor.
  2. The 13-digit SPI and NPI registered in the directory must match the 13-digit SPI and NPI in the successfully processed NewRx.
  3. The 13-digit SPI registered in the directory must contain the Refill Renewal service level. With the exception of controlled substances, other certified services may appear against the learned locations.
  4. A new location is identified for being learned if the address in the NewRx is determined to be unique against the existing directory registration. Key fields such as Practice, AddressLine1, AddressLine2 and Zip Code may be used to determine whether an address is unique. Similar validations that prevent duplication of an SPI with refills at the same physical location will be enforced.
  5. Learned locations will expire after a period of NewRx inactivity at the location or when the registered SPI expires, is changed to another vendor, or is disabled.

How Downloads Will Be Different

The 13-digit SPI is not a unique identifier. The download format does not change; it will include all applicable results. A pharmacy account must have the appropriate "Directory Download Special Rules" in place on their account settings to publish the learned locations to their download files. An "LL" will be included in the Mutually Defined field in the download to indicate the record is a "learned record".

Learned locations for provider vendors will only appear in version 4.6 downloads. A pharmacy vendor will see the learned locations in version 4.4, 4.5 and 4.6 downloads.

Learning Directory Registration

In the example below, the diagram displays how Dr. Bob works in six locations: two under Practice Group 1 and four under Practice Group 2. In Practice Group 1, he uses two different applications (a primary application "A" and a downloaded mobile application "M" which sends only new prescriptions). In Practice Group 2, the provider vendor application "B" explicitly registers Dr. Bob at the primary location of practice. Then, while writing new prescriptions from a different location where he is seeing patients, the subsequent location(s), meeting all of the criteria for being "learned", are added as a "learned location" to the registered SPI-005.

Without the Learning Directory, each location/application for a provider is registered creating a unique SPI. With the Learning Directory, one location is explicitly registered as the provider’s primary location of practice. As new locations are "learned" from new prescriptions, these "learned records" are associated with the registered SPI.

The Importance of Record Consistency

It is important that the provider vendors ensure the formatting and style of the SPI directory listings are consistent with the NewRx content. Alignment of these pieces of information helps ensure the respective databases used across the workflow are in sync. The following table offers recommendations on how to approach key fields within the registration and NewRx content. Please refer to Document References for USPS standardization information.

"Standardization" formatting offers a good foundation. If standardization formatting must be modified to address specific vendor database needs, ensure the formatting is consistent across the respective databases.

Surescripts stresses consistency in the following fields across all platforms used by the customer source information as a best practice to optimize the e-prescribing workflow.

Field Context Suggestions
Practice Name (P) This field can impact the standardization of a record, displaying a specific ZIP9 based on the sub-location of entity (much like suite designation). Inclusion (or omission) can impact standardization.
Address Line 1 (AD1) If connections to other systems, (for example, Billing) restricts the ability to include suites or location identifiers in AD2, Surescripts recommends adding that information to AD1 consistently.

Use abbreviations, such as "ST" for Street, "AVE" for Avenue, "S" for South and "NW" for Northwest. Avoid use of "." (periods) in the addresses.
Address Line 2 (AD2) Reserved for externally identifiable clarifications, such as Suite, Apartment, Building, and Floor designations.

Use abbreviations, such as "STE" for Suite, "BLDG" for Building, "APT" for Apartment", and "FL" for Floor.

Exclude internal-only identifiers such as "Exam Room #" or "Nurses Station".
City (CITY) City-based searches impacted by formats. Use common abbreviations when part of an official city name: "ST" for Saint (e.g. St. Paul), "FT" for Fort (e.g. FT Collins). See this link for USPS products list including "City State Product": http://pe.usps.gov/text/pub28/welcome.htm
State (ST)   Two character state/US Territory abbreviation is required.
ZIP (ZIP)

Including a 9-digit Zip Code (ZIP+4) offers an excellent level of CASS-standardization detail. These address attributes impact the "+4" of the Zip code:

  • Practice Name (P)
  • AddressLine1 (AD1)
  • AddressLine2 (AD2)
 If using the verified ZIP+4 USPS that aligns to your practice/address information, include it. If not verified, use the basic 5-digit ZIP code (ZIP5).
Primary Phone Pharmacies have indicated this is an important field to include - especially to ensure that the phone is correct for the specific practice. Do not include provider mobile/cell numbers; they are not unique to one practice group.
