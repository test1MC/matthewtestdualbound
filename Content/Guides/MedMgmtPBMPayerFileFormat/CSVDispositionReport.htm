CSV Disposition Report

The CSV Disposition Report is created by Surescripts to report the disposition of the PBM/payer messages.  This file contains the actions taken on a message by provider vendor end users. In the diagram, below this file is created in Step 5 and the PBM/payer retrieves the report in Step 6.

Message Flow Disposition Report

Surescripts SFTP Folder

from_surescripts

Timing

This file will be produced on an agreed upon schedule with the PBM/payer vendor.  Timing depends on when the PBM/payer messages expire.   For example if the PBM/payer vendor chooses the default expiration of 30 days, this file will be produced 1-2 weeks after the messages have expired.

  • One Disposition Report will be generated for each PBM/payer vendor payerId
  • The Disposition Report will include any messages created by the PBM/payer vendor payer id where the message is either past the expiry date or a provider has submitted feedback
