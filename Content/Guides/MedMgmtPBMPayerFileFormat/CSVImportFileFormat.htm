CSV Import File Format

The CSV (comma-separated values) file is created by the PBM/payer vendor and contains medication management, Medication Adherence messages, and Proportion of Days Covered (PDC).  In the diagram below, this file is created in Step 1.

Message Flow CSV File

Name

Description

Timing

The PBM/payer vendor will determine the timing of sending the import file to Surescripts.

SFTP Folder

to_surescripts

File Name

MmpocFhir_{PBM Payer ID}_{YYYYMMDD}_{HHMM}.csv

Example: MmpocFhir_T00000000022247_20160513_1320.csv

File Format

CSV

R = A value is required

O = A value is optional

N/A = No value

ELEMENT

OPTIMAL

SUB

OPTIMAL

COMMENT

MESSAGE_ID

R

R

Unique Id Generated by Sender.
This is required to be unique.

Example: 4028a298468b835a01468bc7e9001

Note: Message senders should retain the Message Identifier for troubleshooting purposes, Surescripts will return this Message Identifier in the disposition response

PRACTITIONER_NPI_ID

R

R

Practitioner identifier, the practitioner's NPI.

The NPI from the PBM/payer system that is associated to the message.

Example: 6833179206

MBR_ID

R

R

PBM Unique Member Id. Carrier Account Group + Member Id
The external reference Id that is sent from the PBM/payer in the Eligibility 271 response (Subscriber/NM109).

Example: AB123 DE456 9020040 QR40210

Note: The member is uniquely identified by the provider vendor by referencing the Payer Id and Member Identifier (PBM Unique Member Id).

MBR_DATE_OF_BIRTH
MM/DD/YYYY

R

R

Member Date of Birth.  (MM/DD/YYYY)

Example:  04/14/1974

This is the DOB supplied by the PBM/payer and is provided to the provider vendor to better assist in ensuring the proper member has been matched between the provider vendor and the PBM/payer message

MBR_FIRST_NAME

R

R

This is the name supplied by the PBM/payer and is provided to the provider vendor. It is necessary to assure the proper member has been matched between the provider vendor and the PBM/payer message.

Member Name is displayed in the UI to give the practitioner the ability to confirm that it's the right person

R

Indicates the potential degree of impact of the identified issue on the member.

Validation Errors

The line numbers listed below are example line numbers. Line numbers will vary based on the validation error.

Required Field Validation Errors

Required Field

Required Field Validation Error

MESSAGE_ID

Record at line number: 2 must supply a message ID

PRACTITIONER_NPI_ID

Record at line number: 3 must supply a Practitioner ID

MBR_ID

Record at line number: 4 must supply a Patient Member ID

MBR_DATE_OF_BIRTH MM/DD/YYYY

Record at line number: 5 must supply a Date of Birth of format: MM/DD/YYYY

MBR_FIRST_NAME

Record at line number: 6 must supply a first name for the patient

MBR_LAST_NAME

Record at line number: 7 must supply a last name for the patient

MEDICATION

Record at line number: 8 must supply a medication name

MEDICATION_DOSE_QUANTITY

Record at line number: 9 must supply a dose quantity

MEDICATION_DAYS_SUPPLY

Record at line number: 10 must supply a numerical days supply value

MEDICATION_LAST_FILL_DATE MM/DD/YYYY

Record at line number: 11 must supply a last fill date of format: MM/DD/YYYY

MEDICATION_NDC_CODE

Record at line number: 12 must supply an NDC code

DRUG_CLASS_PDC

Record at line number: 13 must supply a PDC score; Record at line number: 13 must supply a PDC score between 0 and 100 inclusive

OPTIMAL_INDICATOR

Record at line number: 14 must supply an optimal indicator of the case insensitive form: 'optimal' or 'suboptimal'

THERAPY_CLASS

Record at line number: 15 must supply a therapy class of one of: diabetes, cholesterol, or hypertension

OPPORTUNITY_TEXT

Record at line number: 16 must supply opportunity text when record is 'suboptimal'

ORGANIZATION_NAME

Record at line number: 17 must supply an organization name
