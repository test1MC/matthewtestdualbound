Sending Prior Authorization Initiation Requests

A PAInitiationRequest should only be sent when it is reasonable to expect that the selected drug requires a prior authorization. Provider vendors should not request a prior authorization for every drug through an automated process. Provider vendors should use the Prior Authorization flag in the formulary list as the primary source of information regarding the need for a prior authorization.

Unlike other messages, Surescripts does not require provider vendors to determine ePA support by the PBM/payer. Provider vendors can initiate a PAInitiationRequest, no matter who the patient’s PBM/payer is, whenever the provider has reason to believe that the drug requires a PA. All PAInitiationRequests should be addressed to EPAINI.

For PBM/payers not connected to Surescripts for ePA, Surescripts uses PDR to locate and present PDF PA forms within the prescriber’s workflow. Please see PDR Integration for more information about PDR.

Provider vendors should send information (Payer ID, INN (formerly known as BIN), PCN, etc.) from the Eligibility Response in all PAInitiationRequests.

At a minimum, the PayerID (the Surescripts PBM Participant ID) is needed for Surescripts to be able to determine if the PBM/payer is enabled for ePA. If the provider vendor does not send a PayerID, Surescripts will return an error. In order to maximize the potential for a positive ePA response, the provider vendor needs to send all available fields referenced in the table below.
