Cancel Rx

The Cancel Rx Request message is used to notify the pharmacy that a previously sent prescription should be cancelled and not filled. The message is originated by the prescriber system as a Cancel Rx Request message.

The Cancel Rx Response message is sent from the pharmacy to the prescriber system in response to a Cancel Rx Request message.

Cancel Workflow One

  1. Prescriber completes patient consultation and sends a NewRx message with four refills to the patient's pharmacy of choice.
  2. Prescriber realizes that a mistake has been made with the prescription.
  3. Prescriber initiates a Cancel Rx Request message to the pharmacy to notify the pharmacy to cancel a prescription that has not yet been dispensed.
  4. In this scenario, prescribers should send a Cancel Rx Request message to the pharmacy. Sending this message indicates that prescriber wants to cancel the entire prescription.

Scenario 1:

  1. Patient has not picked up the medication; pharmacist cancels the prescription from the system and returns it back to stock. Pharmacist also sends back a Cancel Rx Response message to the prescriber.
  2. In this scenario, pharmacy should send an "A" (Approval) code back in the Cancel Rx Response message to the prescriber. Sending this code back would clearly indicate to the prescriber that the pharmacy has cancelled the entire prescription and no medication was dispensed to the patient.
  3. Prescriber systems should maintain a record within their system, i.e. patient prescription profile to document that a Cancel Rx Request was initiated along with the acknowledgment and the action taken by the pharmacy.
  4. Pharmacy customers should update the patient medication profile within their systems to document along with the appropriate reasons.

Scenario 2:

  1. Patient has picked up the medication and pharmacy cancels the remaining refills. Pharmacy then sends back the Cancel Rx Response message to the prescriber indicating that the prescription has already been picked up, i.e. "first fill" by the patient but the pharmacy has cancelled the remaining refills.
  2. Pharmacy should send an "A" (Approval) code in the Cancel Rx Response message with a Note that specifies how many fills have been dispensed already. This would clearly indicate that partial fills were dispensed to the patient but all remaining refills have been cancelled.
  3. Prescriber systems should maintain a record within their system, i.e. patient prescription profile to document that a Cancel Rx Request was initiated along with the acknowledgment and the action taken by the pharmacy.
  4. Pharmacy should update the patient medication profile within their systems to document along with the appropriate reasons.

Scenario 3:

  1. Patient has picked up all the medications. There is nothing left in the record to be canceled. Pharmacist sends back a Cancel Rx Response message to the physician.
  2. Pharmacy should send an "A" (Approval) code in the Cancel Rx Response message with a Note that specifies how many refills have been dispensed already. This would clearly indicate the number of prescription refills that were dispensed and that there are no remaining refills left. Pharmacy should mark the prescription cancelled so that a refill request will not be sent at a later date.
  3. Prescriber systems should maintain a record within their system, i.e. patient prescription profile to document that a Cancel Rx Request was initiated along with the acknowledgement and the action taken by the pharmacy.

Scenario 4:

  1. Pharmacy can't identify the patient or the patient in the pharmacy system doesn't have the medication that the prescriber wants to cancel.
  2. In this scenario, pharmacies should send a "D" (Denied) code back in the Cancel Rx Response message to the prescriber. Sending this code back would clearly indicate to the prescriber that the pharmacy hasn't cancelled the prescription. Pharmacies should also include the denial reason or denial reason code to further explain the reason.
  3. Prescriber systems should maintain a record within their system i.e. patient prescription profile to document that a CancelRx Request was initiated along with the acknowledgement and the action taken by the pharmacy.

Cancel Workflow Two

  1. Patient has picked up three out of the six prescription fills. Prescriber sends a Cancel Rx Request message asking the pharmacy to no longer continue dispensing any open refills on an active prescription.
  2. In this scenario, prescribers should send a Cancel Rx Request message to the pharmacy. Sending this message indicates that prescriber wants to cancel the remaining prescription refills and assumes previous fills have been dispensed to the patient.

Cancel Rx Prescription Message Flow

The following steps depict the Cancel Rx message flow:

  1. Prescriber initiates a Cancel Rx Request for a patient via a prescriber system.
  2. Prescriber system successfully transmits the Cancel Rx Request to Surescripts.
  3. Surescripts validates the message. If there is an issue with the message, Surescripts sends an Error message back to the requester.
  4. Surescripts utilizes the Recipient ID and routes the message appropriately.
  5. Pharmacy system validates the Cancel Rx Request and responds with a Status, Error (if errors are found), or Verify message.
    1. If a Status 000 was sent in step 5, the pharmacy system would send a Verify.
  6. Surescripts transmits the Status, Error, or Verify back to the sender.
  7. Pharmacy processes the Cancel Rx Request and returns the prescribers' Cancel Prescription Response.
  8. Surescripts validates the message. If there is an issue with the message, Surescripts sends an Error message back to the pharmacy system.
  9. Surescripts transmits the Cancel Rx Response message back to the prescriber system.
  10. Prescriber system validates the Cancel Rx Request Response and returns a Status, Error (if errors are found), or Verify message.
    1. If a Status 000 is sent in step 10, the prescriber system would send a Verify.
  11. Surescripts transmits the Status, Error, or Verify back to the pharmacy system.

Cancel Rx General Best Practices

  1. Cancel Rx Request can be sent for paper or electronic.
  2. Cancel Rx Request should not be faxed.
  3. Cancel Rx Response time – If no response after 12 hours the prescriber should call the pharmacy.
  4. Pharmacy should always treat incoming Cancel Rx Request messages as top priority and as such display them prominently to their end users. If the Cancel Rx Request message has already been processed, further action may be required, but it doesn't need to be treated as top priority.
  5. Prescribers should not send a Cancel Rx Request for a prescription that is expired based on federal or state regulations. There should be programmatic checks in place to allow a Cancel Rx Request up to the expiration date of the prescription based off of the written date of the prescription. For example, the DEA requires Controlled Substance Rx to be filled within 6 months from the date written, and most states limit the filling of non-controlled Rx's to 1 year from the date written.
  6. Pharmacy should always include a Note in an "A" (Approval) Cancel Rx Response message when responding to a Cancel Rx Request message if the patient has picked up the "first fill" or any refills thereafter and the pharmacy is cancelling the remaining refills left on the prescription.
  7. Pharmacy should not send any Refill Renewal Request message after a Cancel Rx Request message is received.
  8. A prescriber who has assumed responsibility for the patient's care may potentially cancel any prescription, even if they are not the original prescriber. It remains up to the pharmacy to determine if the Cancel Rx Request from the prescriber is appropriate.
  9. The Cancel Rx Request should contain the RxNorm in the <DrugCoded>.
  10. The Cancel Rx Request should contain pertinent information for the pharmacy to be able to find the prescription in their system (patient, medication (name, strength, and dosage form), and prescriber). If the pharmacy cannot definitively determine the prescription to be canceled, manual processes will occur to verify the cancellation.
  11. Pharmacies should not delete a Cancel Rx Request message from a processing queue without a response being generated to the requestor. Pharmacy edits should be put in place to not allow a medication to be provided to the patient if a Cancel Rx Response has not been sent.
  12. The prescriber should notify the patient or caregiver to inform them of the cancellation of a prescription. The Cancel Rx Request was not intended to relieve the prescriber of the responsibility of notifying the patient or caregiver to advise of the drug therapy change – it is only intended as a backup to prevent inadvertent drug therapy continuation or resumption at a later date.
  13. Pharmacy should provide clear denial reasons in Cancel Rx Response denial responses. Note the Description of <DenialReasonCode> is the description of the value defined in the NCPDP External Code List. If the <DenialReasonCode> is sent, the <DenialReason> should not contain the echoing of this description as it adds no information.
