NEWRX

 

 

 

PVD

Provider Segment (Pharmacy)

One loop required for the pharmacy. For a new prescription the pharmacy technology vendor needs this segment to route to the appropriate pharmacy.

Variance from NCPDP SCRIPT 10.6:

In NCPDP SCRIPT 10.6, the PVD segment in a NEWRX message is conditional (not required). In this implementation and in NCPDP SCRIPT 10.8, it is mandatory.

In current implementation and in future version (10.8) of SCRIPT

REFREQ

 

 

 

DRU

Drug Segment (Drug dispensed)

One loop is required for the drug dispensed.

Variance from NCPDP SCRIPT 10.6:

In NCPDP SCRIPT 10.6, a second DRU loop for drug dispensed is conditional in a Refill Renewal Request message. In this implementation and in NCPDP SCRIPT 10.8, drug dispensed is Mandatory.

In current implementation and in future version (10.8) of SCRIPT

DRU

Drug Segment

(Drug Prescribed)

One loop is required for the drug prescribed if the request is approved. If the request is denied, the DRU segment is not required, and if it is sent, it should contain the prescribed loop from the Rx Change Request that is being denied.

Variance from NCPDP SCRIPT 10.6:

The DRU segment is not required for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, one loop is required for the drug prescribed if the request is approved. If the request is denied, the DRU segment is not required, and if it is sent, it must contain the prescribed loop from the Rx Change Request that is being denied.

In current implementation and in future version (10.8) of SCRIPT

RXFILL

 

 

 

DRU

Drug Segment (Drug Prescribed)

One DRU loop of Prescribed or Dispensed is required, depending on what the pharmacy has knowledge of.

If RES-020 Code List Qualifier = "AH" (Dispensed) or "AJ" (Partially Dispensed), the prescribed DRU Segment may be sent as clarification.

If RES-020 Code List Qualifier = "AK" (Not Dispensed), the prescribed or dispensed DRU Segment is required, depending upon what the pharmacy has knowledge of.

Variance from NCPDP SCRIPT 10.6:

A DRU loop for the Prescribed and the Dispensed drug is not required for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, either a Prescribed or a Dispensed segment is required.

In current implementation and in future version (10.8) of SCRIPT

DRU

Drug Segment (Drug Dispensed)

One DRU loop of Prescribed or Dispensed is required, depending on what the pharmacy knows

If RES-020 Code List Qualifier = "AH" (Dispensed) or "AJ" (Partially Dispensed), the dispensed DRU Segment is required

If RES-020 Code List Qualifier = "AK" (Not Dispensed), the Prescribed or Dispensed DRU Segment is required, depending upon what the pharmacy has knowledge of.

Variance from NCPDP SCRIPT 10.6:

A DRU loop for the prescribed and the dispensed drug is not required for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, either a Prescribed or a Dispensed segment is required.

In current implementation and in future version (10.8) of SCRIPT

VERIFY

 

 

 

STS

 

Is not required in a Verify message. If used, Status Type must be 010 and reject codes may not be used. Free text may be used.

Variance from NCPDP SCRIPT 10.6:

The STS segment Status Type must be 000 for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, Status Type must be 010.

In current implementation and in future version (10.8) of SCRIPT

DRU

SIG

 

If a Verify message contains a PVD, PTT, DRU, or SIG segment, the message will be rejected.

Variance from NCPDP SCRIPT 10.6:

For the message type Verify, a PVD, PTT, DRU, or SIG segment could be included for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, these segments are not supported.

In current implementation and in future version (10.8) of SCRIPT
