XML

Refer to SS_SCRIPT_XML_10_6MU html or SS_SCRIPT_XML_10_6MU xsd for complete documentation of the XML version of SCRIPT. The HTML version of the documentation provides an indexed version where you can drill down into the detail of each message and each type.

XML Nuances

  1. UIB Date/Time is used when mapping to XML <SentTime>. If the UIB Date/Time is empty, then the UIH Date/Time is used. When mapping from XML <SentTime> is mapped to both UIB Date/Time and UIH Date/Time.
  2. Long XML Dates are in the format CCYY-MM-DDTHH:MM:SS FZ, where the UTC time zone may be specified as Z, + 00:00, or - 00:00. For example 2013-01-01T16:09:04 5Z, or 2013-01-01T16:09:04 5-00:00, or 2013-01-01T16:09:04 5+00:00. Where 16:09:04 5Z would be 16 hours, 09 minutes, 04 seconds, 5 fractional seconds. UTC time is denoted by either the "Z" in the first example or the "-00:00" in the second, or the "+00:00" in the third example. The fractional seconds is not required, (UIB-080, UIH-050, DRU-040, OBS-010-04). Where F represents the conditional fractional seconds, F could be one or more digits.
  3. Short XML Dates are in the format CCYY-MM-DD – example: 2006-12-07 (PTT-020).
  4. When mapping to XML, if the drug name is longer than 35 characters then DRU-010-10-12 are used for the description instead of DRU-010-2.
  5. Do not use field definitions such as lengths and data types from this guide; instead refer to the schema. Also refer to the schema for the order of the fields within each element. The field order in EDIFACT is not the same as it is in XML.
  6. Fields that are concatenated into one field when mapping from EDIFACT to XML
    • UIB-060-01 & UIB-060-02
    • UIB-070-01 & UIB-070-02
    • UIB-080-01 & UIB-080-02 <SentTime>
    • DRU-010-02, DRU-010-10, DUR-010-11 and DRU-010-12 <DrugDescription>
    • DRU-030-02 and 030-03 <Directions>
    • DRU-090 Repeats 3 times <Note>
    • OBS-020 Repeats 2 times <ObservationNotes>
  7. Fields that have a different implied length in XML. These fields have a specific size defined in EDIFACT but in XML have a list of values with an implied length.
    • PTT-040 Patient Gender
    • DRU-010-03 Product Code Qualifier
    • DRU-010-08 DB Drug Qualifier
    • DRU-050 Substitution
    • DRU-070-01 Diagnosis / Clinical Information Qualifier
    • DRU-070-03 & 05 Diagnosis Qualifier
  8. Diagnosis DRU-070-01 – EDIFACT supports values of 1 and 2. Two additional values are supported in XML; Prescriber Supplied–which is equivalent to1, and Pharmacy Inferred– which is equivalent to 2.
  9. Diagnosis Qualifier DRU-070-03 & 05– EDIFACT supports values of E, F, M, ABF, and DX. XML also supports ICD9 which is equivalent to DX and ICD10 which is equivalent to ABF.
  10. Empty Tags. Do not send empty elements within XML messaging. Exceptions are select elements without data such as the Response Codes for a refill where no data is required within the tag.
  11. DRU-020-02 Drug Quantity in EDIFACT. The length is defined as n..10, which doesn't include the decimal point. In XML it is defined as n..11 because the length includes the decimal point.
  12. UIB-060 and UIB-070 To and From tags are defined with a length of 80 in XML, whereas in EDIFACT they are 35.
  13. The order of the Clinic Name and Name elements is different between the Prescriber and Supervisor types. In the Prescriber type, the Clinic Name comes before the Name and in the Supervisor type, the Clinic Name comes after the Supervisor Name.
