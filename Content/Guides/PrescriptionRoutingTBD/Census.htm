Census

In LTC, three changes to the patient status occur regularly that drive processes at the pharmacy but are not necessarily tied to a prescription. Because of this, a message type is necessary to convey patient information “decoupled” from any prescription information. The three key use cases that are relevant in LTC are New Admission, Information Change, and Discharge.

The Census message is originated by the facility in an LTC environment. This message notifies the pharmacy about census events and can be used in three cases - to notify the pharmacy of a new resident, a change to demographic information of a resident, or the discharge of a resident.

The differentiation of the three cases is with the use of the 010 Message Function Coded in the REQ Segment.

Census Message Flow

The following steps depict the Census message flow:

  1. A resident record is added or updated within a facility system.
  2. Facility system submits a Census message via Surescripts to the pharmacy.
  3. Surescripts validates the message. If there is an issue with the message, Surescripts sends an Error message back to the requester.
  4. Surescripts successfully transmits the Census message to the pharmacy.
  5. Pharmacy system validates the message and returns a Status, Error (if errors are found), or Verify message.
    1. If a Status 000 is sent in step 5, the pharmacy system would send a Verify.
  6. Surescripts transmits the Status or Error message back to the facility system.
