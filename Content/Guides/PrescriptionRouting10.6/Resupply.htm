Resupply

In the LTC environment there is a need to send a Resupply message from a facility to a pharmacy. An example use case is when a medication supply for a resident is running low (2-3 doses) and a new supply is needed from the pharmacy, the nurse needs a way to notify the pharmacy that more medication is needed. Since medication has already been authorized on the original order, the Resupply is simply a request for more inventory, therefore this is not a fillable message like the Refill Renewal Response.

Resupply Message Flow

When it is determined that medication supply is running low, the following steps depict the Prescription Resupply message flow:

  1. The Resupply message is successfully transmitted via the facility system to Surescripts.
  2. Surescripts validates the message. If there is an issue with the message, Surescripts sends an Error message back to the requester.
  3. Pharmacy system validates the Resupply and returns a Status, Error (if errors are found), or Verify message.
    1. If a Status 000 is sent in step 3, the pharmacy system would send a Verify.
  4. Surescripts transmits the Status, Error, or Verify message back to the facility system.
