Verify

This NCPDP SCRIPT message is returned only in an asynchronous response. The Verify message confirms delivery of a message to its final destination. The Verify message is an NCPDP message that indicates the acceptance of the request. This message is used to communicate the data content status of a message and to indicate the delivery of the message to its final destination.

A Verify indicates the process is complete. Subsequent acknowledgment messages should not be expected.

Variance from NCPDP SCRIPT 10.6:

The STS segment Status Type must be 000 for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, Status Type must be 010.

Variance from NCPDP SCRIPT 10.6:

For the message type Verify, a PVD, PTT, DRU, or SIG segment could be included for NCPDP SCRIPT 10.6. In this implementation and in NCPDP SCRIPT 10.8, these segments are not supported.
