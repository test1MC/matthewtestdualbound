Refill Renewal

A Refill Renewal Request message is originated by the pharmacy. This message is for the purpose of requesting approval for additional refills of a prescription once the original number of refills has been dispensed.

If a patient has approved refills remaining on a current prescription, no electronic message is required for a dispensing to occur.

A Refill Renewal Response is sent by the prescriber to the pharmacy in response to a request to refill a prescription. The response indicates whether the Refill Renewal Request has been accepted or denied.

Refill Renewal Message Flow

The following steps depict the Refill Renewal message flow:

  1. The Refill Renewal message is successfully transmitted from the pharmacy system to Surescripts.
  2. Surescripts validates the message and utilizes the Recipient ID to route the message to the appropriate prescriber via their prescriber system. If there is an issue with the message, Surescripts sends an Error message back to the requester.
  3. Prescriber system validates the Refill Renewal Request and returns a Status, Error (if errors are found), or Verify message.
    1. If a Status 000 is sent in step 3, the prescriber system would send a Verify.
  4. Surescripts transmits the Status, Error, or Verify message back to the pharmacy system.
  5. Prescriber system processes the Refill Renewal Request and returns the prescriber's Refill Renewal Response message.
  6. Surescripts validates the message. If there is an issue with the message, Surescripts sends an Error message back to the prescriber system.
  7. Surescripts transmits the Refill Renewal Response message back to the pharmacy system.
  8. Pharmacy system validates the Refill Renewal Response and returns a Status, Error (if errors are found), or Verify message.
    1. If a Status 000 is sent in step 8, the pharmacy system would send a Verify.
  9. Surescripts transmits the Status, Error, or Verify back to the prescriber system.

Refill Renewal Best Practices

In SCRIPT 10.8 the mandatory drug loop was changed from “prescribed” to “dispensed”. Since we are moving toward a mandatory segment, we need to require both in the transition period. The reason for sending the prescribed is to assist the prescriber in matching the medication to what was sent on the original NewRx.

On the Refill Renewal Request the pharmacy should use both the prescribed and dispensed medication loops to indicate the number of refills requested.

Surescripts requires that the medication prescribed and medication dispensed loops be included on the Refill Renewal Request. The medication prescribed loop from the pharmacy should not contain the literal prescribed medication information that was provided by the prescriber on a NewRx, but instead should include the pharmacist’s interpretation of the medication ordered by the prescriber. The medication dispensed is the medication that the pharmacy dispensed to the patient. The pharmacy is requesting that the medication listed in the dispensed loop be renewed. The prescriber system should indicate that the dispensed medication is the item being requested for renewal. The medication prescribed information is displayed to the prescriber for cross-reference only.

As stated in the RES segment definition, only certain fields can be changed on a Refill Renewal Response for an Approve. Refer to RES-010, for details.

If the Refill Renewal Response is for a controlled substance, the prescriber is required to sign the medication prescribed information. Even though the medication prescribed is sent for cross-reference on the Refill Renewal Request, it becomes what the prescriber is approving on the Refill Renewal Response. Therefore, the prescriber is actually approving both what was prescribed and dispensed. The pharmacy can still substitute and dispense what was dispensed before.

Example:

NewRx sent by prescriber with substitutions allowed.

Medication Prescribed: LIPITOR 10MG TABLET
Refill Qualifier: R
Refill Value: 1

Pharmacy dispenses SIMIVASTATIN 10MG TABLET

Pharmacy sends Refill Renewal Request

Medication Prescribed: LIPITOR 10MG TABLET (Name can be changed to match what is in pharmacy’s database for this medication.)
Refill Qualifier: P
Refill Value: 2

Medication Dispensed: SIMIVASTATIN 10MG TABLET

Refill Qualifier: P
Refill Value: 2

Prescriber approves and sends Refill Renewal Response

Medication Prescribed: LIPITOR 10MG TABLET
Refill Qualifier: A
Refill Value: 2

Medication Dispensed: SIMIVASTATIN 10MG TABLET

Refill Qualifier: A
Refill Value: 2

Refill Renewal Prescriber Best Practices

  1. Prescriber should respond to all requests received within 48 hours. Prescribers should not delete a request message from a processing queue without a response being generated to the requestor.
  2. If another prescriber is responding to a Refill Renewal Request for non-controlled substances on the original prescriber's behalf (i.e. original prescriber who is out of the office), then one of the following options should be used.

    Option A – Using Prescribing Agent – If the prescriber is out for a short time, then the prescriber field in the Refill Renewal Response should contain the prescriber's name, but the Prescribing Agent field should contain the name of the prescriber that is authorized to approve/deny refills for the prescriber.

    Option B – If the prescriber is out for a long time, the original Refill Renewal Request should be denied using the DeniedNewRxToFollow (DNTF) status type and followed by a NewRx. The DNTF Refill Renewal Response should contain the original prescriber's name, and the NewRx should be from the new prescriber.

  3. If the prescriber order number cannot be used to match a NewRx to a Refill Renewal Request; the following data in the prescribed drug, prescriber, and pharmacy segments in the Refill Renewal Request should be used to link the Refill Renewal Request to the original NewRx:

    • <RelatesToMessageID>

    • Patient matching: name, date of birth, gender

    • Prescribed Drug – one or more of these data elements in the prescribed drug segment: Drug name, dose form, strength, NDC, FMT coded quantity

    • Prescriber – One or more of these elements: SPI, NPI, name, communication number

    • Pharmacy – One or more of these elements: NCPDP ID, NPI, Pharmacy Name, Communication Number

  4. Many of our vendors have found it helpful to couple the Refill Renewal Request/Refill Renewal Response/Denied New Prescription To Follow processes together, so they do not send the Refill Renewal Response until the NewRx has been completed. This helps keep the messages tied together and prevents issues with the pharmacy receiving the Refill Renewal Response, but not the NewRx.
  5. When a prescriber leaves a practice the prescriber should be disabled in the directory from that practice as soon as they leave to prevent renewals from being sent and approved for a prescriber no longer practicing at that location.

Refill Renewal Pharmacy Best Practices

  1. If the prescriber has not responded within 48 hours, another message should not be sent. Instead, the pharmacist should call the prescriber.
  2. A pharmacy may send a Refill Renewal Request for controlled substances regardless of whether the prescriber is set up for the EPCS service.
  3. The pharmacy application should only allow a Refill Renewal Request to be sent when there are no more refills on the original prescription to fill or if the prescription is expired.
  4. When the application receives the following data points, the screen should also display:

    • Prior Authorization Number and Prior Authorization Status

    • Observation Segment: This information (Weight, Height) is especially important for infused, injected, oncology, and pediatric medications

  5. Renewal routing:
    1. Retain the original sending SPI for electronically received NewRx. Evaluate this SPI (e.g., has Refill Renewal Service Level), and send REFREQ to this record.
    2. Instead of selecting the first SPI in the file where the SPI has renewals enabled, evaluate all of the provider's SPIs where renewals are enabled to establish confidence to send the electronic request. Take into account the individual provider and location context necessary to find a prescriber within the appropriate organizational context (e.g., find the Individual [NPI or DEA or Name] at a Location [Address Line 1or Address Line 2 or zip code or phone or fax or clinic name], and Refill Renewal Service level).
    3. If more than one SPI is found with Refill Renewal Service level, it is recommended that additional refinement be performed based on information from the new prescription, or provide options for manual review.
    4. Pharmacies should send Refill Renewal Request via a non-EDI method (i.e. fax, call) when prescriber location is not found.
