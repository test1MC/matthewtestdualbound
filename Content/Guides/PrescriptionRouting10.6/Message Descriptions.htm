Message Descriptions

NewRx

This NCPDP SCRIPT message is a request for a new prescription and is sent from the prescriber via the prescriber system to the pharmacy of the patient's choice.

Refill Renewal Request/Response

This NCPDP SCRIPT message is sent from the pharmacy to the prescriber via the prescriber system to request a renewal for a patient's prescription. The response is sent by the prescriber via the prescriber system to indicate whether the renewal request has been approved or denied.

Rx Change Request/Response

This NCPDP SCRIPT message is sent from the pharmacy to the prescriber via the prescriber system to request a change for a new prescription. For example, the pharmacy may request the use of a generic substitution. The response message is sent by the prescriber to the pharmacy indicating acceptance or denial.

Cancel Rx Request/Response

This NCPDP SCRIPT message is sent by the prescriber via the prescriber system to the pharmacy to request that a previously sent prescription should be cancelled and not filled. The response message is sent by the pharmacy to the prescriber indicating acceptance or denial of the cancellation.

Census

This NCPDP SCRIPT message is sent by the LTC facility to the pharmacy via the prescriber system to notify them about census events. For example, the three instances would be to notify the pharmacy of a new resident, a change to the demographic information of a resident, or the discharge of a resident.

Resupply

This NCPDP SCRIPT message is sent from a LTC facility to a pharmacy via the prescriber system to request a refill.

RxFill

This NCPDP SCRIPT message is sent from a pharmacy to the prescriber via the prescriber system to notify the prescriber of the dispensing status of a prescription.

Acknowledgement Messages

Verify, Error, and Status transactions communicate current information regarding the delivery of a message.

Verify

This NCPDP SCRIPT message is returned only in an asynchronous response. This message is used to indicate the electronic delivery of the message to its final destination.

Error

This NCPDP SCRIPT message indicates that an error has occurred. An error can be generated when there is a communication problem or when an error occurs during the processing of the message.

Status

This NCPDP SCRIPT message is used to indicate the message was accepted for processing.
