Error

At multiple points in the process, an ERROR message can be generated. The following is an example of an Error Message.

In the EDIFACT examples, line breaks are used at the end of the segments for display purposes – live messages should not contain line breaks.

ERROR (EDIFACT)

UNA:+./*'

UIB+UNOA:0++888333:123+++1234563:P::678+1234567890123:D::12345+20020702:140429'

UIH+SCRIPT:010:006:ERROR++991++20020702:140429'

STS+900+007'

UIT++3'

UIZ++1'

Segment

Value

Note

UIB-010

UNOA:0

UNOA is the Syntax identifier and 0 is the syntax version number.

UIB-030

888333:123

888333 is the Message ID; 123 is the Relates To Message ID from the Message being responded to.

UIB-060

1234563:P:::678

1234563 is the identification number (NCPDP ID) of the sender (pharmacy); P is the code qualifier for pharmacy; 678 is the Surescripts-populated Account ID of the sender (pharmacy).

UIB-070

1234567890123:D
::12345

1234567890123 is the identification number (SPI) of the receiver (prescriber); D is the code qualifier for prescriber; 12345 is the Surescripts-populated Account ID of the receiver (prescriber).

UIB-080

20020702:140429

20020702 is the date the message was sent (07/02/2002); 140429 is the time the message was sent (02:04:29 P.M.).

UIH-010

SCRIPT:010:006:
ERROR

SCRIPT is the message type; 010 is the version; 006 is the release to be used in decoding this message. All messages in this set use these values. ERROR is the message function.

UIH-030

991

991 is the prescription number assigned by the prescribing system.

UIH-050

20020702:140429

20020702 is the date the message was sent in format CCYYMMDD (07/02/2002); 140429 is the time in format HHMMSS (02:04:29 P.M.).

STS-010

900

900 – indicates the message was rejected.

STS-020

007

007 indicates Internal Error.

UIT-020

3

3 is the number of segments in the message between and including the UIH and UIT segments.

UIZ-010

1

1 is the Interchange Control Count – the number of messages in this interchange.
