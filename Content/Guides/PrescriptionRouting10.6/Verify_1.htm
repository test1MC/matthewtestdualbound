Verify

Surescripts uses a VERIFY message as a response for certain messages. The following is an example of what a Verify message may look like.

In the EDIFACT examples, line breaks are used at the end of the segments for display purposes – live messages should not contain line breaks.

VERIFY (EDIFACT)

UNA:+./*'

UIB+UNOA:0++888333:123+++1234563:P::678+1234567890123:D::12345+20020702:140429'

UIH+SCRIPT:010:006:VERIFY+1515+991'

STS+010++TRANSACTION RECEIVED'

UIT+1515+3'

UIZ++1'

Segment

Value

Note

UIB-010

UNOA:0

UNOA is the Syntax identifier; 0 is the syntax version number.

UIB-030

888333:123

888333 is the Message ID; 123 is the Relates To Message ID.

UIB-060

1234563:P::678

1234563 is the identification number (NCPDP ID) of the sender (pharmacy); P is the code qualifier for pharmacy; 678 is the Surescripts-populated Account ID of the pharmacy (sender).

UIB-070

1234567890123:D::
12345

1234567890123 is the identification number (SPI) of the recipient (prescriber); D is the code qualifier for prescriber; 12345 is the Surescripts-populated Account ID of the prescriber (recipient).

UIB-080

20020702:140429

20100125 is the date the message was sent (07/02/2002); 140429 is the time the message was sent (02:04:29 P.M.).

UIH-010

SCRIPT:010:006:
VERIFY

SCRIPT is the message type, 010 is the version; 006 is the release to be used in decoding this message. All messages in this set use this value. VERIFY is the message function.

UIH-020

1515

1515 is the prescription number asssigned by the pharmacy.

UIH-030

991

991 is the prescription number assigned by the prescribing system.

STS-010

010

010 indicates the message was successfully accepted by ultimate receiver.

STS-030

TRANSACTION RECEIVED

Free text description of status.

UIT-010

1515

1515 is the Message Reference Number assigned by the pharmacy. This must match the value in UIH-020.

UIT-020

3

3 is the number of segments in the message between and including the UIH and UIT segments.

UIZ-020

1

1 is the Interchange Control Count – the number of messages in this transaction.

<Message xmlns="http://www.ncpdp.org/schema/SCRIPT" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" version="010" release="006" xsi:schemaLocation="http://www.ncpdp.org/schema/SCRIPT SS_SCRIPT_XML_10_6MU.xsd">

  <Header>

    <To Qualifier="D">1234567890123</To>

    <From Qualifier="P">1234563</From>

    <MessageID>888333</MessageID>

    <RelatesToMessageID>123</RelatesToMessageID>

    <SentTime>2010-01-25T08:13:22</SentTime>

    <Security>

      <Sender>

        <TertiaryIdentification>12345</TertiaryIdentification>

      </Sender>

    <Receiver>

        <TertiaryIdentification>678</TertiaryIdentification>

      </Receiver>

    </Security>

    <PrescriberOrderNumber>PON123</PrescriberOrderNumber>

  </Header>

  <Body>

    <Verify>

      <VerifyStatus>

        <Code>010</Code>

      </VerifyStatus>

    </Verify>

  </Body>

</Message>
