PVD – Prescriber Segment

The following should be used for developing the Prescriber and Supervisor Segments.

In EDIFACT the qualifier in PVD-010 will specify the type of data the segment contains. In XML the start/end tag will specify the type of data the segment contains.

For prescriber initiated messages (NewRx, Refill Renewal Response, Rx Change Response, Cancel Rx Request, Resupply, and Census) at least one occurrence must contain the individual (not organizational) NPI of the prescriber and supervisor if the supervisor loop is sent.

If NPI is sent it will be validated against the check digit routine. For specific information see http://www.cms.gov/NationalProvIdentStand/Downloads/NPIcheckdigit.pdf

For prescriber initiated messages for controlled substances at least one occurrence must contain the DEA number.

The exception to this rule is a Public Health provider can use their Social Security Number in place of a DEA number. Military personnel are covered under Public Health providers. Military personnel are also required as part of the 1311 audit to report their branch of service in the DRU-090 notes field.

Each prescriber of Controlled Substances will need to have a unique DEA or Institutional DEA Number registered/ assigned to them in the Surescripts Directory and the EPCS Service level enabled.

Only one DEA number for the location where the prescriber is located should be sent.

If the prescriber does not have a DEA number, the prescription has to be handed over to the prescriber's supervisor who in turn becomes the prescriber.

The SPI is no longer used here since the increase in the number of other identifiers. Use the SPI at the header level.

D

PVD-080

ADDRESS <Address>

  

M = NewRx (Prescriber and Supervisor)

C = All other messages for prescriber and supervisor

The address of the provider of the prescription order should be the practice physical address.

On a Refill Renewal Response the prescriber should change this address to match what is in the Surescripts directory. For address information, customers should use the United States Postal Service (USPS) 28 standard as a guideline.

PVD-100

NAME

<PrescriberAgent>

  

This composite is used to identify the Designated Agent - use for transmitter/submitter name. This composite is to be used in the "PC" PVD Segment loop only. (If present, first name and last name should be used.)

If a supervising prescriber name or ID is required, use another repetition of the PVD Segment placing the ID in the reference number field.

For NewRx, Refill Renewal Response, Cancel Rx Request, Rx Change Response, or Resupply – this field is mandatory if the sender of the message is not the actual prescriber referenced in PVD-020-01.
