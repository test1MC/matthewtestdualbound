Patient Enrollment File Load

Character Set

The character set contains ASCII values 32 – 126, which include:

Letters, upper and lower case A to Z, a to z

Numerals Ø to 9

Symbols ! " # $ % & ' ( ) * + , - . / : ; < = > ? @ [ \ ] ^ _ ` { | } ~

Unprintable characters, such as control characters, are not used within the field sets. Defined unprintable characters are used as delimiters.

For the quotation symbol (") there must be a closed quote or corresponding quote for every opening ".

Best Practice

All non-essential characters should be suppressed from the message where permissible. Nonessential characters include leading spaces, trailing spaces, leading zeros, and trailing zeros.

File Naming and Formats

Patient Enrollment File

The naming convention is CustomerName_PMA_YYYYMMDD-XX.CompressionFormat. It is strongly suggested to compress the file prior to transmitting it to Surescripts. The current zip protocols that are supported are gzip, bzip2, and zip.

Example: ACOCompany_PMA_20161204-01.gz

In the naming convention, "XX" represents the sequence number. Increment this number when sending multiple files in one day.

Patient Enrollment Response File

The naming convention is <inputFileName sans extension>.<sstimestamp>.<sstimestamp>.rsp.

Example: ACOCompany_PMA_20161204-01.20161204012025.20161204012025.rsp

Patient Enrollment File Load

The Patient Enrollment file format will be pipe-delimited (hex7c). Each field is delimited by the pipe character (|). If a pipe-character (hex7c) is used in field data, it must be escaped with \F\. For example, Main | Street would be listed as Main \F\ Street. Each line is to be separated by a new line (hex0A) character.

If Mandatory fields are not included, the Patient Enrollment File and/or patient record will not pass validation. All Conditional fields must be accounted for, either with data or a placeholder delimiter.

Patient Enrollment Response File

Each Patient Enrollment Response File will contain an overall file load status code with description in the header record of the response file. This code indicates if the file loaded with either partial or full success (i.e., patients were loaded from the file), and/or if errors were encountered.

In addition, there will be a record containing an error code and description for every error found in the file. These records will include the line number of the incoming file that had the related error. Line number starts with 1 for the header record and is incremented by 1 for every record in the incoming file.

Line number 1 will indicate an error on the incoming file header. Line numbers 2 - n will indicate errors on subsequent records in the incoming file. There may be multiple records listing an error code and description for the same line number from the incoming file.

The Patient Enrollment Response File format is pipe-delimited (hex7c). Each field is delimited by the pipe character (|). If a pipe-character (hex7c) is used in field data, it will be escaped with \F\. For example, Main | Street will be listed as Main \F\ Street. Each line will be separated by a new line (Hex 0A) character.
