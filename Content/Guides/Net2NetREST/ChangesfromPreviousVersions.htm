Changes from Previous Versions

Changes from Version 1.4 to 1.5

  1. Updated header to include version="CIv1" release="5"
  2. Added the following conditional elements to the Message header:
  • To/@FriendlyName
  • To/@Route
  • From/@FriendlyName

Changes from Version 1.3 to 1.4

  1. Updated header to include version="CIv1" and release="4".
  2. Added multiple-recipient messaging.

Changes from Version 1.2 to 1.3

  1. Updated header to include version="CIv1" and release="3".
  2. Added Patient Demographics section to the header. Previous versions used "x-patient" document data items.
  3. Removed PlainText and HtmlText attachment types. These are no longer discrete attachment types; however, this content can still be sent as a Base64 encoded File attachment.
  4. Removed Network Address from RecipientInformation
  5. Removed unused items from the schema. "HighestVersionSupported" attribute and "Mailbox", "Security", and "ClinicalEvent/MessageRead" elements were removed from the schema.
  6. Increased DocumentName length from 35 to 255 characters.

Changes from Version 1.1 to 1.2

  1. Added DIRECT qualifier on From and To elements. This allows messages to be sent to/from trading partners using a direct address
  2. Recipients collection in the header changed to single Recipient element to reflect that only one (1) recipient is supported.
  3. Moved Status, Error and Verify messages to the N2NMessage body to allow for routing with the DIRECT qualifier.
