Detailed PMBC Error Message Scenarios

The following diagrams depict various scenarios where Error messages or PBMC Response messages with business errors are sent in response to a PMBC message.

Scenario 1: Surescripts Cannot Recognize the Message. Surescripts returns an error message.

1a) A provider vendor sends a PMBC Request to Surescripts.

1b) Surescripts cannot recognize the message (e.g. improperly formed XML) and sends an error back to the provider vendor.

Scenario 2: Surescripts Finds Message Errors in PMBC Request. Surescripts returns an ERROR message.

2a) A provider vendor sends a PMBC Request to Surescripts.

2b) Surescripts recognizes the format but finds errors (e.g. schema validation error) in the message. Surescripts returns an ERROR message.

Scenario 3: PBM/payer Finds Message Errors in PMBC Request. PBM/payer returns an ERROR message.

3a) A provider vendor sends a PMBC Request message to Surescripts.

3b) Surescripts forwards the PMBC Request to the PBM/payer.

3c) During validation, the PBM/payer finds message errors and sends an ERROR message to Surescripts.

3d) Surescripts forwards the ERROR message to the provider vendor.

Scenario 4: PBM/payer Finds Business Errors in PMBC Request. PBM/payer returns a PMBC Response Message with Business Errors.

4a) A provider vendor sends a PMBC Request to Surescripts.

4b) Surescripts forwards the PMBC Request to the PBM/payer.

4c) During business processing, the PBM/payer finds business errors (e.g. Patient Not Found, Drug Not Found, etc.) and returns a PMBC Response message to Surescripts.

4d) Surescripts forwards the PMBC Response to the provider vendor.
