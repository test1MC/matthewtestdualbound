Panel Management RxHistory Response XML Details

Overall File Header

The PatientFileID echoes the Patient Population Identifier sent in the patient load file.

SentTime is the date/time that the Medication History Panel Management File is posted.

TransmissionControlNumber is used to link the Panel Management Patient File Load to the appropriate RxHistory Response file sent back by Surescripts.

Response

When AQ is present in the ApprovalReasonCode, a second Panel Management RxHistoryResponse will be sent from Surescripts to the customer.

A Denied response will be accompanied by Free Text (DenialReason). If there is an error in retrieving the data, DenialReason will be populated. See Error Scenarios on page 1 for more information.

Prescriber

The Prescriber element is used to populate the LastName (organization name or prescriber last name) and the NPI for organization or individual prescriber.

Optionally, the Prescriber element is used in each MedicationDispensed loop to indicate the prescribing physician for the drug listed.

Pharmacy

Optionally, the Identification element will include the reference number for the pharmacy.

Patient

The Patient ID provided in the Patient Load file will be populated in the MutuallyDefined identification field.

Medication Dispensed

Zero to 300 loops of the drugs in the patient’s history. If there are more than 300 drugs in the patient’s history, a second RxHistoryResponse will be included in the Panel Management RxHistory file. The most recent drug according to date filled is shown first.
