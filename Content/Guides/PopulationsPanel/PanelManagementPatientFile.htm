Panel Management Patient File Load

The patient file is a list of patients sent by the customer to Surescripts and is used to locate the medication history for each patient in the list.

Character Set

Alphanumeric data must only contain ASCII characters with decimal values of 32 through 126, excluding the pipe (hex7c) delimiter character and line feed (hex 0a). All characters outside this range will be rejected. See http://www.asciitable.com/ for a list of valid characters.

Numeric data is composed of the characters 0-9.

Data should be UTF-8 encoded.

File Naming and Formats

Patient Input File Naming and Formats

The recommended patient input file naming convention is CustomerName_PNM_YYYYMMDD-XX.CompressionFormat. It is strongly suggested to compress the file prior to transmitting it to Surescripts. The current zip protocols that are supported are gzip, bzip2 and zip. See the following for an example Patient Input File name:

Patient Input File: ACOCompany_PNM_20141204-01.gz

Validation Response File Naming and Formats

The naming convention for the Validation Response File that the customer will receive is: <inputFileName sans extension>.<sstimestamp>.<sstimestamp>.rsp.

If the input file was not compressed, the <sstimestamp> will not be included in the file name.

See the following for an example Validation Response File name:

Validation Response File: ACOCompany_PNM_20141204-01.20141204012025.20141204012025.rsp

RxHistory Response File Naming and Formats

The RxHistory Response File naming convention is: <PatientPopulationIdentifier from the Panel Management Load Delimited File><Surescripts’ assigned ID>_<yyyyMMddHHmmss>.gz

See the following for an example RxHistory Response File name:

ACO132PatientPopulationA12345_20141204012025.gz

Patient Input File Load Delimited File

The input file format will be pipe-delimited (hex7c). Each field is delimited by the pipe character (|). If a pipe-character (hex7c) is used in field data, it must be escaped with \F\. For example, Main | Street would be listed as Main \F\ Street. Each line is to be separated by a new line (hex0A) character.
