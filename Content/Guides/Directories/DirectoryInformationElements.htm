DirectoryInformation Elements

DirectoryInformation is common to all Add and Update messages and their corresponding responses.

Requirements Designation

Each segment and data element has a requirement designation. If a data element is listed as Mandatory, it must be present in the transmission. If a Conditional complex type data element is used, all mandatory data elements contained within it are required.

Element Attributes:

Code

Description

M

Required/Mandatory - the element must be used – per the specification (e.g. XML schema validation).

B

Required – the element must be used per the Surescripts business rule.

C

Situational/Conditional - the element must be used if conditions are met. Some fields do not have specific conditions. Data should be sent if available.

M

ActiveEndTime

UtcDate Type

Date when the record is disabled and will no longer be considered enabled on the Surescripts Directory.

Note: To disable a record, ActiveEndTime (AET) must be set to the current date and ServiceLevel must not be present in the message. The record is removed from the full directory download file 7 days after the AET.

Note: The DeactivateProviderLocation and DeactivateOrganization messages remove all service levels.

DirectorySpecialties

DirectorySpecialtiesType

DirectorySpecialties is always required for the Pharmacy Organization type.

M

  DirectorySpecialty

DirectorySpecialtyType

Required if DirectorySpecialties is present. The schema allows up to 50 DirectorySpecialty elements.

M

  DirectorySpecialtyName

an1..35

Required if DirectorySpecialty is present. Surescripts Directory identifier that can be used to designate additional information and characteristics that apply to Providers (Optional) or the Pharmacy Organization (Required) record.

Values are case sensitive.

Provider Values - Description:

  • Doctor - Used for certified physician.
  • PA - Physician Assistant.
  • InStoreClinic - Provider is at an instore clinic versus a stand-alone practice, e.g. Urgent Care.
  • NP - Nurse Practitioner.
  • Resident - Provider practicing medicine, usually in a hospital or clinic under the direct or indirect supervision of an attending physician.
  • Telehealth - Used when the provider uses a stand-alone telehealth platform for their patient encounters.

Pharmacy Organization Type: One of the following is required for Pharmacy Organization type and is not used for General or Payer Organization types:

  • MailOrder - All prescriptions are handled through the mail.
  • Retail - A physical brick and mortar location where customer can receive medication.
  • InHouseDispensing - An internal, in plan or closed group for the corresponding plan participant.

Additional Pharmacy Values: May include one or more of the following directory specialties/characteristics:

  • Compounding - A pharmacy which can provide compounding-based services.
  • DME - Locations that provide durable medical equipment and related supplies.
  • FaxPharmacySurescripts – Surescripts enables customers using the Surescripts Fax Gateway service; will be delivered via fax. This value is not sent by customers.
  • Kiosk – A free-standing, self-service dispensing location.
  • LongTermCare - Location that accepts and works with prescriptions for long-term care patients and facilities.
  • Specialty - Location that accepts prescriptions for specialty identified medications.
  • SupportsDigitalSignature - Location that accepts digital signatures for controlled substances.
  • TwentyFourHourStore - Pharmacy location that is open and available for service 24 hours a day. These locations should be verified directly with the store.

Note: Additional values may be added by Surescripts. Systems should not fail if an unexpected value is received.

C

FaxBackup

flagType

Indicates that messages addressed to this directory entry allow Surescripts to attempt fax delivery for failed messages if all attempts for electronic delivery have failed.

Values (case insensitive):

  • N = Off
  • Y = On

Note: Set to "N" for pharmacies located in states not permitted to use fax backup.

C

ServiceLevels

ServiceLevelsType

ServiceLevels is a collection of ServiceLeveName elements in the schema that indicate the messages a record is enabled to transact. One ServiceLevelName is allowed for each ServiceLevel element. Do not send if there are no services.

M

  ServiceLevel

ServiceLevelType

Required if ServiceLevels is present. Indicates that a ServiceLevel is enabled on a record. Currently a maximum of 50 service levels are allowed.

M

     ServiceLevelName

an1..50

Service Levels(s) associated with the account record. Designated by the account record's capabilities. Values are case sensitive. For more detail, please refer to the Service Level Descriptions section.
