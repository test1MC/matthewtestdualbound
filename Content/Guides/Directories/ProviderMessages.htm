Requirements Designation

Each segment and data element has a requirement designation. If a data element is listed as Mandatory, it must be present in the transmission. If a Conditional complex type data element is used, all mandatory data elements contained within it are required.

Element Attributes:

M

B

C

M

Address

MandatoryAddressType

Surescripts strongly recommends that the address element be standardized across multiple data sources from the same vendor (directory vs message content).

S.103: The sending customer shall send an address within the message which matches the associated Directory record. Note: When a provider system utilizes the Surescripts Learning Directory service, this requirement is satisfied.

The address of the provider of the prescription order should be the practice physical address in which the patient encounter took place. For LTC based encounters, the provider should use the facility based location, or, for home-based service, their practice of record for prescription orders.  For Telehealth based encounters that are not part of an associated provider practice location, the practice address within the message, and correlating directory record, should be associated with a provider associated credentialed location in the State from which the provider is licensed.

On an RxRenewalResponse the provider should change this address to match what is in the Surescripts directory. For address element formatting and consistency information, customers should use the United States Postal Service (USPS) 28 standard as a guideline.

Used in clinical messaging to identify an endpoint (sender or receiver) when information is exchanged.
