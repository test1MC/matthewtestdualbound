Directory Download

Surescripts exports two types of downloadable flat files: Full and Nightly (delta) for both Provider Locations and Organizations. The Nightly version of the download files includes only the records that have changed since the preceding day. To ensure that all updates are made and included in the download, updates must be sent (and confirmation with a response) no later than 10 PM GMT.

The download will include records based on service level, activity on the network and account type. For a full listing of differences between the Full and Nightly file, please see the DirectoryDownload Elements section.

D.003 Customers shall integrate a process to download the Nightly or Full version of the Surescripts download file daily. Customers must use the DirectoryDownload message type for these processes.

D.004 If downloading the Nightly version daily, the customers shall integrate a weekly download of the Full version.

For D.003 and D.004, see corresponding notes in the Application Certification Requirements section.

A manual download of the file is available to Admin Console users for seven (7) days from publication.

Flat File Format

Each field element is delimited by the pipe character (|). Each line is separated by a new line (hex 0A) character. The tilde character (~) is used as a repetition character. If a pipe character (hex 7C) is used in field data, it will be escaped when returned in the directory download file with \F\. For example, Clinic | Name would be returned in the directory download as Clinic \F\ Name.

Full directory download files:

  • Records with ActiveEndTime more than 7 days in the past will not be included in directory files.
  • The record detail will remain in the Full download file for 7 days from the ActiveEndTime for a disabled provider.

Nightly directory download files:

  • All records with ActiveStartTime within the last 24 hours will be included with RecordChange field value of "N" (for New).
  • All records with ActiveEndTime within the last 24 hours will be included with the RecordChange field value of "D" (for Deactivated).

A disabled record contains no service levels and an ActiveEndTime in the past.

  • If anything other than the ActiveStartTime and ActiveEndTime has changed within the last 24 hours, the record will be included with a RecordChange field value of "U" (for Updated).

Provider Download Overview

Provider demographic and routing information are valuable assets to our provider network customers. A provider vendor has access to demographic data for providers they maintain. An organization/pharmacy vendor has access to demographic data for all e-prescribing providers. In addition, all vendors who support clinical messaging have access to demographic data for providers registered with a clinical messaging service level, such as CIMessage or CIEvent, regardless of which vendor registered the provider.

Newly registered e-prescribing providers will be active in the directory immediately. However, pharmacy customers will see the Refill service level in the download only after five successful NewRx messages have been sent by the provider.

Directory download files include all records to which the requester has access.

Organization Download Overview

There are three types of Organizations: Pharmacy, Payer or General (such as a clinic, private practice, or hospital department).

Organizations have access to demographic data for organizations they maintain. In addition, organizations who support clinical messaging have access to demographic data for organizations registered with a clinical messaging service level, such as CIMessage or CIEvent, regardless of which vendor registered the organization.

A provider vendor has access to data for all Pharmacy, Payer, and General organizations with matching service levels. For example, provider vendors with a clinical messaging service level will have access to organizations with a clinical messaging service level and e-prescribing providers will have access to e-prescribing organizations, etc.
