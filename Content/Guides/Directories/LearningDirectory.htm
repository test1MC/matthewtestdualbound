Learning Directory

The Learning Directory utilizes address demographic information content from within the NewRx message sent from the provider customer. It appends previously unknown provider locations to the Surescripts Directory and supplements the primary SPI registered in the directory.

Criteria for Learning a New Provider Location

  1. Learning Directory functionality must be enabled on the portal configuration for the provider customer.
  2. The SPI and NPI registered in the directory must match the SPI and NPI in the successfully processed NewRx.
  3. With the exception of controlled substances, other certified services may appear against the learned locations.
  4. A new location is identified for being learned if the address in the NewRx is determined to be unique against the existing directory registration. Key fields such as AddressLine1, AddressLine2 and PostalCode may be used to determine whether an address is unique. Similar validations that prevent duplication of an SPI with refills at the same physical location will be enforced.
  5. Learned locations will expire when:
    • there have been at least 6 months of inactivity,

    • the registered SPI expires,

    • a technology vendor registers the provider for the same address as the learned location with the refill service level, or

    • a learned record is disabled.

How Downloads Will Be Different

The SPI is not a unique identifier. The download format does not change; it will include all applicable results. A pharmacy account must have the appropriate "Directory Download Special Rules" in place on their account settings to publish the learned locations to their download files. An "LL" will be included in the Mutually Defined field in the download to indicate the record is a "learned record".

Learning Directory Registration

In the example below, the diagram displays how Dr. Bob works in six locations: two under Practice Group 1 and four under Practice Group 2. In Practice Group 1, he uses two different applications (a primary application "A" and a downloaded mobile application "M" which sends only new prescriptions). In Practice Group 2, the provider vendor application "B" explicitly registers Dr. Bob at the primary location of practice. Then, while writing new prescriptions from a different location where he is seeing patients, the subsequent location(s), meeting all of the criteria for being "learned", are added as a "learned location" to the registered SPI-005.

Without the Learning Directory, each location/application for a provider is registered creating a unique SPI. With the Learning Directory, one location is explicitly registered as the provider’s primary location of practice. As new locations are "learned" from new prescriptions, these "learned records" are associated with the registered SPI.

The Importance of Record Consistency

It is important that the provider vendors ensure the formatting and style of the SPI directory listings are consistent with the NewRx content.

D.010 When Provider details are included within a message transaction on the Surescripts network, the following fields shall match the information in the Surescripts Provider Directory Download for the sender and/or recipient segment:

  • SPI
  • NPI
  • Provider First Name
  • Provider Last Name
  • Address (Standardization permissible)
  • Phone (Primary preferred, Secondary accepted)
  • Fax (Primary preferred, Secondary accepted)

For D.010, see corresponding notes in the Application Certification Requirements section.

Alignment of these pieces of information helps ensure the respective databases used across the workflow are in sync. PostalCode

Including a 9-digit Postal Code (ZIP+4) offers an excellent level of CASS- standardization detail. These address attributes impact the "+4" of the Postal Code:

  • Business Name
  • AddressLine1 (AD1)
  • AddressLine2 (AD2)

If using the verified Postal Code (ZIP+4) USPS that aligns to your practice/address information, include it. If not verified, use the basic 5-digit Postal Code (ZIP5).

Do not include provider mobile/cell numbers; they are not unique to one practice group.
