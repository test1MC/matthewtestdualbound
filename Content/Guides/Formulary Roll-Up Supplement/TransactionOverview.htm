Transaction Overview

Formulary Roll-up Transaction flow

The Surescripts Prescription Benefit & Medication History service provides physicians with the electronic delivery of PBM/payer member data in an ambulatory setting. Through the prescriber vendor interface, the physician can request patient information such as eligibility and pharmacy benefit coverage, formulary, and medication history. The graphic below depicts the transaction flow between participants in this process.

The PBM/payer sends in Formulary and Benefits files. There is no change to the format or process.

  1. Surescripts processes the incoming files.
  1. Surescripts determines which Drug Databases are available for roll-up.
  1. Surescripts determines the representative NDC using the Drug Database.
  1. Surescripts rolls-up all data in the files utilizing this Drug Reference NDC to Representative NDC file. A separate roll-up is performed for each drug database.
  1. Surescripts creates exception reports that show data that needs to be reviewed.
  1. Surescripts makes the rolled up data available to the prescriber vendors.
  1. Prescriber vendors download the rolled up version of the data that was rolled up using prescriber vendors' contracted database.
  1. Surescripts makes the exception reports available to the PBM/payers. The reports will be in the same location where the formulary response file is located.
