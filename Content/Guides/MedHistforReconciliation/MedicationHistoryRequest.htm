Medication History Request XML Requirements

This Guide includes data elements only where Surescripts has specific requirements or further explains the field usage. Refer to the SS_SCRIPT_XML_10_6MU.xsd for a complete list of fields.

Field Name

Details

Header

To

To/@Qualifier

Value:

From

Must be populated with the requester's Participant ID.

Value:

MessageID A minimum set of standards/algorithms should be used when generating message IDs to ensure uniqueness. If possible, customers should utilize Global Unique Identifiers (GUIDs).

Security/Sender/SecondaryIdentification

Must be populated with the requester’s Surescripts-assigned password.

Security/Sender/TertiaryIdentification

Must be populated with the value of S00000000000002.

TestMessage

Required in the Staging environment.

TertiaryIdentifier

Must be populated with the value NTR (Non-targeted Request) to indicate it is the version of MHR that does not require a 270/271 eligibility request. However, 'NTR' is not passed on to the PBM/payer.

Patient

Address/ZipCode

It is recommended to send the Address/ZipCode to aid in patient matching.

Prescriber

Prescriber/Identification

Organizational NPI is to be sent in this field. If an Organizational NPI does not exist, then an Individual NPI of the prescriber should be provided.

The NPI will be validated against the industry standard check digit routine (LUHN check). For specific information see: https://www.cms.gov/Regulations-and-Guidance/Administrative-Simplification/NationalProvIdentStand/Downloads/NPIcheckdigit.pdf

Benefits Coordination

Note: These are the only fields used.

EffectiveDate

EffectiveDate is the beginning date for the desired history (e.g., 1 year ago).

Note: For EffectiveDate, errors will occur if the date range is more than one year, or if it is into the future.
ExpirationDate

ExpirationDate is the end date for the desired history.

This should be set to the date the RXHREQ was initiated.

Up to one year of medication history data can be requested.

Note: For ExpirationDate, errors will occur if the date range is more than one year, or if it is into the future.

Consent

Values:

Y = Consent given

X = Parental/Guardian consent on behalf of a minor for prescriber to receive the medication history from any prescriber
