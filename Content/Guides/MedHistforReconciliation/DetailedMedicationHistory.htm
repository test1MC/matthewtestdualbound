Detailed Medication History Message Flow

  1. The requester sends a Medication History Request for a patient to Surescripts.
  2. Surescripts validates the Medication History Request message.
    1. If an error occurs, Surescripts sends back an ERROR response to the requester.
    2. If no error occurs, Surescripts sends STATUS to the requester acknowledging that the request was received and the process continues per Step 3.
  3. Surescripts attempts to locate patient.
    1. If patient is not found, Surescripts sends Medication History Response back to the requester indicating the patient was not found.
    2. If patient is found, the process continues per Step 4.
  4. The request is routed to the appropriate PBM/payer(s) for processing. For fill data, the request is handled internally using the pharmacy database.
  5. Surescripts compiles the claim and fill data results.

  6. Surescripts sends one response back to the requester with up to 300 aggregated medication records. If the patient has more than 300 medication records, Surescripts sends a second response that may include up to 200 additional medication records, for a total maximum of 500 medication records.

  7. The requester sends a STATUS message to Surescripts acknowledging each response.
