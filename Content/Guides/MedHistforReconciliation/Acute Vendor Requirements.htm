Application Certification Requirements

Numeric Representation

The decimal point is represented by a period and shall (*106M-001) be used as follows:

  • only when there are significant digits to the right of the decimal
  • when there is a digit before and after the decimal point
  • not with whole numbers

For example, consider the following possible values:

Correct:

2.515

251.5

25.15

2515

0.2515

2.5

Incorrect:

.2515

2515.

3.00

 

 

 

XML Empty Tags

All non-essential characters shall (*106M-002) be suppressed from the message where permissible. Non-essential characters include: leading spaces, trailing spaces, leading zeroes, and trailing zeroes where permissible.

Participant shall (*106M-005) not send empty elements within XML messaging with exception of the RxHistoryResponse/Approved element.

Handling Optional Fields

The participant shall (*106M-007) be able to process any valid incoming transaction request or response, including syntactically valid maximum and minimum population messages.

Optional data elements (and values therein) shall (*106M-008) not cause message failure.

Common Message Header Elements

Each message sent from a participant shall (*106M-009) have a unique Message ID.

Error Handling

An Error message shall (*106M-023) never be responded to with an Error message; this could create an endless loop.

Phone Number: Pharmacy, Prescriber and Patient

Phone Number shall (*106M-025) be in the format 5552223333X4444 - where 555 is the area code, 2223333 is the main number and X4444 is the extension.

Phone number shall (*106M-028) not be populated with all zeros or other default values. This field is 80 characters long to handle email addresses. (Please note the extension X4444 is just an example. Less/more than four digits are allowed).

XML

Most symbols should be ‘safe’ within most alphanumeric fields; however, XML validation has issue with the following characters and they shall (*106M-035) be escaped. The most common escape encodings are:

ASCII Characters

XML Escape

quote ("")

&quot;

apostrophe (')

&apos;

ampersand (&)

&amp;

less than (<)

&lt;

greater than (>)

&gt;

String lengths are calculated based on the un-escaped value. For example, “>98 degrees” should be encoded as “&gt;98 degrees” but would only be counted as a string length of 11, not 14. The quote and apostrophe only need to be escaped if they are used within an attribute.

The application shall (*106M-012) display a disclaimer regarding use of medication history retrieved via Surescripts.

  • Sample Disclaimer: Certain information may not be available or accurate in this report, including items that the patient asked not be disclosed due to patient privacy concerns, over-the-counter medications, low cost prescriptions, prescriptions paid for by the patient or non-participating sources, or errors in insurance claims information. The provider should independently verify medication history with the patient.

The application shall (*106A-001) provide access to all history data available for a specific request. When a medication history response indicates more medication history is available, the application shall systematically retrieve all medication history available.
