File Loads

Surescripts processes files from customers using a first-in, first-out (FIFO) file processing approach. Files are processed by file type, in the order that they are received from customers. Excluding Formulary files, Surescripts processes and appropriately responds to each file receipt per customer within a 24-hour period from confirmed receipt of each file, unless specified otherwise in a product specific implementation guide.

Directory Download Files

Directory download files (FULL and NIGHTLY ) will be available by 7:00AM ET for all customers except when immediately following a scheduled maintenance event.

Master Patient Index (MPI)

Prescription Benefit Managers (PBMs) and other health care payers (Payers) provide Surescripts with PBM ID Load files containing their member rosters to populate the Surescripts Master Patient Index (MPI). Surescripts uses the PBM ID Load files to identify individuals across the Surescripts network. The Surescripts search process uses demographics to identify a patient and then uses the PBM/Payers’ unique member ID to communicate with PBM/Payers. Loading updated PBM ID files is critical for accurate, effective, and efficient day-to-day MPI operations. Delays or inaccuracies lead to patient matching failures.

PBMs/Payers are required to actively manage their members by loading the PBM ID Load file at regular intervals, as specified by current application certification requirements, or no less than weekly. PBM/Payers should have in place the necessary workflow/protocols to ensure that the Surescripts member directory is up-to-date for accurate patient matching.

Directory Management & Connectivity

Directory Management encompasses all actions required to register and maintain organizations, including Pharmacies, Payers, and General organizations and ProviderLocations using the Surescripts network and the end-user system locations. Directory Management is critical to the accurate, effective and efficient day-to-day operations of the Surescripts network. Delays in directory updates or vendor sourced information leads to message misrouting, delayed delivery, and transmission of electronic messages via alternative delivery methods such as fax, or message delivery failures.

Surescripts provides the following mechanisms for customers to manage their directories:

  • Surescripts Admin Console

  • Surescripts Directory Messaging API

  • Surescripts Directory Download Export Files

Customers are required to actively maintain their local directory information that is published to the Surescripts directory at regular intervals, either as specified by current application certification requirements, or within a week if requirements do not exist. Customers should also establish necessary workflow/protocols to ensure the Surescripts directory and the end-user local customer’s directory are audited on a regular bases. Customer ProviderLocation/Organization records updated prior to 7:00 PM ET will be included in the following day’s directory file output.

Customers are also required to retrieve Surescripts network directory information on a daily basis and apply updates to end-user systems as specified by published application certification requirements, or within 48 hours of the previous directory retrieval.

Formulary File Loads

Formulary files are an important part of e-prescribing and keeping this information up to date is required on a regular basis due to frequent changes. Customers should refer to current application certification requirements to determine appropriate formulary download frequency. Customers must ensure adequate bandwidth for download speed as well as off-hours process scheduling to download these weekly updates in support of e‑prescribing.
