XDR Message Details

Surescripts has implemented the XDR extensions defined in XDR and XDM for Direct Messaging Specification Version 1, 9 March 2011). To interact with the Surescripts network the customer must at a minimum implement the same extensions. This includes the Direct address block and the minimal meditative specification. Surescripts accepts full XDS messaging.

Message Identification

Certain clinical workflows are augmented if the senders and recipients can uniquely identify messages and indicate when a message is a reply to a previous message. The Web Services Addressing 1.0 - Core W3C recommendation defines the MessageID and RelatesTo headers. Surescripts uses these headers with the customer to uniquely identify each XDR message and its replies.

When a customer user sends an outbound message, Surescripts stores the message identifier. If Surescripts receives a reply to that message, the original message identifier is inserted into the RelatedTo SOAP header with the XDR that we send to the customer's XDR interface. Using this data, the reply message can be associated with the correct workflow item (such as an open referral) in the customer's system.
