Application Certification Requirements

General Formulary and Benefits Requirements

F.109: The participant shall implement within its application updated formulary information published by PBMs within one week of its availability.

Presentation of Formulary Status

Presentation of Formulary Alternatives Coverage and Copay

F.108: For active coverages, during the prescription writing process and prior to the summary screen, the application shall display, at a minimum, alternative medications when the target medication is an On-Formulary/Non-Preferred, Non-Formulary, or Non-Reimbursable medication.

Best Practice

  • The application may further filter alternatives by utilizing decision support logic (e.g. only showing alternatives applicable due to gender, age, diagnosis, etc.).
  • If the target medication is a brand medication and the prescriber indicates that generic substitution is permitted, the application does not need to show alternatives if the generic equivalent has a higher formulary status.

Payer Provided Alternatives

  1. If the payer provides a Formulary Alternatives List, the application shall show all alternatives that have a higher On-Formulary status.
    1. For Non-Formulary or Non-Reimbursable medications, show alternatives that are On-Formulary/Non-Preferred or On-Formulary/Preferred.
    2. For On-Formulary/Non-Preferred medications, show On-Formulary/Preferred.
    3. Do not show alternatives for Unknown medications.
  2. Show all strengths available that have a higher formulary status for the NDC provided in the Formulary Alternatives List.
  3. The alternatives provided in a Formulary Alternatives List shall be ordered according to the stated preference level in that list from high to low.
  4. Show all alternatives in the Formulary Alternatives List, even if the target medication is an OTC, multi-ingredient medication, or supply.
  5. The application shall not show drug database-provided alternatives if the PBM provides a Formulary Alternatives List.

    6. A provider vendor may display Therapeutic alternatives if a Payer Alternative List is present, but an alternative match for the target NDC does not exist.

  6. The application shall display formulary status and full or abbreviated retail copay data for all alternatives. Retail copay (defined as Pharmacy Type Retail, Any, or Null) data displayed shall be, at a minimum, the Copay Tier, Flat Copay Amount, or Percent Copay Rate.

    7. If the application cannot display the complete list of alternatives on one page, the application may allow the user to take action to view additional alternatives.

Drug Database Provided (Therapeutic Alternatives)

  1. If the PBM does not provide a Formulary Alternatives List, the application shall show all commercially available, generic, and branded alternatives with a higher On-Formulary status in the same therapeutic class as the target medication as determined by an approved medication data source.
    1. For Non-Formulary or Non-Reimbursable medications, show alternatives that are On-Formulary/Non-Preferred or On-Formulary/Preferred.
    2. For On-Formulary/Non-Preferred medications, show On-Formulary/Preferred.
    3. Do not show alternatives for Unknown medications.

    Best Practice

    • If the target medication is OTC, the application should not display alternatives.
    • If the target medication is a Supply, the application should not display alternatives.
    • If possible, the application should only show alternatives for the same route of administration as the target medication.
  2. If the target medication is single-ingredient, the application shall only display single-ingredient alternatives.
  3. If the target medication is multi-ingredient, the application shall only display multi-ingredient alternatives.

    4. The application shall display formulary status and full or abbreviated retail copay data for all alternatives. Retail copay (defined as Pharmacy Type Retail, Any, or Null) data displayed shall be, at a minimum, the Copay Tier, Flat Copay Amount, or Percent Copay Rate. If the application cannot display the complete list of alternatives on one page, the application may allow the user to take action to view additional alternatives.

Formulary Guidance Items

F.110: PBMs should provide a list of therapeutic alternatives for display using the Payer Specified Alternatives List.

F.111: PBMs should send coverage and copay information.

CMS requires the coverage and copay list for Medicare Part D participants.

F.112: Prescriber vendors should refrain from displaying unnecessary information to prescribers. For example, NDC numbers, Formulary IDs, Coverage IDs, PBM unique IDs.
